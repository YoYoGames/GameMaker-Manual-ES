Errores de compilación

El segundo tipo de error que puede ser reportado para su juego es el error del compilador. Un error del compilador ocurre cuando su juego encuentra algún tipo de error que el verificador de sintaxis puede no haber sido capaz de detectar - es muy fácil olvidar declarar una variable o escribir erróneamente un string como un real, por ejemplo - o cuando un error está relacionado con la forma en que ha configurado las herramientas de compilación de las Preferencias de Plataforma para la plataforma de destino. Estos errores serán detectados por GameMaker mientras el juego está siendo compilado y esta información también se mostrará en la salida de errores del compilador, como se muestra a continuación:

NOTA Estos errores no suelen mostrar una ventana y simplemente muestran "Build Failed" en la salida del compilador. Si aparece una ventana, suele tratarse de un error de ejecución, que se explica aquí.

Los mensajes de error del compilador seguirán todos el mismo formato:

[object] - [event] - [Line Number]: [error string]

Si el error se encuentra en un script entonces será simplemente:

[script] - [Line Number]: [error string]

A continuación, puede hacer doble clic en en cualquiera de las entradas de error del compilador para abrir el asset dado en la posición marcada como error, y luego puede utilizar la información contenida en el mensaje para localizar exactamente en qué parte de object o script se ha producido el error. Sin embargo, a veces estos errores pueden parecer un poco crípticos, así que a continuación puedes encontrar una lista completa de todos los errores y una breve explicación de lo que significan:
