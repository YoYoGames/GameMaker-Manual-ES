Tipos de datos de las plumas

Feather utiliza data types para proporcionar una comprobación inteligente de la sintaxis al escribir el código, asegurándose de que no se utiliza el data type incorrecto para una variable o parámetro de función.

También le permite especificar la dirección data types para los parámetros y valores de retorno de sus propias funciones de script, utilizando los comentarios de JSDoc.

Las etiquetas JSDoc @param y @return permiten especificar cualquiera de los siguientes tipos:

NOTA Los tipos de JSDoc no distinguen entre mayúsculas y minúsculas, lo que significa que Id.DsList e id.dslist son lo mismo.

NOTA * La columna de Ejemplos de Especificadores enumera algunos ejemplos de especificadores, pero no todos. Por ejemplo, puede haber más tipos posibles en los grupos Id, Activo y Constante, que no se enumeran aquí.

Es posible que vea un tipo Any* en IDE cuando utilice Feather, lo que indica que el tipo del identificador (que puede ser una variable, un parámetro, un valor de retorno, etc.) no puede ser discernido por Feather en este momento. Cuando se añada más código para ese identificador, es posible que pueda asumir un tipo específico.

Especificadores

Se añade un especificador después de la base data type utilizando un punto ., para especificar el tipo exacto de datos en ese grupo.

Tipos como Id.DsList, Asset.GMObject y Constant.Color utilizan especificadores.

Tipos de colección

Tipos como Array y Id.DsList, que son data structures que contienen múltiples valores, pueden especificar un único data type para todo su contenido.

Para ello se utilizan los tipos de colección, que se adjuntan al tipo mediante paréntesis angulares <>.

Por ejemplo, un array que contenga strings sería Array<String>, y una Lista DS, que contenga un Array, que contenga Reales, sería Id.DsList<Array<Real>>.
