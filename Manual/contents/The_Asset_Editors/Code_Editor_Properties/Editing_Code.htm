Código de edición

Cuando empiece a utilizar el editor script, sólo tendrá que escribir el código que desee para el script asset o para el evento, siguiendo la sintaxis GML correcta. Mientras lo haces, recibirás ayuda de la ventana de autocompletar:

Esta ventana aparecerá mientras está escribiendo su código (normalmente tras una ligera pausa, pero esto puede configurarse en las Preferencias) y le ofrecerá una selección de las respuestas "más adecuadas" en orden alfabético.

Al utilizar el editor de código, puedes hacer clic en y arrastrar el ratón para seleccionar una fila de código o mantener pulsado y luego hacer clic en y arrastrar para seleccionar columnas de código. Todo lo que se seleccione de este modo puede copiarse, cortarse y pegarse utilizando los atajos de teclado estándar de / + " C", / + "X" y / + "V".

Vale la pena señalar que puedes comentar tu código de varias maneras. Comentar tu código es muy importante para mantener la productividad, ya que mantiene las cosas claras y separa las secciones de código de tal manera que es obvio lo que hace cada parte y por qué está ahí. También significa que, si trabajas en equipo, puedes dejar notas para otros miembros y que ellos sepan exactamente lo que has hecho o para qué sirve una sección específica del código. Incluso si eres un desarrollador en solitario, dejar comentarios en tu código es importante, especialmente si tienes que dejar el proyecto y volver a él más adelante.

Comentar tu código es sencillo, ya que sólo tienes que utilizar "//" antes de cualquier línea de texto o puedes comentar párrafos enteros utilizando "/* text here */". Incluso hay dos atajos de teclado para comentar y descomentar las líneas de código seleccionadas: / + "K" para comentar, y / + + "K" para descomentar. La siguiente imagen ilustra el código escrito utilizando estos dos métodos, así como algunos comentarios de JS Doc (explicados con más detalle a continuación):

Los scripts también pueden tener ciertos comentarios de estilo JSDoc añadidos para definir las propiedades de las funciones que contienen para que cuando uses las funciones en tu código aparezcan en autocompletar junto con sus argumentos y otros detalles. Puedes encontrar más información sobre esto en la sección de comentarios JSDoc Script .

Por defecto, todo el código que se escriba tendrá iconos de plegado de código junto a cada conjunto de llaves coincidentes {}. Estos tienen la forma de unos pequeños iconos de más + o menos -, y al hacer clic en ellos se "doblará" el código dentro de las llaves correspondientes para que ocupe menos espacio visual.

También puedes establecer manualmente regiones de plegado de código utilizando las etiquetas especiales #region para marcar una sección de código. El funcionamiento es el siguiente: añades una etiqueta #region (junto con cualquier texto de comentario) en la parte superior de una sección de código y luego añades la correspondiente etiqueta #endregion en la parte inferior de la sección y entonces puedes "plegar" el código, es decir, ocultarlo para poder reducir el número de líneas de código visibles en el editor en cualquier momento:

El código real es algo así:

#region This is an example region

width = sprite_get_width(sprite);
height = sprite_get_height(sprite);
xoff = sprite_get_xoffset(sprite);
yoff = sprite_get_yoffset(sprite);

#endregion Further comment here if required

Como habrá observado, algunas partes del texto de script están coloreadas. El editor de script "conoce" las variables y funciones existentes en objects, etc... y las codifica con colores. La codificación por colores ayuda mucho a evitar errores, ya que puedes ver inmediatamente si has escrito mal algún nombre o has utilizado una palabra clave como variable, o tienes un error de sintaxis. Sin embargo, si no te gusta la codificación por colores, en las Preferencias puedes switch activarla y desactivarla (también puedes usar F10 para activarla o desactivarla en el editor) así como personalizar los colores que se utilizan.

También puede hacer clic con el botón derecho del ratón en en cualquier lugar del editor para abrir el siguiente menú:

Aparte de las funciones habituales para cortar, copiar y pegar líneas de código, también tiene la opción de comentar o descomentar cualquier texto seleccionado, añadir o eliminar sangrías, así como dividir la ventana de código en 1, 2 o 3 paneles separados (tenga en cuenta que las opciones relacionadas con #regions no estarán disponibles si no ha definido ninguna en el código que se está editando).

La última opción para dividir en paneles dividirá la ventana actual en diferentes partes en las que podrá arrastrar un script o un evento y hacer que se muestre junto a los demás para compararlos y editarlos.

Esto es excepcionalmente útil para comparar dos scripts o para ver scripts conectados al mismo tiempo, sin necesitar otra ventana o tener que switch constantemente entre pestañas.

Las otras opciones de este menú son alternar un breakpoint para usarlo con el Depurador, y convertir el código de GML en GML Visual o viceversa. Tenga en cuenta que si convierte el código a GML Visual, entonces no creará acciones individuales para cada línea de código, sino que utilizará una única acción de código para mantener todo lo que estaba previamente en el evento o script.
