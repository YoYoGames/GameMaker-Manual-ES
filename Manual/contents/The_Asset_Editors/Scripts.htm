El editor de Script

Esta sección trata del editor de scripts (también llamado editor de texto) que es donde se escribe todo el código que su juego requerirá para funcionar. Este editor se utiliza para crear script assets, así como para codificar eventos de objetos, para añadir código de creación de salas, y para muchas otras cosas.

Esta sección simplemente detalla cómo funciona el editor script, y no cómo crear script assets ni cómo utilizar GML Code o GML Visual para hacer scripts. Para ello, consulte las siguientes secciones:

  • Código GML Scripts
  • Guiones visuales GML

Al crear un script asset, es posible que se le pida que elija entre GML Visual y GML Code. Consulte Lenguaje GameMaker para obtener más información.

El editor script es el lugar donde se escribe el código GameMaker Language, que es el lenguaje de programación incorporado que utiliza GameMaker (ver la sección Lenguaje de GameMaker para más detalles). Una vez que te familiarices con GameMaker y quieras utilizarlo en toda su extensión, es aconsejable que empieces a aprender a utilizar este lenguaje, ya que amplía enormemente tus posibilidades a la hora de crear juegos.

Se puede cambiar el nombre de un script asset haciendo clic con el botón derecho del ratón en en el Navegador de Activos y seleccionando Renombrar, pero tenga en cuenta que el nombre de script debe ajustarse a las reglas de scripting para assets, por lo que sólo pueden ser alfanuméricos, deben comenzar con una letra y el único símbolo que pueden contener es el símbolo de la barra inferior "_". El editor de script en otros lugares no puede ser renombrado y tendrá un nombre específico para lo que se está editando, como un evento object por ejemplo.

Cuando abra la ventana del editor de script tendrá las siguientes opciones y diseño:

Pestaña Script / Evento

El editor de script se abre en una ventana con pestañas en la parte superior para permitirle tener varios scripts o eventos en una sola ventana (aunque este comportamiento puede cambiarse desde las Preferencias para dar una nueva ventana a cada script o evento). Puede hacer clic en en una pestaña y luego arrastrarla para cambiar el orden de las pestañas, o -si lo prefiere- puede sacarla de la ventana actual y colocarla en workspace para crear una nueva ventana para ese script (o añadirla a una ventana diferente), y también puedes maximizar el editor de script para crear un nuevo workspace también. Si arrastras una pestaña de script fuera de la ventana del ventana del IDE, se creará un nuevo IDE para albergar este recurso script, y se puede utilizar como lo haría con la ventana principal.

Tenga en cuenta que si está editando el código de un evento de objeto en el editor script y ha maximizado el editor script o lo tiene en una ventana separada o workspace, entonces el menú del editor del botón derecho tendrá algunas opciones adicionales:

  • Restaurar al espacio de trabajo: Esto sacará el editor de script de un estado maximizado/espacio de trabajo y lo volverá a encadenar a la object en el workspace.
  • Vayaa Objeto: Esto le llevará a la workspace en la que se encuentra el object con el código y se centrará en el object.
  • Añadir /Abrir Evento: Esto le permite añadir un nuevo evento a la object a la que pertenece la actual script, y abrirá una nueva pestaña de código en el Editor Script para el evento añadido. Si el evento seleccionado ya tiene código, éste se abrirá en una nueva pestaña. se abrirá en una nueva pestaña.

Para obtener información sobre el resto de las opciones del menú del botón derecho del ratón, consulte la sección sobre Edición, más adelante.

Canalón

El canalón se utiliza para mostrar los números de línea de su código y también para transmitir algunas piezas específicas de información sobre el estado del código. Lo más importante es que si cometes un error tipográfico o construyes el código de forma incorrecta, el IDE de GameMaker te informará del problema marcando la línea de código con un signo de exclamación rojo que muestra un error de sintaxis. El canalón también marcará cualquier línea del script a la que se le haya añadido un punto de interrupción . Un breakpoint es simplemente un lugar en el script donde quieres que el depurador detenga la ejecución de tu juego cuando se alcanza. Puedes activar un breakpoint desde cualquier línea de cualquier evento de script o object pulsando "f9" o utilizando el menú del botón derecho . Buscar / Reemplazar

Al activar esta opción, la función de búsqueda sólo resaltará aquellos strings que coincidan con la entrada completa string. Por ejemplo, con la función desactivada, una búsqueda de "random" mostrará todas las palabras que contengan este string - como irandom(), o randomise() - mientras que si se activa sólo se mostrará la función random().
Cuando esta opción está activada, le está diciendo a GameMaker que compruebe no sólo el contenido de la búsqueda string, sino también el caso. Por ejemplo, si tienes un sprite llamado "spr_Dog" y haces una búsqueda de " perro" con esta opción desactivada, entonces el sprite string se resaltará, sin embargo si la opción está activada no lo hará ya que "perro" ya no se considera igual que "perro".

Tenga en cuenta que el término "siguiente" se considera el siguiente después de la posición actual del cursor, y puede saltar a otros diferentes utilizando los botones de flecha en la parte superior derecha de la ventana de búsqueda.

Tenga en cuenta que si desea realizar una búsqueda global (es decir, buscar en todo el proyecto y no en el script o evento actual), puede pulsar / + + "F".

Editor de código principal

El editor de código principal es donde escribirás todo tu código para crear el script asset o llenar un evento object. La edición real de tu código está cubierta en la sección de Edición más adelante en esta página.

Información

La barra de información en la parte inferior de IDE le muestra el número de línea en el que se encuentra y la posición en la línea. También es donde puede ver el ayudante de código, que es una línea de texto que muestra la función que está editando actualmente junto con con los argumentos que requiere. A medida que rellene la función en el editor, los argumentos se resaltarán para mostrarle cuál está editando actualmente.

Tenga en cuenta que cualquier argumento opcional en sus funciones script se mostrará con [] paréntesis alrededor de ellos. Si ha utilizado los comentarios de JSDoc Script dentro de un script asset entonces la información que haya suministrado también se mostrará aquí.
