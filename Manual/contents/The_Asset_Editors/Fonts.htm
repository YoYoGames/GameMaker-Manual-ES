El editor de Font

Cuando usted quiere dibujar un texto en su juego este texto se dibuja en una fuente estándar Arial de 12 puntos, pero para hacer textos más interesantes o de aspecto único probablemente querrá utilizar diferentes fonts. Por lo tanto, para utilizar diferentes fonts que tenga en su ordenador debe crear un activo de fuente en GameMaker. Esto se hace haciendo clic con el botón derecho del ratón en en el Navegador de Activos y seleccionando "Crear" -> "Fuente". Para cada font asset que cree, debe especificar un tipo particular de font de su ordenador que luego puede ser utilizado en su juego utilizando la acción o el código para establecer un font para dibujar.

Cuando se crea por primera vez un recurso font se abre la ventana del editor font con las siguientes opciones:

Detalles

Aquí puedes darle a tu font un nombre único (sólo con caracteres alfanuméricos y el símbolo "_" debajo de la barra), así como elegir el font a utilizar. El font debe ser instalado en su máquina y debido a cuestiones de licencia, GameMaker no almacena los archivos reales font con el archivo de proyecto por defecto, sino que el font se renderiza en una página de textura como una serie de glifos de mapa de bits. La página de textura será asignada a un grupo de texturas, que puede elegir desde la opción de Grupo aquí también.

Si desea distribuir el archivo font real (sólo *.ttf o *.otf ) junto con el proyecto al hacer un Marketplace asset o por cualquier otra razón, puede marcar la opción Copiar al proyecto. Este abrirá el explorador de archivos donde podrá seleccionar el archivo font y se añadirá a los archivos incluidos en el proyecto.

¡IMPORTANTE! Sólo debe utilizar fonts para cuya redistribución o utilización en su proyecto esté legalmente autorizado.

Presets

En esta sección puede establecer el tamaño (en puntos - el valor predeterminado es 12pt) y si el font debe dibujarse en negrita o cursiva o si se le aplica antialiasing. La ventana de vista previa de la derecha le mostrará el texto aproximadamente como se verá en su juego con el tamaño y las transformaciones que haya especificado.


Controles de la gama

Al crear un recurso font es raro que quieras incluir todos los glifos de font. Por eso GameMaker utiliza por defecto sólo los caracteres del 32 al 127 del font elegido, ya que esto nos da los números y las letras minúsculas y mayúsculas junto con ciertos signos de puntuación. Sin embargo, puedes cambiar el rango de caracteres utilizado para ayudar a optimizar tus juegos, es decir, si sólo necesitas los números de un font específico, entonces selecciona sólo los números. Para ello Para ello, primero debe hacer clic en el botón Eliminar para borrar el rango actual (puede que tenga que seleccionar el rango para que esto funcione haciendo clic en en la ventana de en la ventana de vista previa) y luego hacer clic en el botón Añadir para agregar un nuevo rango. Esto abrirá la siguiente ventana encadenada:

Esta ventana tiene algunos botones que le ayudarán a establecer un rango estándar para su fuente:

- El rango normal de 32 a 127

- El rango ASCII de 0 a 255

- El rango de Dígitos que sólo contiene los 10 dígitos

- El rango de Letras que contiene sólo todas las letras mayúsculas y minúsculas

Se pueden utilizar otros rangos escribiendo el primer y el último índice de caracteres del rango que se desea establecer en las casillas de entrada del rango de caracteres (si un carácter no se encuentra en el rango, se sustituye por un espacio). Además de estos botones y cuadros de entrada dispone de otras dos opciones especialmente útiles:

  • Desde el código: si haces clic en este botón y luego en OK, GameMaker analizará automáticamente el código de tu juego para strings y luego creará diferentes rangos de caracteres para cubrir todo el texto de tu juego. Tenga en cuenta que busca todos los strings (ya sea dentro de "" o '') y por lo tanto también puede incluir nombres de archivos en los rangos de caracteres.
  • Desde archivo: funciona de forma similar a la opción "desde código" explicada anteriormente. Si hace clic en ella, se le pedirá que proporcione un archivo y, una vez hecho esto, deberá hacer clic en Aceptar. GameMaker analizará el archivo archivo y creará rangos de caracteres para el texto que se encuentra en él.

Tenga en cuenta que puede editar manualmente el rango de texto añadiendo o eliminando glifos de texto en la ventana de vista previa directamente.

Controles de textura

Como se mencionó anteriormente (en la sección de Detalles de la Fuente) por defecto todos los glifos de font se renderizarán en una página de textura como parte de un grupo de texturas para convertirlos en recursos gráficos, y cuando el juego se compila el font se añadirá a una textura temporal para crear una sola imagen con todos los glifos font requeridos y luego esto se añadirá al grupo de texturas principal para la exportación. Los botones aquí son para ajustar cómo se genera la textura font o para regenerar una nueva textura font.

El botón Opciones abrirá las opciones avanzadas de textura de font:

Estas opciones pueden alterar, a veces radicalmente, la forma en que un font se mostrará en el juego, y no todos los fonts requerirán los mismos ajustes. Las siguientes opciones están diseñadas para ofrecerte la mayor personalización posible para tu fuente:

  • Hinting: Font hinting (también conocido como instructing) es el uso de instrucciones matemáticas para ajustar la visualización de un contorno font de manera que se alinee con una cuadrícula rasterizada. En resoluciones de pantalla bajas, el hinting es crítico para producir un texto claro y legible, y aquí puedes seleccionar entre diferentes algoritmos de hinting para obtener el mejor resultado posible. El valor por defecto es "normal".
  • Opciones de glifos: Estas opciones se pueden utilizar para cambiar la forma en que se cargan los glifos, activando/desactivando diferentes operaciones. Estas son:
    1. Desactivar Hinting: Esto deshabilitará el hinting nativo de fonts y el auto-hinter.
    2. DesactivarAuto-Hinter: Esto deshabilitará el auto-hinter y en su lugar utilizará el hinter nativo de fonts (si tiene uno). Tenga en cuenta que esto anula la opción "Prefer Auto-hinter".
    3. Preferir Auto-Hinter: Esto le dirá a GameMaker que prefiera usar el auto-hinter sobre cualquier hinter nativo que el font pueda tener. Tenga en cuenta que esto es anulado por las opciones "Disable Auto-hinter" y "Disable Hinting".
    4. Deformación: Al activar esta opción, los glifos se deforman ligeramente para mejorar la alineación de la cuadrícula. Tenga en cuenta que esto requiere el puntero automático y un modo de insinuación "Normal".
    5. Sin escala: Cuando esta opción está marcada, mantendrá los glifos en las unidades de font e ignorará el tamaño solicitado. Esto implica que el hinting está desactivado.
    El estado por defecto de todas las opciones anteriores es desactivado.
  • Intérprete: Aquí puede seleccionar la versión de TrueType que GameMaker debe utilizar para el renderizado de font. Por defecto es TrueType v40.
  • Redondeo: Esta opción es para establecer el método de redondeo cuando se convierte de un tamaño decimal de punto fijo a un entero. Por defecto es "floor".
  • Aplicar kerning: Controla si los datos de kerning del archivo font se utilizan para el renderizado en el juego. Debe desactivarse si los datos de interletraje del archivo font son incorrectos y hacen que font no se muestre correctamente.

El botón Regenerar puede utilizarse para regenerar la textura de font. Esto es necesario si quieres actualizar cualquier cambio que hayas hecho en un font existente, o si has creado un nuevo font y quieres ver cómo queda en la Ventana de Vista Previa. Tenga en cuenta que también puede regenerar fonts desde el menú del botón derecho para el font (o fonts) en el Navegador Asset.

Es importante tener en cuenta que cuando se crea por primera vez un nuevo font, si no se regenera la textura en el editor, la textura se creará la primera vez que se ejecute o pruebe el juego, pero esa es la única vez que se generará automáticamente. Esto significa que una vez que un font tiene una textura generada, cualquier cambio realizado en el recurso font requerirá que se regenere el font para que los cambios sean visibles en el juego, de lo contrario se seguirá utilizando la textura "antigua" font.

GameMaker renderizará la textura font generada como un archivo PNG junto con el resto de los archivos del proyecto, y si usted carga el proyecto y la base font no se encuentra, entonces el juego seguirá mostrando el texto correctamente usando la textura almacenada. Sin embargo, la falta de font significará que ya no podrás cambiar ninguna de sus propiedades, salvo su nombre, la familia font y el grupo de texturas (desde el editor de grupos de texturas).

Modo de vista previa

En esta sección puede elegir que la Ventana de Vista Previa muestre algún texto de muestra del font seleccionado, o la imagen de textura renderizada que se ha generado para el font.

Vista previa

La ventana Ventana de vista previa es donde puede ver una vista previa de la font, ya sea como una muestra de texto, o como una textura. Cuando el Modo de previsualización está ajustado a Muestra, puedes escribir cualquier texto que quieras probar directamente en la ventana de vista previa para ver cómo queda, mientras que si seleccionas Vista previa del juego mostrará la imagen de la textura generada para el font. Si no hay ninguna imagen presente, entonces debe hacer clic en el botón Regenerar para generar la textura.

Tenga en cuenta que la imagen de la textura puede ser diferente a la muestra, ya que la textura no se regenera a menos que se indique explícitamente. Esto significa que después de haber generado inicialmente una textura font, los cambios realizados en el recurso font no se trasladarán al juego a menos que se regenere la textura.
