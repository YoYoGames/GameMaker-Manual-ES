El editor de Extension

Un paquete de extensión puede utilizarse para añadir una serie de funciones y constantes adicionales a GameMaker Language (GML) y transferirlas fácilmente entre proyectos. Las funciones de Extension también pueden vincularse a SDKs externos o a código fuente escrito en otros lenguajes (como Java para Android, Objective-C para iOS, etc.) que pueden utilizarse para ampliar la funcionalidad de tu juego (normalmente se utiliza para implementar APIs de terceros, como para anuncios, compras dentro de la aplicación, y más).

Las extensiones se pueden utilizar para crear una biblioteca de funciones que utilices con frecuencia en todos tus proyectos, así que en lugar de tener que importar el mismo scripts en cada proyecto, puedes simplemente añadir un extension asset que contenga una colección de tus funciones y constantes favoritas. Dichas funciones también pueden ser específicas para una determinada plataforma, y puedes elegir fácilmente en qué plataformas se puede utilizar un extension.

Para ver un ejemplo paso a paso de cómo crear un extension, consulte: Creación de una extensión

Para desactivar extensions en función de la plataforma de destino, consulte: ¿Cómo desactivar Extensions?

Importación de extensiones

Normalmente, los paquetes de terceros extensions provienen del GameMaker Marketplace, y aquellos a los que se ha suscrito aparecerán en la ventana de la Biblioteca del Marketplace, desde donde se pueden instalar. Si usted no obtuvo su paquete extension desde el Marketplace, todavía puede añadirlo a un proyecto haciendo clic con el botón derecho del ratón en en cualquier lugar del Navegador de Activos y seleccionando Añadir Existente, luego navegando hasta la ubicación de la extensión para seleccionarla.

NOTA Si tiene un paquete .yymps/.yymp/.gmez, puede arrastrarlo a IDE para importarlo, o ir al menú "Herramientas" y seleccionar "Importar paquete local".

Plataformas de destino

Una vez que haya añadido un extension al Navegador Asset, deberá darle un nombre y un número de versión y también establecer a qué plataformas desea exportarlo. Desmarcando las plataformas seleccionadas, puedes decirle a GameMaker que no exporte el extension como parte del ejecutable para esas plataformas. También debes asegurarte de que las propiedades de extension coinciden con la(s) plataforma(s) de destino seleccionada(s), y seleccionar objetivos de exportación compatibles para los archivos de extension basándote en lo siguiente:

  • Todos los objetivos: GML extensions. Estos son extensions realizados con el GameMaker Language (GML) y son compatibles con todas las plataformas de destino. (Tenga en cuenta que este es el único tipo de extension que soporta actualmente el objetivo Opera GX.)
  • Windows (Estándar, UWP, Xbox): Puedes utilizar archivos Windows DLL (Dynamic Link Libraries). Tenga en cuenta que si ha marcado la opción "Usar x64 Windows Runtime" en las opciones de juego de Windows, todas las DLL del proyecto deben ser compatibles con los 64 bits. Las DLL de 32 bits no funcionarán con un ejecutable de 64 bits.
  • macOS: El objetivo macOS puede utilizar los archivos estándar DyLib para extensions.
  • Ubuntu (Linux): Para el objetivo Ubuntu necesita asegurarse de que su extension utiliza los archivos .so.
  • HTML5: Estas plataformas pueden utilizar JavaScript extensions.
  • Android: Esta plataforma requiere que se escriba un extension en Java o un Android Framework SDK.
  • iOS / tvOS: Extensions para esta plataforma está escrito en Objective C y requiere un archivo *.mm y otro *.h, o si se utiliza algún framework, un archivo nativo precompilado *.a.
  • PlayStation: Para PlayStation su extensions debe ser PRX archivos, con cada objetivo que requiere un PRX diferente para ajustarse a las especificaciones del objetivo (PS3, PS4 o PS5). Si trabajas en una PSVita, el extension debe ser SUPRX.

NOTA Algunos extensions deben seguir convenciones de nomenclatura específicas para que GameMaker identifique correctamente la plataforma y la arquitectura de la CPU para la que están diseñados. Consulte las convenciones de nomenclatura para obtener más información.

Dado que un extension puede tener varios archivos asociados, es posible crear un único extension para múltiples plataformas, en cuyo caso deberá especificar todas las plataformas compatibles en el editor principal de extension (es decir: marque la casilla que aparece junto a ellas) y, a continuación, en las propiedades de los archivos individuales de cada parte del extension, establezca la plataforma a la que deben exportarse esos archivos.

Para cambiar la plataforma de destino de un archivo extension o marcador de posición específico, primero debe abrir las propiedades de extension (haga doble clic en el extension en el navegador Asset ), y luego, una vez en las propiedades de extension, seleccione el archivo de recursos de la derecha que desee editar. Descomprimir al construir

La opción "Descomprimir al construir" puede utilizarse para los archivos comprimidos destinados a una plataforma de destino específica. Si esta opción está activada, el contenido de ese archivo se extraerá en el directorio de archivos incluidos del paquete construido, pero sólo después de ser copiado en la plataforma de destino.

Por ejemplo, si habilita esto para un archivo .zip en su extension y luego inicia una compilación de macOS desde Windows, ese archivo .zip se copiará tal cual desde Windows a macOS y luego se descomprimirá durante la compilación en la plataforma de destino (que, en este contexto, es macOS).

Llamada a las funciones de Extension

Una vez que hayas configurado tu extension, puedes utilizar las funciones y constantes para él igual que lo harías con las funciones y constantes incorporadas que se utilizan en el GameMaker Language. Si tu juego es multiplataforma (por ejemplo, está diseñado para ejecutarse en objetivos Windows y Android ), entonces debes utilizar las constantes y funciones del sistema operativo para comprobar la plataforma actual antes de llamar a las funciones específicas de extension para evitar problemas, por ejemplo:

switch(os_type)
{
    case os_android:
        call_android_extension_function(x, y, z);
    break;

    case os_windows:
        call_windows_extension_function(x, y, z);
    break;
}

Tenga en cuenta que para diferentes plataformas su extensions puede compartir los nombres de las funciones, por lo que si tiene un extension con archivos distintos para (por ejemplo) hacer aparecer un mensaje en Windows y para Android, ambos archivos extension pueden compartir los mismos nombres de las funciones, siempre y cuando ambos estén configurados para exportar sólo a sus respectivas plataformas.

Extensiones oficiales de terceros

Las versiones anteriores de GameMaker tenían mucha funcionalidad de terceros incorporada, pero esto era ineficiente y difícil de mantener, ya que si alguna funcionalidad específica era cambiada por el tercero, entonces todo el IDE y el Runtime de GameMaker tendría que ser actualizado. Por ello, las versiones actuales han adoptado el uso de paquetes extensions para añadir soporte de terceros a sus juegos. Esto significa que puedes añadir sólo aquellas funciones extra que necesites y que incluso puedes adaptarlas y editarlas para que se ajusten a tus necesidades específicas.

La mayoría de estos extensions se descargarán e instalarán por ti cuando selecciones una opción de juego específica (como Facebook o Google Mobile Ads, por ejemplo), pero puedes encontrar más información sobre ellos en la sección correspondiente a cada plataforma en la Base de conocimientos de GameMaker:

  • Plataformas y configuración deGameMaker

Puede encontrar los propios extensions en la siguiente página Marketplace:

  • Oficial Marketplace Activos

Tenga en cuenta que estos extensions vienen con una demo y un archivo de ayuda (en formato *.pdf ) que explican cómo usarlos, y como tal sus funciones no se cubren en este manual.
