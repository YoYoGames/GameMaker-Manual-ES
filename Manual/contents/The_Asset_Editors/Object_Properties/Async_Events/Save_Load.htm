Guardar / Cargar

Este evento será activado por la llamada de retorno de ciertas funciones relacionadas con la carga y guardar buffers en archivos, así como al cargar o descargar audio de la memoria. El evento en sí mismo contendrá el mapa async_load DS, que se rellenará con las claves necesarias para la función específica. Éstas se enumeran en las secciones siguientes.

Buffers

Cuando se utilizan las funciones buffer_save_async() o buffer_load_async() un evento asíncrono se activará cuando la transferencia de datos haya finalizado. Este evento rellenará el mapa async_load con los siguientes pares clave/valor

  • "id": el ID de la función asíncrona devuelto por la función utilizada.
  • "status": devolverá true si los datos se han guardado/cargado correctamente, y false en caso contrario.

Esto le permite sondear el progreso de guardado/carga y mostrar un mensaje o cambiar rooms etc... cuando el proceso se ha completado.

Grupos de audio

Cuando se trabaja con Grupos de Audio, se pueden cargar y descargar de la memoria utilizando las funciones audio_group_load() y audio_group_unload(). Cuando se utiliza la función de carga, se activará este evento cuando el conjunto completo de archivos de audio para el grupo se haya cargado en la memoria y rellenará el mapa con los siguientes pares clave/valor:

  • "type": nos indica el tipo de evento al que se llama y será "audiogroup_load" para cargar el audio.
  • "group_id": devolverá el ID del grupo de audio que se ha cargado (tal y como se ha definido en el Editor de grupos de audio).

Cuando se haya cargado todo el audio de un grupo, este evento se activará y podrá utilizarse para cambiar rooms, o reproducir una pista musical, etc.
