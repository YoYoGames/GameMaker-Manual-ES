Imagen cargada

Este evento se activa cuando se carga una imagen en GameMaker, ya sea en HTML5, o a través de una URL. Por ejemplo, digamos que usted quiere cargar una imagen de sprite, y sólo cambiar el actual sprite para la instancia a la nueva después de que ha cargado. Usted tendría algo como esto en el evento Create (o cualquier otro evento):

var _url = "http://www.angusgames.com/game/background1.png";
spr = sprite_add(_url, 0, false, false, 0, 0);

Esto comenzará a cargar la imagen en el dispositivo o en el navegador, pero no bloqueará a GameMaker mientras espera que el archivo sea cargado. En su lugar, GameMaker seguirá funcionando normalmente hasta que la imagen se cargue y el callback dispare el Evento de Imagen Cargada, donde un Mapa DS es creado y almacenado en la variable especial async_load. El mapa contiene la siguiente información:

  • "filename": La dirección completa path al archivo solicitado.
  • "id": El ID del recurso que has cargado. Será el mismo que la variable a la que has asignado el recurso.
  • "status": Devuelve un valor inferior a 0 para un error.

A continuación, se asignaría la imagen recién cargada a un sprite en este evento. El siguiente ejemplo de código demuestra cómo se utilizaría la información devuelta:

if ds_map_find_value(async_load, "id") == spr
{
    if ds_map_find_value(async_load, "status") >= 0
    {
        sprite_index = spr;
    }
}

El código anterior comprobará primero la id del mapa DS que se ha creado, y luego comprobará la status de la llamada de retorno. Si el valor es mayor o igual a 0 (señalando el éxito) el resultado de la devolución de llamada se utilizará para establecer el índice sprite a la imagen recién cargada.
