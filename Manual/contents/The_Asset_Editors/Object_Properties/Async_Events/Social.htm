Social

Este evento sólo puede ser activado por los callbacksde varias Funciones Sociales específicas y Funciones UWP Xbox Live y devolverá un Mapa DS almacenado en la variable async_loadque contiene diferentes pares clave/valor dependiendo de la llamada de retorno de la función que ha activado el evento.

Este evento también puede ser activado por la reproducción de vídeo.

El mapa DS contendrá una serie de claves, la más importante de las cuales es la clave "id". Ésta devolverá una constante que puede ser comprobada en el código para determinar cuál de las muchas devoluciones de llamada ha recibido el evento. Una vez que hayas analizado esta clave y comparado el valor devuelto con las constantes disponibles, puedes continuar extrayendo el resto de la información del mapa.

Las páginas de funciones sobre Logros y Tablas de Clasificación en la sección de Referencia detallan las funciones y las devoluciones de llamada, así como las constantes utilizadas para desencadenar este evento, pero también hay ciertas circunstancias en las que se desencadenará sin una llamada a la función desde el juego que se está ejecutando:

  • cuando el juego se lanza desde el tablero del sistema operativo
  • al completar un reto
  • cuando otro jugador completa un reto
  • cuando se inicia el juego desde el tablero del sistema operativo seleccionando un reto

Cualquiera de los callbacks anteriores activará el Evento Social y se generará un mapa async_load con los siguientes detalles (nótese que hay un valor clave diferente "id" para definir cada una de las diferentes razones por las que se llamó al evento, pero todos los demás contenidos del mapa son los mismos):

  • "id" - El valor de esta clave dependerá del tipo de llamada de retorno que desencadene el evento (principalmente utilizado por extensions).
  • "playerid" - La identificación del jugador para el desafío.
  • "issuerid" - La identificación de la persona que emitió el desafío.
  • "state" - El estado del desafío, que tendrá un valor de 0 a 3 (como string) para inválido, pendiente, completado o rechazado.
  • "message" - El mensaje de texto para el desafío.
  • "issueddate" - La fecha de emisión de la impugnación
  • "completeddate" - La fecha de finalización del desafío.
  • "type" - El tipo de desafío planteado.
  • "identifier" - El identificador string para el desafío.
  • "score" - El marcador empatado con el desafío.
