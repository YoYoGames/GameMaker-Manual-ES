Nube

El Evento de la Nube es aquel que se desencadena por la llamada de una de las cloud_*() funciones, como cloud_synchronise(). En realidad genera un mapa DS que es exclusivo de este evento y se almacena en la variable especial async_load (por favor, vea las funciones individuales para los ejemplos de código que explican el uso de este evento con más detalle). Este mapa DS tiene la siguiente estructura:

  • "status": Contiene el código de estado, donde un valor negativo denota un error, cuya descripción estará contenida en el "errorString". Un valor 0 (o un valor positivo) indica un éxito (ver más abajo los valores valores exactos), y la "resultString" contendrá los datos devueltos o un mensaje de éxito.
  • "id": El id que fue devuelto por la función llamada. Si disparas una serie de peticiones de cloud_ entonces necesitas saber a cuál estás recibiendo la respuesta, y así usarías este valor para compararlo con el valor que almacenó cuando originalmente envió la solicitud para encontrar la correcta.
  • "description": La descripción del último archivo cargado.
  • "resultString": Contiene un string que es el blob de datos devuelto desde la nube.
  • "errorString": devuelve un Error String para cualquier error.

El significado exacto de la entrada del mapa de "estado" devuelto se explica en la siguiente tabla:

Valor del estado errorString / resultString Descripción
-1 errorString = "No ha iniciado sesión en <SERVICIO>" No ha iniciado la sesión con éxito en el Servicio en la Nube dado
0 resultString = datos recuperados Nuevos datos del juego descargados de la nube (tras una llamada a cloud_synchronise() )
1 resultString = "AlreadySynchronized" No hay datos nuevos desde la última vez que se llamó a cloud_synchronise
2 resultString = "ConflictDeferral" Se encontró un conflicto, pero el jugador decidió ignorarlo
3 resultString = "GameUploadSuccess" los datos de cloud_string_save() o cloud_file_save() se subieron con éxito a la nube
-n errorString = Descripción del error Cualquier otro número negativo significa un fallo de sincronización
