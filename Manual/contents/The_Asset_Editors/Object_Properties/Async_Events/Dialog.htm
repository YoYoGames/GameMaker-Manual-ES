Diálogo

El evento asíncrono Dialog sólo se activa cuando recibe una llamada de retorno de una de las funciones funciones especiales de diálogo asíncrono, como get_login_async() (por favor ver esta función para un ejemplo de código extendido de cómo se utiliza este evento).

Estos eventos son los que piden algún tipo de entrada al usuario, que puede ser un nombre, datos de acceso, un número o un color, etc. Como a la mayoría de los dispositivos no les gusta sentarse en un loop esperando una respuesta, tienen que ser asíncronos y GameMaker GameMaker continuará ejecutándose en segundo plano mientras estas funciones tengan un diálogo abierto y hasta que obtengan la entrada del usuario requerida que desencadene este evento. Esto significa que si el diálogo está abierto mientras un juego está en progreso, debe tomar medidas para pausar el juego hasta que se reciba la llamada de retorno del diálogo.

Una vez más, se devuelve un Mapa DS con el ID que se mantiene en la variable especial async_load. Los valores contenidos en este mapa dependerán de la función utilizada, y deberá consultar las entradas individuales de cada función en este manual para obtener más detalles.

NOTA Las funciones de diálogo asíncronas son sólo para uso de depuración y pruebasy no deben utilizarse en juegos lanzados. Para ello, debes crear una interfaz de usuario personalizada para recibir las entradas de los jugadores, de modo que tengas un control total sobre el aspecto y el comportamiento de los diálogos.
