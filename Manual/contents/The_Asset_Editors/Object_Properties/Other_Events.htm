Los otros eventos

En esta página se explica cada uno de los diferentes eventos que se recogen en la categoría de Otros eventos en el Editor Object:

Sala exterior

Este evento se dispara cuando una instancia sale de la room, y se basa en una comprobación que se hace con la sprite asignada (y sus propiedades) de la instancia, de modo que aunque se haya configurado la escala x o y de la imagen a un valor distinto de uno, este evento sólo se activará cuando la totalidad de sprite esté fuera de la pantalla. Si la instancia no tiene sprite, entonces se utiliza la posición de la instancia y en el momento en que su posición x o y esté fuera del room entonces también se disparará el evento. Este evento es típicamente usado para cosas como balas, donde son destruidas una vez que dejan el room para que no termines con millones de balas volando infinitamente y causando que tu juego se ralentice. Tenga en cuenta que este evento sólo se activa una vez cuando la instancia deja el room inicialmente.

Intersección de límites

Este evento se dispara cuando una instancia "toca" el borde interior de room, y, al igual que el evento room exterior, se basa en una comprobación realizada con el sprite asignado a la instancia, de modo que incluso si se ha establecido la escala x o y de la imagen a un valor distinto de uno, este evento se disparará en el momento en que una parte de sprite cruce el límite de room. Si la instancia no tiene sprite, entonces se utiliza la posición de la instancia y en el momento en que su posición x o y intersecte el room límite, también se activará el evento. Puedes usar este evento para hacer cosas como decirle a una instancia que "rebote" hacia el room de nuevo en lugar de salir.

Vistas

Las vistas de cámara se definen normalmente en el editor de room y se utilizan para mostrar sólo una pequeña zona de una gran room en un momento dado. Esta categoría de eventos tiene dos subcategorías separadas(Outside View e Intersect Boundary), con con ocho eventos diferentes en cada una que corresponden a los ocho puertos de vista disponibles. Estas dos categorías funcionan exactamente igual que los respectivos eventos de room, sólo que toman el límite de la vista de la cámara como lo que la instancia debe comprobar en lugar de room.

Inicio del juego

Este evento especial se activa sólo una vez en todo el juego y sólo para aquellas instancias con acciones o código colocado en el evento. Estas instancias deben estar presentes en la primera room del juego y haber sido colocadas allí en el editor room, en lugar de que haber sido creadas dinámicamente mediante código o acciones. Hay que tener en cuenta que este evento ocurre después del evento de creación de todas las instancias y por lo tanto puede contener código o acciones con variables que han sido previamente definidas en ese evento. Este evento se define típicamente en un solo "controlador" object y se suele utilizar para inicializar variables globales, iniciar música, leer de archivos y cualquier otra cosa que normalmente se hace una sola vez al inicio de un juego.

Fin del juego

Al igual que el evento Game Start mencionado anteriormente, este evento sólo se activa una vez en todo el juego y es justo antes de que se cierre la ventana del juego. De nuevo, para que una instancia con este evento haga algo, tiene que estar en la página room cuando se cierra el juego. el juego se cierra. Tenga en cuenta que este evento en realidad sólo es viable para Windows, Ubuntu ( Linux) y los juegos de macOS como los dispositivos móviles pueden simplemente cerrar el juego sin dar GameMaker la oportunidad de llamar a este evento. Para los juegos de navegador hechos con el módulo HTML5, este evento debería funcionar cuando se cierra la pestaña del juego o el navegador, pero no todos los navegadores soportan esta característica (Firefox, Chrome y Safari lo hacen en el momento de escribir este artículo) por lo que hay que tener cuidado por lo que hay que tener cuidado al utilizarlo. Típicamente se usaría este evento para guardar datos.

Sala de inicio

Este evento ocurre para todas las instancias inicialmente en un room cuando se inicia el room. Ten en cuenta que ocurre después del evento de creación y por lo tanto puede depender de variables y cosas definidas previamente en ese evento.

Fin de la habitación

Este evento ocurre a todas las instancias que están en el room cuando el room termina. Este evento es muy útil para "limpiar" después de que un nivel haya terminado, por ejemplo puedes borrar los recursos cargados aquí, o eliminar un sistema de partículas para evitar fugas de memoria de memoria, etc...

Fin de la animación

En GameMaker sprites no son cosas estáticas ya que pueden tener sub-imágenes y ser animadas a diferentes velocidades. Cada fotograma de una animación (llamado sub-imagen) tiene su propio número, comenzando en 0, que puede ser comprobado en el código o a través de acciones, pero a veces todo lo que necesitas saber es cuando la animación ha terminado. Es entonces cuando se dispara este evento, justo al final de la animación cuando el índice de la sub-imagen muestra que se ha alcanzado el último fotograma. Este evento es realmente útil para muchas cosas, por ejemplo una explosión object donde puedes configurar la instancia para que se destruya a sí misma cuando se alcance el último fotograma de la animación.

Actualización de la animación

Este evento está diseñado sólo para ser usado con las Funciones de Animación Esquelética. Es un evento especial que se dispara cada paso en una instancia que utiliza una animación esquelética sprite, y está diseñado para "interceptar" los datos de los huesos después de que la orientación de los huesos haya sido calculada para el estado actual de la animación, pero antes de que estos datos sean comprometidos para ser utilizados para el dibujo. Esto le permite hacer modificaciones a los datos de los huesos usando las funciones apropiadas.

Evento de animación

Este evento está diseñado sólo para su uso con las Funciones de Animación Esquelética. Este evento se dispara en un objeto que tiene una animación esquelética sprite asignada, y donde el object está usando el dibujo por defecto para el sprite asignado, o el sprite está siendo dibujado vía draw_self(). Actualmente no se activará en una animación de esqueleto sprites dibujada de otra manera. manera. El evento está diseñado para capturar y transmitir datos que han sido asignados a eventos particulares en la animación sprite como se define en el editor utilizado para hacerlos.

Cuando el Evento de Animación se dispara, se creará un Mapa DS especial llamado event_data map (muy parecido al mapa async_load para un evento async). Este mapa tendrá todos los siguientes pares clave/valor que puedes analizar para obtener información sobre el evento:

  • "name": el nombre del evento (como se define en el programa de animación)
  • "track": el índice de la pista en la que se reproduce la animación (por defecto es 0)
  • "integer": un valor entero asociado al evento (como se define en el programa de animación, por defecto es 0)
  • "float": un valor flotante asociado al evento (como se define en el programa de animación, por defecto es 0,0)
  • "string": un valor de string asociado al evento (como se define en el programa de animación, por defecto es un string vacío "")

Camino terminado

Las rutas pueden ser una parte importante de la creación de cualquier juego, ya sea para un juego de tipo de defensa de la torre, o un shoot-em-up o lo que sea, y es a menudo muy importante que una instancia hace alguna acción particular cuando llega a el final de su path. Este evento detecta esto y se desencadena cuando sucede, lo que permite añadir código o acciones para hacer frente a las circunstancias, por ejemplo, en un juego de defensa de la torre que se utiliza para destruir la instancia y eliminar un punto de salud punto de salud de la meta object.

Eventos de los usuarios

Estos son eventos especiales que no son desencadenados por GameMaker automáticamente, sino que tienen que ser llamados implícitamente por usted usando código o acciones mientras el juego se está ejecutando. Hay 12 eventos de usuario en total y pueden contener código o acciones y pueden ser llamados en cualquier momento en el juego loop, haciéndolos muy útiles para crear tus propios eventos que sucedan cuando tú lo decidas.

Mensaje de difusión

Este evento se activa cada vez que un sprite o un sequence genera un mensaje de difusión string. Para más información, consulte la página sobre mensajes de difusión.
