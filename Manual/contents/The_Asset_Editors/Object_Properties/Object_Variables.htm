Object Variables

Al hacer clic en el botón Variables se abrirá la ventana Object Variables, que puede utilizar para generar cualquier número de variables antes del Evento de Creación del object:

En esta ventana puede hacer clic en el botón Add para añadir una nueva variable a la lista. A continuación, puede nombrar la variable, así como establecer su tipo y ajustar sus propiedades:

El nombre de la variable debe empezar por una letra y sólo puede contener letras, números y el símbolo de subrayado '_' con con una longitud máxima de 64 símbolos. Por lo tanto, las variables válidas son cosas como fish, foo_bar, num1, y las no válidas serían 6fish, foo bar , o *num. Una vez que hayas dado el nombre de la variable, tienes que establecer su tipo, que puede ser cualquiera de los siguientes: Vale la pena señalar que esta característica es particularmente útil cuando se trabaja con instancias padre/hijo - ya que puede tener un padre object con un conjunto de variables definidas y luego simplemente modificarlas en el hijo object (ver la sección Object Variables Y Niño Objects a continuación para más detalles) - y para cuando agrega instancias en el EditorRoom (ver la sección de Capas > Capa de Instancia), ya que también puede modificar estas variables para instancias individuales que se han colocado en el room.

A continuación se enumeran los diferentes tipos de variables que se pueden crear:

  • Real: Un número real es cualquier número que no es un entero y puede ser positivo o negativo. Así, 124.5, 45639.566546456, 0.9, -45.5 etc... son todos ejemplos de números reales. Todos los números reales se almacenan como valores de coma flotante de 64 bits (dobles). Los tipos de datos reales (y enteros) tienen una opción para establecer un rango de valores a la que se puede acceder haciendo clic en el botón Opciones :Cuando se marca esta opción se puede introducir un valor inicial y un valor final y en lugar de tener un valor fijo mostrado para la variable en la ventana de Variables Object, tendrá un deslizador que se ajusta a estos valores:
  • Número entero: Un entero es un número entero y puede ser positivo o negativo, por ejemplo 30004, 19, 0, -300. Tenga en cuenta que puede establecer un rango de valores para el entero de salida, como se explicó anteriormente para los números reales.
  • String: Un string es cualquier cosa que se haya colocado entre comillas dobles, por ejemplo "pez", "Hola Mundo" o "12345".
  • Booleano: Un booleano es un valor que es verdadero o falso. En la ventana de Variables del Editor Object, se muestra simplemente como una casilla que se marca para verdadero y se desmarca para falso.
  • Expresión: Una expresión es una frase matemática que puede contener números ordinarios, variables, strings, o funciones GML así como uno o más operadores. Por ejemplo sqrt(85 * 6) + 5.5 es una expresión.
  • Recurso: Un recurso es simplemente cualquiera de los recursos que ha definido en el Navegador Asset. Cuando seleccione este tipo de recurso, podrá hacer clic en el botón Abrir el Explorador Asset para abrir el Explorador Asset y seleccionar el recurso que necesite.
  • Lista: Seleccionar un tipo de entrada de lista significa que puede crear una selección de valores (pueden ser strings, reales, expresiones, etc...) de los cuales puede elegir uno o más para que la variable los devuelva. Para definir los elementos que irán en la lista, debe añadirlos primero en las Opciones (haga clic en el botón ):
    Podrá elegir el elemento como valor por defecto para la variable en la ventana principal desde el menú desplegable:
    Tenga en cuenta que si tiene marcada la opción Multiselección en las Opciones de la lista, la variable se convertirá en una matriz de todas las opciones seleccionadas:
  • Color: El tipo de color sirve para definir un valor de color que se almacenará en la variable. Puede introducir un número real (de 0 a 16777216, que tendrá un valor alfa de 255), o un valor hexadecimal (en el formato de $RRGGBBAA), o puede puede hacer doble clic en en la muestra de color para abrir el selector de color y definir el color allí.

Object Variables y Niño Objects

Una característica importante de las Variables Object es que son heredadas por cualquier hijo objects que tenga en el Navegador Asset (vea la sección sobre Padre Objects para más información), lo que significa que usted lo que significa que puedes elegir anular o cambiar alguna o todas ellas si lo deseas.

Cuando se crea un recurso hijo object de otro recurso object que tiene variables definidas para él, éstas aparecerán en la ventana de Variables Object de la siguiente manera:

Puede ver en la imagen superior que el Padre object tiene cuatro Variables Object y en la imagen inferior estas también se muestran, sólo que "atenuadas" y con el icono Heredado del Padre para indicar que han sido heredadas. Estas variables heredadas pueden ser editadas si hace clic en el botón Anular Variable , de modo que puede ajustar el deslizador de rango o los valores, o seleccionar diferentes elementos de las listas, etc. Tenga en cuenta que cuando edita una variable padre sólo puede cambiar el valor definido pero no el nombre, ni puede cambiar las opciones de la variable. También tenga en cuenta que puede añadir nuevas Object Variables a las secundarias objects también (y en la imagen de ejemplo de arriba puede ver que "Tipo de carácter" es una nueva variable Object sólo para la secundaria object).

Si ha editado una variable heredada de Object, puede hacer clic en el botón Borrar para eliminar la edición, pero no la variable. La variable La variable seguirá siendo heredada del padre, pero utilizará el valor por defecto del padre de nuevo. Si necesitas eliminar completamente la variable, entonces debes hacerlo desde el padre object.
