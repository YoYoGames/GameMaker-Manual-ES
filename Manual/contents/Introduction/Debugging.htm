Depuración

Al programar su juego, ya sea usando GML Code o GML Visual, puede ser muy fácil cometer errores - usar las variables equivocadas, pasar los argumentos equivocados, o usar las funciones equivocadas son sólo algunos de los errores más comunes que todos cometemos - y estos errores no siempre son detectados por el verificador de sintaxis que se incluye con el editor de scripts/acciones. Esto significa que estos errores sólo se harán evidentes cuando se ejecute o compile el juego, e incluso entonces puede ser difícil saber exactamente lo que está fallando. Por lo tanto, puede ser de vital importancia saber cómo está funcionando tu juego en el dispositivo elegido, así como ver qué instancias hay en el room, qué variables globales se han creado y toda una serie de otros detalles. Comprobar estas cosas se llama "depurar tu código", y cualquier error o problema que puedas encontrar se llama "bugs".

Ahora, puedes comprobar muchas de estas cosas por ti mismo añadiendo el llamado "código de depuración" a tus juegos para mostrar en la pantalla la información que necesitas, o puedes hacer que muestre mensajes cuando hay un error, pero esto es a menudo mucho trabajo y si no estás seguro de dónde está el problema en primer lugar, entonces puede ser un caso de prueba y error que consume tiempo para rastrear tu error. Para ello, GameMaker tiene un modo de prueba especial llamado Modo de Depuración que iniciará el Módulo de Depuración, y también hay un sistema de reporte de errores completo que muestra mensajes en la Compilación para ayudarle a rastrear cualquier problema con su juego.

Las secciones siguientes cubren las diferentes maneras en que puedes depurar tu juego:

Informe de errores

Cuando se produce un error durante la compilación y ejecución de un proyecto, se informa de ello con un mensaje en la ventana de salida de errores de compilación, muy parecido al que se muestra a continuación:

La mayoría de los mensajes de error del compilador siguen el mismo esquema, con un mensaje que indica el object en el que se ha producido el error, luego el Evento junto con el número de línea, y finalmente una breve descripción del error en sí. Este tipo de información proporcionada por el informe de errores es muy importante, ya que detecta e informa de los errores relacionados con la sintaxis y el lenguaje que van a hacer que tu juego terminado se cuelgue, pero no informará de errores más sutiles, ni da información sobre el rendimiento. Para ese tipo de cosas deberías usar el modo de depuración (explicado más abajo).

Aparte de estos errores de compilación, también puede obtener errores de sintaxis. Estos se detectarán a medida que añada sus acciones o código de GML Visual y se mostrarán en la ventana de salida de errores de sintaxis, similar a la imagen que se muestra a continuación:

Al igual que con los errores del compilador, la ventana de errores de sintaxis le dirigirá a la object, el evento y las líneas específicas que están dando el problema, y a menudo se mostrará con una breve explicación de lo que es el error.

Modo de depuración

Si necesitas comprobar las cosas con más detenimiento, puedes ejecutar el juego en modo de depuración haciendo clic en el botón de ejecución de depuración . Cuando lo hagas, se abrirá el módulo de depuración junto con tu juego. Este módulo te permite monitorizar cómo funciona todo en tu juego, desde el rendimiento global, hasta los valores de las variables individuales en instancias individuales. Puedes encontrar más información sobre el Módulo de Depuración y sus posibilidades y cómo utilizarlo en la siguiente página:

  • El depurador

Ten en cuenta que cuando utilizas el módulo de depuración, también puedes añadir puntos de interrup ción a tu código o GML Visual. Un breakpoint es un punto en el juego loop en el que quieres que el módulo de depuración haga una pausa y te permita recorrer el código del juego línea a línea. Puedes añadir breakpoints usando la tecla "F9", y se mostrarán en la Ventana de Salida donde puedes habilitarlos/deshabilitarlos según sea necesario.

Herramientas adicionales

Como herramienta adicional para debugging juegos compilados, hay ciertos parámetros de línea de comandos que se pueden utilizar cuando se ejecuta el paquete final del juego en Windows. Para obtener información sobre cómo funciona esto, consulte aquí.

También puedes generar una superposición de depuración cuando estés probando tu juego que mostrará información adicional de depuración sobre el rendimiento de tu juego mientras se está ejecutando, y también puedes enviar mensajes de depuración a la consola de compilación donde puedes proporcionar tu propio texto para ayudar a depurar el comportamiento de tus juegos. Para más información, consulta las siguientes páginas:

  • show_debug_overlay
  • show_debug_message
  • debug_event
