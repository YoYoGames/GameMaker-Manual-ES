Creación de un tutorial de Marketplace

La creación de un tutorial es similar a la creación de un paquete normal Marketplace asset , sólo que ahora tiene que añadir archivos específicos incluidos, así como incluir una página inicial del tutorial (y una página de vídeo opcional). El Tutorial workspace tendrá un aspecto similar al siguiente:Como se ha explicado anteriormente para el paquete general Marketplace Assets, debe seleccionar un editor y luego dar a su tutorial un nombre y un ID único (formato de URL inverso), y luego en el desplegable "tipo" seleccionar Tutorial. Esto le dará opciones adicionales para añadir un archivo HTML de base para iniciar el tutorial, así como un archivo de vídeo (opcional) y una URL. El archivo de vídeo debería utilizar la plantilla suministrada desde Marketplace - al igual que el resto del tutorial - que puede obtener aquí, y cuando el usuario cargue el tutorial, el vídeo se cargará en un marco separado dentro de IDE. Tenga en cuenta que las páginas que da para la página de inicio y la página de vídeo son relativas paths dentro de los Archivos Incluidos, y puede abrir un explorador para elegir algo distinto a los predeterminados con los que GameMaker rellena los campos.

En realidad, la creación de su tutorial requerirá que añada Archivos Incluidos en su proyecto de tutorial, y estos tienen que ser añadidos de una manera muy específica para que el IDE detecte correctamente qué archivos se van a utilizar para el tutorial. La estructura básica de carpetas se muestra en la imagen siguiente:

Así, tenemos dos carpetas principales:

  • TutorialContent: Todos los archivos de su tutorial HTML, el CSS para estilizarlos, y las imágenes requeridas para ellos (no las imágenes requeridas para los recursos, sino las que se mostrarán dentro del tutorial), así como la plantilla de vídeo si se requiere, van aquí. También puede tener subcarpetas si desea dividir el tutorial en capítulos, por ejemplo, pero debe tener al menos un archivo base HTML que se utilizará como página de inicio del tutorial.
  • TutorialResources: Esta carpeta tiene otras dos subcarpetas para imágenes y recursos de sonido . Si su tutorial requiere que el usuario añada un sprite (por ejemplo) y usted quiere incluirlo en el tutorial, entonces colocaría el recurso de imagen en la carpeta Images. Estas carpetas se importan en el proyecto del tutorial y cuando el usuario abre el editor sprite y va a añadir una imagen, el path por defecto para la ventana del explorador será a esta ubicación, haciendo más fácil para el usuario encontrar el assets necesario.

Para que tu tutorial funcione dentro de IDE debes tener esta estructura de carpetas, así como una página de inicio base HTML con un archivo CSS para el estilo. Hay una serie de enlaces especiales que se pueden utilizar dentro de sus proyectos de tutoriales que saltarán a partes de un vídeo, o abrirán el manual en una sección o función específica, y todos ellos se dan en las plantillas oficiales de Tutorial de la Marketplace.

¡IMPORTANTE! Le sugerimos encarecidamente que descargue las plantillas de tutoriales del Mercado antes de intentar crear sus propios tutoriales. Contienen ejemplos de todas las partes importantes del HTML, así como CSS e imágenes para recrear botones y menús dentro del tutorial, dejándole libre para preocuparse por el contenido en lugar de cómo mostrarlo.

Una vez que haya configurado los archivos incluidos y suministrado la información requerida, puede hacer clic en Okay y el tutorial se subirá a Marketplace, abriendo un workspace en la lista de asset cuando haya terminado para que pueda editarlo y añadir iconos, texto, capturas de pantalla, etc. Una vez que el tutorial asset haya sido publicado, los usuarios podrán encontrarlo dentro de la sección de Tutoriales de la Página de Inicio, donde podrá ser descargado e instalado.
