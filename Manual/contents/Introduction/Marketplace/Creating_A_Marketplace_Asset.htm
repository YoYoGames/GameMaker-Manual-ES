Creación de un activo Marketplace

Cuando seleccione la opción de crear un paquete en el menú del mercado IDE, accederá al siguiente espacio de trabajo: Aquí En primer lugar, debe seleccionar la identidad del editor con la que desea publicar el paquete asset (que debería haber configurado desde su panel de control de editor) y, a continuación, dar un nombre para mostrar y un número de versión. A continuación, se creará un ID de paquete basado en el nombre de visualización y la cuenta de editor utilizados, pero puede editarlo para que sea otro si lo desea, siempre que mantenga el formato de URL inverso. A continuación, deberá elegir el tipo de paquete asset que desea crear:

  • Activo: Es una colección de assets (sprites, scripts, objects, etc...) que desea vender en el Mercado
  • Demostración: Se trata de una demo para demostrar algún concepto o idea de juego
  • Tutorial: El asset que se está subiendo es un tutorial, diseñado para ser ejecutado desde el GameMaker IDE (ver esta página para más detalles)

Una vez hecho esto, puedes elegir qué assets del proyecto quieres añadir al paquete Asset utilizando la sección de la derecha. Puedes añadir todos tus assets, o sólo algunos de ellos así como eliminar alguno que hayas añadido por error. Se asume que usted ha creado el asset para ser cargado previamente en GameMaker y que el proyecto actual contiene todo lo que usted requiere para cargar.

¡RECUERDE! Para poder crear o utilizar los paquetes de asset, debes haber creado previamente un certificado de perfil de usuario. Puedes hacerlo desde las Preferencias de Marketplace, que se explican aquí.

Cuando esté satisfecho con la selección de assets y quiera cargar su paquete en Marketplace, simplemente haga clic en el botón Okay de la parte inferior, y el paquete asset se creará y se cargará en Marketplace. Cuando la carga se haya completado, se abrirá un nuevo workspace con una vista de navegador de la página asset para que pueda editarla. Para obtener más información sobre cómo configurar esto, consulte la siguiente página:

  • Cómo poner un activo en la lista

Si necesita actualizar un Marketplace Asset, debe hacer más o menos lo mismo que cuando creó el paquete original, sólo que ahora tiene que especificar el paquete a actualizar de una lista desplegable de paquetes disponibles:

Al actualizar, no podrá cambiar el ID del paquete, y debe tener en cuenta que el número de versión que se muestra será el número de versión actual y deberá incrementarlo según corresponda, de lo contrario la actualización fallará (ya que no puede tener dos paquetes de activos con el mismo número de versión). También hay un campo en el que puede añadir algún texto para el registro de cambios de asset, que se cargará junto con asset y se mostrará en la página Marketplace.

IMPORTANTE: Assets realizado con la versión 2.3+ de GameMaker sólo estará disponible para aquellos usuarios que utilicen la versión 2.3 y superiores. Si estás actualizando un asset de una versión anterior, ese asset seguirá estando disponible para los usuarios que no hayan actualizado y coexistirá con la versión 2.3+ actualizada.

Carga a través de la página web

Puede cargar assets a través de la página web Marketplace en lugar de IDE, algo que puede ser necesario si la carga a través de IDE ha fallado por cualquier razón. Para empezar, cree el paquete e intente subirlo a través de IDE como se muestra arriba, y si ha fallado, vaya a la ventana de salida y compruebe el registro para ver la línea que muestra dónde se guardó el paquete que está creando antes de intentar la subida. Por ejemplo:

En Windows:

Created package file:
C:\Users\WINDOWSUSERNAME\AppData\Roaming\GameMaker\GM_USERNAME\Marketplace\Packages\com.publishername.assetname.yymp

En macOS:

Created package file:
/Users/MACUSERNAME/.config/GameMaker/GM_USERNAME/Marketplace/Packages/com.publishername.assetname.yymp

Tienes que ir a la ubicación de ese archivo y copiar el archivo *.YYMP a una ubicación segura, ya que será el archivo que subirás a la página web Marketplace.

En este punto debe abrir la página Marketplace en su navegador web y navegar hasta el listado asset para el asset y luego ir a la sección de Versiones, y luego hacer clic en el botón de Cargar Versión, como se muestra en la imagen de abajo:

Esto le llevará a la página de carga de la nueva versión, donde puede añadir información sobre la versión que se está cargando si es necesario y también suministrar el archivo haciendo clic en el botón Choose FIle y seleccionando el archivo *.YYMP creado previamente:

Al añadir el archivo se cargarán las versiones y se mostrará una barra de "progreso", y cuando la carga haya finalizado y esté satisfecho con los detalles, puede hacer clic en el botón Guardar para guardar el asset en el listado. Tenga en cuenta que el asset estará en la etapa de procesamiento durante unos minutos antes de que pueda ser publicado.
