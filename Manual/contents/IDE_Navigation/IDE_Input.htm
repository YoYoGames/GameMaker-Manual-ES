Entrada y navegación del IDE

El IDE de GameMaker acepta la entrada del ratón y del teclado, y muchas operaciones pueden llevarse a cabo utilizando uno u otro o ambos. En general, puede pulsar el botón izquierdo del ratón para seleccionar cualquier cosa, utilizar / + para seleccionar múltiples elementos, mantener para arrastrar elementos a diferentes muelles o a la workspaces, y también el para abrir menús específicos del contexto. Tenga en cuenta que si ejecuta GameMaker en un sistema macOS y utiliza un ratón de un solo botón, utilice + para hacer clic con el botón derecho del ratón .

También hay un gran número de atajos de teclado que se pueden utilizar para navegar por los diferentes elementos y editores de recursos de workspace, y vamos a repasar rápidamente algunos de los más importantes (puedes encontrar una lista completa aquí):

  • / + "Z" : Esto deshará la acción anterior y funciona en la mayoría de los editores. También tiene múltiples niveles de deshacer, por lo que puede pulsar esto varias veces para "retroceder" los cambios.
  • / + "Y" : Esto rehará una acción anterior que fue deshecha y funciona en la mayoría de los editores. Al igual que deshacer, también tiene múltiples niveles de rehacer.
  • / + "S" : Esto forzará a GameMaker a guardar su proyecto.
  • / + "T" : Esto abrirá la ventana Goto que le permite buscar rápidamente e ir a cualquier asset en su juego, así como buscar en las Preferencias y Opciones del Juego. Se le presentará una lista de todo y, a medida que escriba el término de búsqueda, esta lista se seleccionará para adaptarse, por lo que si todos sus rooms tienen el prefijo "rm_", por ejemplo, al escribirlo se seleccionará la lista para mostrar sólo los assets que comienzan con ese nombre.

  • / + : Esto abrirá la ventana de visión general del espacio de trabajo que puede utilizarse para navegar rápidamente entre los elementos que están abiertos en los diferentes workspaces. La ventana permanecerá abierta mientras se mantenga o , y si se pulsa una vez se pasará al siguiente elemento de la ventana. Si se suelta el atajo, se navegará hasta el elemento que se haya seleccionado.

    NOTA Si reasigna la vinculación de esta ventana para utilizar más de un modificador (como + + B) o una sola tecla (como B solamente), entonces la ventana permanecerá abierta después de pulsar el acceso directo, independientemente de si lo mantiene pulsado o no, hasta que la cierre manualmente o elija un elemento de la ventana.

  • : Esto abrirá el manual. Tenga en cuenta que al utilizar GML Visual o GML code también puede seleccionar una acción o una función (o palabra clave o lo que sea) y luego pulsar o hacer clic en el botón central del ratón para abrir el manual en la página correspondiente.
  • : Se puede utilizar para contraer o expandir todas las diferentes ventanas ancladas en el IDE.

Otras formas de navegar por workspace incluyen el uso del atajo de teclado / + + <Arrow Keys> para desplazarse entre las ventanas abiertas en la dirección pulsada, y manteniendo pulsado el botón central del ratón y luego arrastrando el ratón también puedes desplazarte por workspace.

Aparte de estos métodos de entrada, también hay una compatibilidad limitada con dispositivos de lápiz, y también un modo especial para aquellas personas que trabajan en proyectos utilizando un ordenador portátil. Ambos se explican a continuación:

Modo portátil

Si utilizas GameMaker en un ordenador portátil, tendrás otra opción en la parte superior de IDE para el modo portátil:

Esta opción está activada por defecto, pero puede desactivarse si lo prefiere pulsando este botón botón. El modo portátil se combina con algunas herramientas en el IDE para hacer una experiencia mucho mejor cuando se utiliza un touchpad, simplificando las 3 interacciones principales del ratón: Pan, zoom y desplazamiento. Para ello utiliza dos teclas modificadoras: Izquierda / e Izquierda . Cuando el modo portátil está activado, Izquierda / y Izquierda se "reservan" para interacciones con el ratón como el desplazamiento del editor room, donde Izquierda / es el zoom, Izquierda es desplazamiento y paneo. El desplazamiento en el modo portátil se comporta igual que el paneo, por lo que debe ser intuitivo y lo suficientemente preciso como para no tener que usar la barra de desplazamiento. Tenga en cuenta que al mantener estos botones, simplemente tiene que mover el ratón - no hacer clic ni arrastrar, sólo mover - y que si hay un acceso directo general que normalmente requiere el uso del botón / o se deben utilizar las teclas teclas de la derecha del teclado en su lugar.



Soporte de pantalla táctil

Una última cosa que mencionaremos aquí es que el IDE para GameMaker también tiene un soporte mínimo para pantallas táctiles. En todos los sistemas operativos se puede utilizar la pantalla táctil para hacer clic y arrastrar elementos en la página principal workspace, y se admiten 2 punteros simultáneos simultáneos, donde un segundo toque realizará un clic con el botón derecho. Ten en cuenta que en Windows 8 y superiores, el GameMaker IDE también admite dispositivos de lápiz.
