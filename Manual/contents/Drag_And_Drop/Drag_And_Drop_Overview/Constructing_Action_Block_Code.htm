Construcción de código de bloque de acción

Para añadir comportamientos a objects puedes construir tu código usando Acciones de las diferentes librerías disponibles en la Caja de Herramientas de Acciones. Para empezar necesitarás hacer un nuevo proyecto de GML Visual y luego hacer un nuevo objeto (puedes asignar un sprite a object también si es necesario). En tu nuevo object puedes empezar a añadir eventsy en los eventos añade tus acciones de código GML Visual.

Observa que cuando añades un nuevo evento, se abre una ventana de "código" con una pestaña para el evento en cuestión (ver la imagen de arriba), y ahora puedes arrastrar cualquier acción que desees de la Caja de Herramientas de la derecha al panel del Bloque de Acción de la ventana de código. Ahora bien, aunque es cierto que puedes añadir cualquier acción, eso no significa que todas vayan a funcionar o que el proyecto vaya a compilar con ellas. Algunas acciones requieren al menos una variable para funcionar, mientras que otras -como las acciones de Dibujo- sólo funcionarán si se utilizan en un evento específico. ¿Cómo saber cuáles utilizar? Bueno, en general, es simplemente una cuestión de usar la lógica... si una acción requiere una variable y no hemos definido una todavía, entonces no deberíamos usarla hasta que hayamos añadido una acción para crear la variable.

Cuando arrastras una acción desde la Caja de Herramientas al bloque de acción principal workspace, se expandirá para mostrarte losargumentos disponibles (parámetros) que puedes rellenar y cambiar para establecer el comportamiento. Puede ver que la nueva acción también se muestra a la izquierda de la ventana de código de forma abreviada. Esta lista de acciones, llamada Visión General de la acción, puede ser pulsada para navegar rápidamente a esa acción para editarla. Puede continuar añadiendo acciones al evento si lo necesita, con cada nueva acción "encadenada" a la anterior para mostrar el flujo del código de GML Visual que está construyendo. Ten en cuenta que el área en la que puedes colocar más acciones está resaltada debajo de la acción inicial y, dependiendo de la acción que estés utilizando, se resaltarán diferentes áreas para mostrar en qué parte de la cadena puedes añadirla:

Algunas acciones colocarán el código en una cadena separada del flujo principal - cosas como If Variable crearán una sub-cadena de acciones que deben ocurrir si se cumplen las condiciones correctas antes de continuar con la cadena principal:

Tenga en cuenta que cuando se utilizan acciones que pueden tener un bloque de cadena lateral como este, la acción tendrá dos áreas resaltadas para soltar más acciones:

  • Uno debajo de la acción como normal
  • Uno a la derecha de la acción para mostrar que se quiere crear un bloque de acciones separado que se ejecute cuando la condición de la primera acción sea verdadera

Puedes cambiar la posición de las acciones dentro de la cadena simplemente haciendo clic en y arrastrándolas a la nueva posición que necesites, y si haces clic en y mantienes pulsado un segundo y luego mueves el ratón, puedes mover la acción dentro de workspace manteniendo su lugar dentro de la cadena de bloques de acción.

Esto es lo básico para utilizar el editor de código GML Visual, pero hay otros detalles importantes que se explican en las secciones siguientes:

Variables objetivo

Es importante tener en cuenta que muchas acciones ofrecen una variable "objetivo", que puede marcarse como "Temporal", por ejemplo:

Esto significa que puedes suministrar una variable que es el objetivo para mantener el valor devuelto por la acción. En el ejemplo de la acción anterior, la acción devolverá el volumen del recurso de audio que selecciones, por lo que suministrarás una variable de destino para mantener este valor y poder referenciarlo más tarde.

Ahora, esta variable de destino necesitará haber sido declarada previamente, ya sea usando Asignar Variable (que crea una Variable de Instancia) o Declarar Variable Temporal (que crea una Variable Local Temporal) Si marca la opción "Temp", entonces puede simplemente añadir un nombre de variable y la acción creará esa variable y la establecerá para mantener el valor de retorno para usted (creando una variable local temporal). Cualquier acción utilizada después de esto puede ahora acceder al valor de la variable temporal, pero sólo dentro del mismo Evento o Script. Las variables temporales sólo están disponibles dentro del ámbito en el que fueron creadas. Para obtener más información sobre las variables y el ámbito de las variables, por favor vea aquí.

Ámbito de actuación

No sólo las variables tienen ámbitos (ver la sección de Variables de Destino , más arriba), las acciones también pueden tener diferentes ámbitos. Auto Complete

Al trabajar con GML Visual tendrá que añadir variables y expresiones en los diferentes campos de entrada de las acciones. Sin embargo, al hacerlo, a menudo aparecerá la ventana de Autocompletar para ayudarle:

Esta ventana emergente listará todas las variables, constantes y funciones incorporadas en GML (GameMaker Language), así como cualquier recurso que contenga las letras iniciales de lo que estaba escribiendo. Se puede utilizar para encontrar rápidamente el recurso o la variable que desea para la acción sin tener que escribirlo todo usted mismo. Por ejemplo, si todos sus rooms tienen el prefijo "rm_", al escribirlo y esperar un momento se mostrará la ventana de autocompletado con todos los assets que comienzan con "rm_". Tenga en cuenta que todas las variables incorporadas que se muestran en la ventana de autocompletado pueden utilizarse en cualquier lugar donde se espere una variable o expresión en una acción, al igual que la mayoría de las funciones de GML.

Ampliación de la acción

A veces, al utilizar una acción, verá un pequeño icono de signo más a un lado. Esto significa que puede ampliar la acción para realizar tareas adicionales o tomar más argumentos. Por ejemplo, si miras la acción Declarar Variables Temporales, puedes ver que tiene este icono:

Al hacer clic en el icono, la acción se ampliará y permitirá declarar más variables, lo que facilita y agiliza la definición de múltiples variables al mismo tiempo.
