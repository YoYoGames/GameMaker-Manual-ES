Resumen de {GML_Visual}

GML Visual es una herramienta de scripting visual que puede utilizarse para crear tus juegos sin tener que escribir ningún código. Esto no quiere decir que no estés programando código cuando utilizas GML GML Visual, ya que lo haces, sólo que tu código se crea y se presenta de forma visual utilizando "bloques" de acciones en lugar de texto. Las acciones son simplesexpresiones osentencias que pueden "encadenarse" para que una instancia de un object haga algo cuando se coloca en un room. Por ejemplo, podrías tener una acción en un Evento de Creación para mover una instancia en una dirección aleatoria, y otra acción en un Evento de Colisión para hacer que rebote en las cosas con las que colisiona (para más información sobre los diferentes eventos disponibles y cómo funcionan juntos, ver la sección de Eventos de Objeto).

La imagen de arriba muestra un típico object abierto en un espacio de trabajo para ser editado. Tiene una ventana "Evento" y una ventana "Código" encadenadas, y en la ventana de código se puede ver donde hemos construido nuestro GML Visual. Fichas de eventos

Cuando añades un evento a tu object, se abre una nueva pestaña en la ventana de código para mostrar el contenido de ese evento. Puedes añadir varios eventos y tener varias pestañas abiertas al mismo tiempo, cambiando entre ellas y editándolas como quieras. También puedes cerrar las pestañas en cualquier momento para reducir el desorden (si seleccionas de nuevo el evento en object se volverán a abrir), y puedes hacer clic en y arrastrar la pestaña fuera de la ventana a su propia ventana de código, en cuyo caso el cierre de object o de la ventana de código no cerrará la nueva ventana. Las pestañas también se pueden arrastrar a la izquierda o a la derecha para reordenarlas.

Resumen de la acción

A medida que vayas añadiendo bloques de acción a tu evento, éstos aparecerán en forma "abreviada" como una visión general de lo que está sucediendo. El orden de la vista general es el orden en el que se ejecutará cada uno de los bloques de acción para ese evento, y puedes hacer clic en cualquier elemento de la lista para que la vista del bloque de acción se amplíe a esa posición para su edición.

Espacio de trabajo del bloque de acción

Esta área workspace es donde se arrastran las acciones desde la caja de herramientas para crear su código GML Visual. Haga clic en en una acción y arrástrela a esta área para añadirla a la lista de acciones y editarla. La mayoría de las acciones tienen variosargumentos (parámetros) que puede establecer, y la mayoría también tomará palabras clave, variables de instancia, o incluso funciones GML - por ejemplo, puede llamar a la acción Establecer dirección y utilizar la función GML random(val) para establecer una dirección aleatoria.

A medida que añada bloques de acción a su código, se "encadenarán" visualmente para indicar el orden en que se producirán las diferentes acciones, aunque la forma en que se cree esta cadena dependerá del bloque inicial que se haya utilizado. Por ejemplo, un bloque If Variable tendrá dos cadenas - una a los bloques que deben ser llamados si la condición que se comprueba es true y luego el siguiente bloque después de eso. Para obtener más información sobre la construcción de bloques de acción en el código, consulte las secciones vinculadas a continuación.

Mientras estás en el bloque de acciones workspace, puedes mantener pulsado el botón central del ratón para desplazarte y la rueda del ratón para desplazar la lista de acciones hacia arriba o hacia abajo. Si mantienes pulsado mientras mueves la rueda del ratón, la vista se desplazará hacia la izquierda/derecha, y si mantienes pulsado / y mueves la rueda del ratón, la vista se acercará y alejará, permitiéndote ver más de la workspace con las acciones. Puedes hacer clic en en cualquier acción de la lista de acciones de la izquierda para centrarte en esa acción, restableciendo el zoom y la posición. También puedes cambiar el tamaño de los bloques de acción moviendo el ratón hacia los bordes del bloque y luego -cuando el cursor cambie- arrastrando en la dirección que desees.

También puede hacer clic con el botón derecho del ratón en cualquier lugar de GML Visual workspace para que aparezca el menú RMB, que se explica en detalle aquí.

Caja de herramientas

La caja de herramientas es donde se almacenan todas las acciones de GML Visual, con diferentes secciones que contienen colecciones (bibliotecas) de acciones que son similares o están relacionadas. Cada colección de acciones está relacionada con un tema específico o un propósito común y basta con hacer clic en y arrastrar la acción requerida de la caja de herramientas al bloque de acciones workspace para añadirla al evento actual. Para simplificar las cosas -ya que hay muchas acciones- puedes utilizar la barra de "Búsqueda" de la parte superior para buscar una acción específica o para filtrar las acciones visibles (escribe "dibujar", por ejemplo, para obtener todas las acciones de dibujo), y también puedes hacer clic en y arrastrar los iconos de cualquier biblioteca a tu carpeta de favoritos para mantener juntas las acciones más utilizadas. Esto es especialmente útil si hay ciertas cosas que vas a hacer una y otra vez. La carpeta de favoritos puede borrarse haciendo clic en el pequeño icono de la "papelera".

Para ver una lista completa de las acciones disponibles, así como los argumentos que toman y cómo se pueden encadenar, consulte la sección de Referencia visual de GML.

Puede utilizar el botón derecho del ratón sobre cualquier acción de la caja de herramientas para abrir el siguiente menú:

Aquí tienes las siguientes opciones:

  • Usar<acción>: Esto añadirá la acción que estaba bajo el ratón cuando se pulsó el botón derecho del ratón al evento actual workspace.
  • Hacer / Eliminar Favorito: Esto añadirá o eliminará la acción a la biblioteca de favoritos en la parte superior de la caja de herramientas. Esta biblioteca es donde puede crear un grupo personalizado de acciones que utiliza regularmente, reduciendo la cantidad de tiempo necesario para buscarlas en sus bibliotecas reales.
  • Borrar favoritos: Al seleccionar esta opción se eliminarán todas las acciones de la biblioteca de favoritos.
  • Borrar Usado Recientemente: Al seleccionar esta opción se eliminarán todas las acciones de la biblioteca de Usados Recientemente.
  • Contraer todo: Esta opción contraerá todas las bibliotecas de GML Visual para que sólo sean visibles los títulos de las bibliotecas, y no los nodos de acción. Cada biblioteca puede expandirse de nuevo haciendo clic en ella.
