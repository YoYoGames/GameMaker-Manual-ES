Aplicación de acciones a otras instancias

La mayoría de las acciones en las bibliotecas de GML Visual tienen una opción para aplicar la acción de diferentes maneras. Esto se llama establecer el ámbito de la acción y puede ser una de varias cosas:

Esencialmente lo que le estás diciendo a GameMaker es la instancia que debe ejecutar la acción. El ámbito de acción por defecto es self, lo que significa que una vez que el object es creado como una instancia en el room, esa instancia ejecutará el código de la acción. Sin embargo, esto no siempre es lo que quieres, y puedes querer que algunas acciones afecten a otras, o incluso a todas las instancias de room. Aquí es donde entra en juego el cambio del ámbito de la acción.

A continuación se enumeran los diferentes ámbitos para realizar acciones, pero hay que tener en cuenta que si se cambia el ámbito de una acción de este modo, sólo se aplicará el nuevo ámbito a esa acción y no a las acciones posteriores de la cadena. Si desea aplicar un cambio de ámbito de acción a varias acciones encadenadas, utilice primero la acción Aplicar a...

Auto

Este es el ámbito por defecto para una acción y simplemente establece que la acción sólo debe ser llamada por la instancia que está ejecutando el código.

Otros

El ámbito other tiene dos funciones principales, y el valor que devuelve dependerá de dónde y cómo lo uses. En el evento de colisión, other devolverá el valor del ID de instancia único (un valor único que se utiliza para distinguir las instancias individuales de cada object), por lo que puedes, por ejemplo, crear una instancia "bala" y tener un evento de colisión con una instancia "jugador" y en eso utilizar el ámbito other para eliminar los puntos de golpe del "jugador" object y luego volver al ámbito self para destruir la instancia "bala".

Fuera del evento de colisión, la configuración de other se comportará como si se hubiera establecido en noone a menos que se llame desde un bloque de acciones de alcance. Esto significa que si se cambia el alcance de un grupo de acciones a un object específico, entonces mientras esas acciones están siendo llamadas, el alcance other devolverá el ID de la instancia que inicialmente llamó al grupo de acciones. Por ejemplo, puede ejecutar una acción Aplicar a... y luego en el siguiente bloque de código establecer el ámbito a other para realizar una acción en la instancia que ejecuta todo el bloque de eventos, y no la instancia que se está aplicando en los bloques de código apply to. Todo

Cuando usted abarca una acción para todos, le está diciendo a GameMaker que ejecute ese bloque para cada instancia activa dentro de la sala actual. Por ejemplo, al crear una acción de destrucción de una instancia de Object y estableciendo su alcance a all hará que todas las instancias de room desaparezcan, sin importar desde qué object hayan sido creadas.

Objeto

A una acción también se le puede dar un objeto como ámbito. Lo que esto significa es que todas las instancias de la object dada ejecutarán esa acción al mismo tiempo que es llamada. Así que si tienes 100 instancias de enemigos en el room, por ejemplo, y quieres que todos apunten hacia un punto específico. Llamarías a la acción Establecer dirección del punto y establecerías el ámbito de la acción en object "obj_Enemigo" y cuando se llama, todas las instancias de ese object cambiarán de dirección.

Expresión

El campo de entrada de la Expresión es para que introduzcas el ID de una instancia específica sobre la que quieres que funcione la acción.
