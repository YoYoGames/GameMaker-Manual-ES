Opciones del menú del botón derecho del ratón

Cuando trabaje en su código de GML Visual, puede hacer clic con el botón derecho del ratón en la acción, en la página workspace, o en el texto de la lista reducida de la izquierda para abrir el menú del botón derecho del ratón:

Este menú tiene las siguientes opciones:

  • Cortar / Copiar / Pegar: Esto le permitirá cortar, copiar o pegar cualquier acción que esté seleccionada en el workspace. Tenga en cuenta que puede usar + fuera de las acciones y arrastrar, o usar / + en acciones individuales, para seleccionar más de una y aplicar el comando a todas ellas.
  • Comentar / Eliminar comentario: Esta opción le permite añadir y eliminar comentarios de sus bloques de código de GML Visual. Cuando seleccione Comentario, se abrirá un editor de texto a la derecha del bloque seleccionado de GML Visual, y allí podrá añadir cualquier texto que necesites, ya sea un recordatorio para cambiar algo, o un comentario útil para un tutorial, o incluso instrucciones para tus compañeros de equipo:
    Una vez que haya añadido el comentario, puede hacer clic en él y arrastrarlo para posicionarlo vertical y horizontalmente - aunque sólo a la derecha del nodo padre - y al colapsar el nodo GML Visual también se colapsará el comentario y se mostrará un resumen de su contenido (en cuyo caso no podrá reposicionarse). Si el comentario ya no es necesario, puede eliminar todo el el texto del comentario, o utilizar la opción Eliminar comentario del menú. Tenga en cuenta que cuando copie un nodo, el comentario se copiará con él, y si convierte el nodo de GML Visual a GML, el comentario se conservará y se añadirá al GML que se cree, como se ilustra en la imagen siguiente:
    Como se ha mencionado anteriormente, un comentario sólo puede aplicarse a un único nodo, pero puede ampliarse para cubrir múltiples nodos simplemente añadiendo saltos de línea para extenderlo - aunque esto puede requerir que se reposicione el comentario para evitar solapamientos con los nodos posteriores:
    Una vez que se ha añadido un comentario, puede seleccionar la opción "Comentario" del menú RMB o puede hacer doble clic en para activar la edición.
  • Desactivar / Activar: Estos comandos se pueden utilizar para deshabilitar o habilitar la acción (o acciones) seleccionada. Cuando una acción está deshabilitada, sigue siendo visible en el workspace pero no se ejecutará al compilar el juego. De esta manera puedes añadir/quitar acciones para probar y debugging.     Puede ver que las acciones "Si la puntuación" y "Ir a la siguiente habitación" han sido desactivadas - se han puesto en gris, visualmente - y así se saltarán cuando se pruebe o compile el juego.
  • Alternar punto de interrupción: Este comando activará o desactivará un breakpoint para la acción seleccionada. Un breakpoint es simplemente un punto en el código de GML Visual donde el depurador debe detenerse cuando el juego cuando el juego se ejecuta en modo de depuración. Una acción con un breakpoint asignado a ella será resaltada, como se muestra en la imagen de abajo (la acción "Si la salud" se ha establecido como un breakpoint):
  • Eliminar todos los puntos de interrupción: Con este comando puede eliminar todos los breakpoints del código de GML Visual.
  • Contraer todo / Expandir todo: GML Las acciones visuales pueden ser colapsadas o expandidas, donde una acción colapsada simplemente mostrará el nombre de la acción pero no ninguno de los valores internos maximizando así el espacio disponible para el workspace. Por ejemplo:
    Como puede ver, las acciones ocupan mucho menos espacio, y todavía puede ver lo que hacen desde la breve descripción que aparece a la derecha. Puede seleccionar la opción "Expandir todo" para deshacer esto para cada nodo en el workspace.
  • Restablecer diseño: Esto restablecerá el GML Visual workspace, estableciendo el nivel de zoom al 100%, centrándose en la acción actualmente seleccionada, y expandiendo cualquier (y todas) las acciones colapsadas también.
  • Vista previa en vivo: Cuando seleccione esta opción, activará o desactivará la ventana de previsualización de código GML Visual to GML (GameMaker Language). Esto se explica con más detalle en la página Cambiar GML Visual a Código.
  • Convertir a GML: Esto convertirá todas sus acciones en el evento actual en código GML (GameMaker Language). Tenga mucho cuidado con esto ya que no puede convertir GML de nuevo en las acciones de GML Visual de las que fue creado. Puedes convertir el evento de vuelta a un evento GML Visual desde el código, pero cualquier código será simplemente envuelto en una acción de Ejecutar Código. Esto se explica con más detalle en la página Cambiar GML Visual a Código.
  • Ventana: Esta sección abre el siguiente submenú:
    Los elementos de menú disponibles aquí son:
    • Ir al objeto: Esto desplazará el workspace al object que está llamando al evento
    • Abrir Evento Heredado: Esta opción sólo está disponible si el object que se está editando es un hijo de un padre object, y abrirá el evento que se está editando del padre object y desplazará el workspace hacia él.
    • Un Panel / 2 Columnas / 3 Columnas: Estas opciones se pueden utilizar para dividir el workspace actual en 1, 2 o 3 "paneles" diferentes. Cada panel es un workspace separado que puede tener scripts o eventos arrastrados a ellos para que pueda editar y comparar 2 o más bloques de código de GML Visual a la vez. Las dos opciones siguientes sólo están disponibles cuando el menú RMB se abre en un nodo GML Visual:

  • Panorama: Al seleccionar esta opción en cualquier nodo, el workspace hará un paneo para rodear ese nodo hacia el centro.
  • Mostrar ayuda... : Al seleccionar esta opción se abrirá el Manual en un nuevo workspace en la página correspondiente al nodo que se ha seleccionado. También puede conseguir los mismos resultados utilizando en el nodo.
