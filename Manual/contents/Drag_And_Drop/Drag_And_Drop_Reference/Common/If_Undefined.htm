Si no está definido

Esta acción se utiliza para comprobar si una variable o expresión devuelve un valor de "indefinido", y si lo es entonces esta acción devolverá true y si no lo es devolverá false. "Indefinido" puede ser devuelto por un número de razones un número de razones, y un valor de undefined puede causar una caída o un error grave en su proyecto, por lo que puede utilizar esto para comprobarlo y atraparlo antes de que ocurra. Si flag el argumento "Not", entonces usted está comprobando para ver si un valor es no es "indefinido".

Tenga en cuenta que para añadir acciones en el bloque "if", éstas deben colocarse al lado de la acción, como se muestra en la imagen siguiente:

Estas acciones se ejecutarán ahora si el "if" se evalúa como true, mientras que cualquier otra acción que se lance en otro lugar se ejecutará después del bloque "if". Tenga en cuenta que puede "apilar" las comprobaciones de undefined en la misma acción haciendo clic en el icono más junto a la acción, y dando otra variable para comprobar. Esto comprobará si todas las variables dadas son undefined en una sola comprobación y sólo realizará el código de acción añadido si lo son.
