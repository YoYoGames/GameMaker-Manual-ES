Crear sistema de partículas

Esta acción inicializará el sistema de partículas para que puedas crear y utilizar tipos de partículas y emisores con él. Usted da el nombre de la capa como string - por ejemplo "Particle Layer" (que debe existir en el editor room ) - y luego flag el sistema como persistente o no. Si el sistema se marca como Persistente, entonces permanecerá activo en todos los rooms después de haber sido creado, y se le asignará una capa gestionada si la capa inicial no existe en los siguientes rooms. Si no se marca la persistencia, el sistema se destruirá automáticamente al final de la room en la que fue creado, al igual que cualquier emisor asociado al sistema. Si desea eliminar manualmente el sistema de la memoria, deberá utilizar la acción Destruir sistema de partículas.

También es necesario suministrar una variable de destino que contendrá el valor único de ID del sistema de partículas. Este valor de ID es lo que se utilizará en otras funciones para establecer las propiedades del sistema o para decirle a GameMaker qué sistema utilizar al crear partículas. Usted puede optar por flag esta variable como una variable local temporal, pero eso significará que se descarta al final de la acción script o evento, lo que significa que el sistema seguirá existiendo, pero te dejan sin ningún medio para acceder a ella, por lo que el cuidado debe tenerse cuidado al usar esto (en general los sistemas nunca necesitarán ser almacenados en una variable local temporal, pero para ciertas cosas, como dibujar partículas a una superficie, puede ser necesario).

NOTA: Un sistema de partículas persistente, una vez creado, permanece residente en la memoria del sistema durante la duración del juego, por lo que si necesita recrearlo en cualquier momento (por ejemplo, si llama a la acción Reiniciar Juego ), entonces debe destruir el sistema actual primero, de lo contrario se obtiene una fuga de memoria. Una fuga de memoria se produce cuando un recurso se añade a la memoria y luego se "de-referencia", lo que significa que GameMaker no tiene manera de interactuar con ese recurso. Esto lleva a que la memoria sea obstruida por recursos inaccesibles que a su vez pueden causar problemas con su juego. Para destruir un sistema de partículas persistente cuando ya no es necesario utilizar la acción Destruir Sistema de Partículas.

Sintaxis de la acción:

Argumentos:

Argumento Descripción
Layer La capa donde se dibujarán las partículas de este sistema
Persistent Si el sistema de partículas debe permanecer activo después de que el actual room haya terminado
Target La variable de destino para mantener el valor de identificación único del sistema creado

Ejemplo:

El bloque de acción anterior crea una variable global, luego crea un nuevo sistema de partículas y asigna su valor de identificación único a la variable global. El sistema de partículas creado se marca como persistente, por lo que permanecerá activo incluso después de que el actual room haya terminado.
