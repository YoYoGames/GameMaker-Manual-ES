Crear emisor de partículas

Para poder ver tus partículas necesitas haber creado primero un Sistema de Partículas, luego haber definido algunos tipos de partículas y finalmente haber creado un emisor de partículas usando esta acción. Para crear el emisor, se llama a esta acción y se asigna el emisor a un sistema de partículas que se ha creado previamente (ver Sistema de partículas) y luego se le da una variable de destino para almacenar su valor de identificación único para su uso en acciones posteriores. Puede elegir flag esta variable como una variable local temporal, pero eso significará que se descarta al final de la acción script o del evento, lo que significa que el emisor seguirá existiendo pero dejará sin ningún medio para acceder a él, por lo que hay que tener cuidado al usar esto (en general, los emisores nunca necesitarán ser almacenados en una variable local temporal, pero para ciertas cosas, como dibujar partículas en una superficie, puede ser necesario).

¡IMPORTANTE! Si su sistema de partículas es persistente entonces debe destruir todos los emisores al final de cada room usando Destruir Emisor de Partículas y volver a crearlos en el siguiente room según sea necesario, o almacenar sus valores de ID en variables globales para que se pueda acceder a ellos en cualquier momento en cualquier room, o tenerlos creados por un object persistente, para que el object persista y pueda acceder a ellos a través de variables de instancia. También tenga en cuenta que si llama a la acción de Reiniciar el Juego debe destruir el sistema de partículas de antemano para que el sistema y los emisores asignados sean eliminados de la memoria antes de comenzar de nuevo. Si no se eliminan correctamente los emisores de la memoria del sistema como se ha indicado anteriormente dará lugar a una fuga de memoria. Esto sucede cuando un recurso es añadido a la memoria y luego "desreferenciado", lo que significa que GameMaker ya no tiene forma de interactuar con ese recurso. Esto hace que Esto conduce a la memoria que se obstruye por los recursos inaccesibles que a su vez puede causar problemas con su juego.
