Crear juego de retroceso

Esta acción alberga una nueva partida multijugador. Toma el número de jugadores que deben estar conectados para que la partida comience, y opcionalmente permite desactivar la prueba de sincronización.

Cuando se llama con la Prueba de Sincronización desactivada, el sistema espera a que la cantidad dada de jugadores se una antes de comenzar el juego. Asegúrate de no crear o modificar ninguna instancia gestionada hasta que el juego comience.

CONSEJO Puedes forzar el inicio de la partida antes de que todos los jugadores se hayan unido, ejecutando Iniciar partida.

Actualmente sólo puede haber hasta 4 jugadores en una partida.

ADVERTENCIA No se recomienda establecer una región por defecto en esta acción a menos que permita al jugador seleccionarla a través de un menú.

Sintaxis de la acción:

Argumentos:

Argumento Descripción
Count of players El número de jugadores que deben estar conectados para que el juego comience. Si se utiliza Definir jugador, este es el número de instancias que se crean automáticamente.
Sync test Especifica si la prueba de sincronización debe estar activada, y está activada por defecto. Cuando se desactiva, se habilita la funcionalidad en línea.
Region Especifica la región en la que se creará la página virtual room. Para obtener información sobre cómo obtener regiones, consulte rollback_create_game().
