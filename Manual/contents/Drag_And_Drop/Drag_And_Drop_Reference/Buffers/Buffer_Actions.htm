Biblioteca de acción de amortiguadores

Un buffer es una región de memoria física que se utiliza para mantener temporalmente los datos mientras se mueven de un lugar a otro o se manipulan de alguna manera, por ejemplo puedes utilizar un buffer para mantener los datos del estado del juego. GameMaker tiene una serie de acciones específicas relacionadas con la creación, modificación y carga/guardado de buffers, pero antes de utilizarlas tenga en cuenta los siguientes puntos:

  • Cuando se crea un buffer, se borra automáticamente y se llena a 0.
  • El argumento "size" de las funciones de buffer se refiere siempre al tamaño en bytes de buffer.
  • GML Visual buffers siempre está alineado a 1 byte. La "alineación" se refiere a cómo se almacenan los datos dentro de un buffer y se utiliza cuando se usan las funciones de GML buffer , pero para GML Visual no es relevante.
  • Los datos en buffers se leen y se escriben de forma secuencial (es decir, se llama a la acción Write Buffer tres veces, entonces se escribirá el primer dato, se avanzará, a la derecha el segundo dato, se avanzará y luego se escribirá el último dato), con algo llamado el valor "seek" para indicarle en qué lugar de buffer tendrá lugar la siguiente lectura/escritura.
  • Todos los datos se escriben en buffer y se devuelven desde buffer como una cadena, por lo que si ha almacenado algún número real tendrá que utilizar una de las acciones de tipo de datos para convertirlo desde string al valor.

La memoria utilizada para crear buffers es la memoria del sistema, por lo que incluso cuando el juego no tiene el foco, cualquier dato almacenado en un buffer debería estar a salvo, sin embargo, si la aplicación se cierra o se reinicia entonces se perderá. También vale la pena señalar que buffers también puede ser guardado como archivos y luego cargado de nuevo en el juego utilizando las acciones de archivo.
