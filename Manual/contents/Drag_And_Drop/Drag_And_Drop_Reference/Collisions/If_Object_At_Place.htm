Si Object En el lugar

Esta acción se utiliza para comprobar y ver si hay una colisión con una o más instancias de un determinado object con la instancia actual situada en un punto del room, basándose en la máscara de colisión de la instancia que llama. La acción se evaluará como true si se detecta una colisión, o false en caso contrario. Se da el object para comprobar y la posición para "colocar" la instancia para realizar la comprobación - que puede ser una posición absoluta en el room o una posición relativa a la instancia - y también puedes comprobar el No flag para comprobar si no hay una colisión en la posición dada, en cuyo caso la acción sólo se evaluará como true si no se encuentran colisiones, en caso contrario se evaluará como false.

Después de dar el object y la posición, puede marcar/desmarcar la opción de devolver la lista. Si se marca esta opción, la acción creará y rellenará una estructura de datos de la lista con con todas las instancias en colisión con la instancia y establecerá la variable Target con el ID de la lista DS. Si no está marcada, sólo establecerá la variable Objetivo a un valor menor que 0 cuando no se detecte ninguna colisión, o el valor de ID único de una de las instancias en la colisión (ten en cuenta que si hay varias instancias en colisión, no tienes forma de saber qué ID de instancia se devolverá cuando no se utiliza una lista).

Si ha seleccionado devolver una lista - y la variable Target no está configurada como Temp - entonces también puede seleccionar la opción Free Target. Si se establece en true, la lista contenida en la variable de destino será destruida y una nueva lista creada (y devuelta a la variable de destino) cada vez que llame a la acción. De este modo, puede reutilizar la misma variable sin preocuparse por las fugas de memoria, ya que, si no marca la opción Free Target, se hace responsable de destruir la lista antes de volver a utilizar la variable en la acción. Tenga en cuenta que, independientemente de si esta opción está marcada o no, si ha configurado la acción como Devolución de lista, la variable de destino siempre contendrá un ID de lista, y como tal, tendrá que ser liberada cuando la instancia sea destruida o el room termine (puedes usar el evento Limpiar para esto, por ejemplo, junto con la acción Free Data Structure ). Vale la pena señalar que puedes configurar la opción de Liberar Objetivo para utilizar una variable predefinida o una expresión y así controlar cuándo se libera la estructura de datos de la lista estableciendo la variable en otro lugar (o teniendo diferentes resultados para la expresión) siempre que se evalúe como true/false.

Por último, se suministra la variable Target que se desea que contenga el valor devuelto o el ID de la lista. Si flag la variable Target para mantener el valor devuelto es una variable Temp (local), entonces la acción creará esta variable para mantener el valor devuelto sólo hasta el final del evento, en este caso la opción Free Target no hace nada y usted será responsable de destruir la lista DS si esa opción ha sido marcada. Si no se suministra ninguna variable de destino y la opción Lista de Retorno está marcada, no se creará ninguna lista.

¡IMPORTANTE! Las colisiones sólo se registrarán para aquellas instancias que tengan una máscara de colisión válida, es decir, que tengan una sprite asignada a la sprite_index, o una sprite asignada a la mask_index. Si cualquiera de las instancias en Si cualquiera de las instancias en la colisión no tiene máscara de colisión, la colisión no se detectará, independientemente de lo que la instancia esté dibujando en ese momento.
