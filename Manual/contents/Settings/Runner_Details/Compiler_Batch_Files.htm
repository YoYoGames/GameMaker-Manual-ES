Archivos / scripts de compilación por lotes

El compilador de GameMaker le permite colocar archivos Batch (Windows) o Shell scripts (en macOS y Linux) en el directorio de su proyecto, y en el directorio de cada extension en su proyecto.

Colocación

Se puede colocar un archivo script en la raíz del directorio del proyecto, junto al archivo .yyp del proyecto.

Un archivo script también puede colocarse en el directorio de un extension, que puede tener el siguiente aspecto: <directorio_de_proyecto>/extensiones/<carpeta_de_extensión>/.

Las versiones extension de un archivo script se ejecutarán primero, y su versión raíz se ejecutará en último lugar. El orden de ejecución de extensions no puede controlarse, por lo que un desarrollador de extension debe asegurarse de que las versiones de una extensión scripts no dependen de las versiones de otra extensión scripts.

Ejecución

Los archivos de script con nombres específicos, colocados en cualquiera de los directorios descritos anteriormente, se ejecutan en varios puntos durante el proceso de compilación:

Nombre del archivo por lotes de Windows macOS/Linux Shell Script Nombre Descripción
pre_run_step.bat pre_run_step.sh Esto se ejecuta antes de que el despliegue del juego esté a punto de comenzar
post_run_step.bat post_run_step.sh Se ejecuta cuando el juego está preparado y listo para ejecutarse; tras la ejecución del script se inicia el juego
pre_build_step.bat pre_build_step.sh Esto se ejecuta antes de que se inicie el compilador asset para construir el juego
post_build_step.bat post_build_step.sh Esto se ejecuta después de que el compilador asset se haya iniciado para construir el juego
pre_package_step.bat pre_package_step.sh Esto se ejecuta antes del paso final de empaquetado, que es cuando todos los archivos están listos pero el archivo ZIP final o el paquete de la tienda está a punto de ser creado
post_package_step.bat post_package_step.sh Se ejecuta después de que se haya completado el último paso de empaquetado y se haya preparado el archivo ZIP final o el paquete de la tienda
post_texturas.bat post_texturas.sh Esto se ejecuta después de que se hayan generado las texturas para su juego. Puedes utilizar este paso para optimizar las texturas de tu juego manualmente.

Este script recibe una variable de entorno llamada TexturesDir, que almacena el path a las texturas generadas.

Estos archivos script son compatibles con la construcción en todas las plataformas de destino.

Desactivación de Scripts para una extensión

Si utiliza varios extensions que ejecutan scripts, por ejemplo, Steamworks y GDK, entonces querrá desactivar Steamworks al exportar a GDK, y viceversa.

Para ello, consulte: ¿Cómo desactivar Extensions?
