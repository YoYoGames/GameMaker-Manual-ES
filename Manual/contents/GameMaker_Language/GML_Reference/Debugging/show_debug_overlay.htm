show_debug_overlay

Esta función se puede utilizar para switch activar y desactivar la superposición de depuración estándar cuando se prueba el juego y está desactivada por defecto. La superposición de depuración muestra una barra gráfica de uso de la CPU/GPU en la propia ventana del juego junto con el valor real actual de fps, el número de intercambios de texturas y el número de lotes de vértices (tenga en cuenta que los intercambios de texturas y los lotes de vértices nunca serán cero y normalmente mostrarán valores de 2 o 3, ya que incluso con un room vacío y sin objects GameMaker todavía tiene que dibujar y hacer lotes de cosas).

Esta barra está dividida en secciones, cada una de las cuales tiene una duración de 1/60 de segundo. Como puedes ver en la imagen de arriba, la barra se compone de varios colores:

  • Verde - Procesamiento de entrada/salida (es decir, teclado, ratón, gamepad, red, etc.)
  • Rojo - La velocidad de actualización del evento de paso
  • Amarillo - El tiempo requerido para el evento de sorteo
  • Naranja -Tiempo de actualización de la depuración, que normalmente sólo es visible cuando se utiliza el módulo de depuración
  • Blanco - Tiempo de sobra de la GPU, que es el tiempo que se pasa esperando a que la GPU termine de renderizar el fotograma antes de que pueda empezar el siguiente
  • Cyan - El tiempo de representación del texto
  • Gris - El tiempo necesario para limpiar la pantalla en cada paso del sorteo
  • Azul - El tiempo necesario para que el recolector de basura se ejecute
  • Rojo oscuro - El flush de la GPU, que es el tiempo que tarda la GPU en borrar las imágenes de la memoria

Utilizando esta función puedes añadir la superposición tanto en modo de depuración como en otro, y de esta manera, puedes ver la eficiencia de tu juego y obtener una señal visual de cómo está utilizando los recursos disponibles, sin la sobrecarga de tener el debugger corriendo al lado.

NOTA: Esta función no funcionará en el objetivo HTML5 debido a la forma diferente en que funciona en comparación con todos los demás objetivos.

Sintaxis:

show_debug_overlay(enable);

ArgumentoTipo Descripción
enable switch on (true) o apagado (false) the debug overlay.

Devuelve:

Ejemplo:

if global.debug
{
    show_debug_overlay(true);
}
else
{
    show_debug_overlay(false);
}

El código anterior activará o desactivará la depuración dependiendo del valor de una variable global.
