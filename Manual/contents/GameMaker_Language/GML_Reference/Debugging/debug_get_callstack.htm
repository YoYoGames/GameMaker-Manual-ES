debug_get_callstack

Esta función genera una matriz de strings como la "pila de llamadas" donde el actual script aparece primero, y luego todos los otros scripts que se ejecutaron para que el actual script se ejecutara. El formato exacto de string variará dependiendo de la plataforma de destino elegida, pero en su mayoría tendrá el nombre del script/evento, luego dos puntos : y el número de línea, similar a esto:

"gml_Script_script2:1"
"gml_Script_script1:32"
"gml_Script_script0:64"
"gml_Object_object0_Create_0:1"

La función permite pasar un argumento opcional, que es la profundidad máxima de la pila de llamadas devuelta. Este valor es el número de scripts que se incluyen en el array devuelto empezando por el actual script. Si no se especifica este argumento, se devolverá la pila de llamadas completa.

Tenga en cuenta que la matriz devuelta siempre contendrá 0 en su última posición, después de enumerar la pila de llamadas scripts.

Sintaxis:

debug_get_callstack([maxdepth])

Argumento Tipo Descripción
[maxdepth] Real OPTIONAL La profundidad máxima de la pila de llamadas que se devuelve

Devuelve:

Array

Ejemplo:

if debug_mode
{
    if keyboard_check(vk_escape)
    {
        var _a = debug_get_callstack(4);
        for (var i = 0; i < array_length_id(_a); ++i;)
        {
            show_debug_message(_a[i]);
        }
    }
}

El código anterior comprueba si el modo de depuración está activado y, si lo está, comprueba si se mantiene pulsada una tecla. En ese caso, muestra la pila de llamadas actual en la consola, con una profundidad máxima de 4.
