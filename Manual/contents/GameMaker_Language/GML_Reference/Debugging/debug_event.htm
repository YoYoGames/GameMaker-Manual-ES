debug_event

Esta función genera un evento de depuración personalizado que se mostrará en la Vista Gráfica de la debugger cuando un juego se esté ejecutando en Modo Depuración. Si necesitas que los mensajes se muestren cuando no estés en modo de depuración, utiliza show_debug_message().

La función también tomará dos reservados strings para ayudar a realizar debugging utilizando aplicaciones externas como Visual Studio. Estos strings son:

  • "OutputDebugOn" - Esto permite una llamada a OutputDebugString para el objetivo de Windows, lo que significa que toda la salida de depuración - todo lo que se ve en la ventana de salida - puede ser visto por Visual Studio o por aplicaciones de terceros.
  • "OutputDebugOff" - Desactiva el comportamiento descrito anteriormente.
  • "BreakOnError" - Esta opción es sólo para las construcciones de Windows YYC, y significa que los proyectos no mostrarán la pantalla habitual de error de código si el runtime detecta un error, sino que simplemente continuará y se bloqueará. Esto le permite ver el stacktrace dentro de Visual Studio si debugging.
  • "ResourceCounts" - Enumera todos los recursos actualmente activos, como Data Structures, Fuentes de Tiempo, Superficies, etc.
  • "DumpMemory" - Da información sobre el uso actual de la memoria.

Sintaxis:

debug_event(string)

Argumento Tipo Descripción
string Cadena The custom debug event string to use.

Devuelve:

N/A

Ejemplo:

if !surface_exists(global.EffectsSurface)
{
    debug_event("Recreating Effects Surface");
    global.EffectsSurface = surface_create(room_width, room_height);
}

El código anterior comprueba si existe una superficie y, si no existe, se activa un evento de depuración en la vista gráfica de debugger (el juego debe haberse ejecutado en modo de depuración para que esto sea visible) y se recrea la superficie.
