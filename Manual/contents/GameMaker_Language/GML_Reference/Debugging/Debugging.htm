Depuración

Al hacer un juego, a menudo es muy importante que usted conozca ciertos detalles de la mecánica del juego en puntos específicos en el tiempo, o ser capaz de introducir ciertos valores clave para algunas funciones para probar diferentes métodos e ideas para su juego. Para ello, GameMaker te proporciona una serie de funciones de depuración que mostrarán mensajes y pedirán entradas mientras estés en modo de depuración (o en un objetivo Windows ), pero que serán ignoradas cuando se compilen para otros módulos de destino debido a que estas funciones ponen el juego en un loop cerrado hasta que la entrada se resuelva.

NOTA Aunque pueden funcionar en determinadas plataformas, no deben utilizarse para nada más que debugging.

  • debug_mode
  • debug_event
  • debug_get_callstack
  • exception_unhandled_handler
  • get_integer
  • get_string
  • show_error
  • show_message
  • show_question
  • show_debug_message
  • show_debug_overlay
  • code_is_compiled
  • fps
  • fps_real

Aparte de estas funciones especializadas, para ayudarte a crear tu juego, GameMaker también tiene una larga lista de mensajes de error que pueden mostrarse tanto al escribir el código de tu juego, como al compilarlo, o cuando el propio juego se está ejecutando. Estos mensajes señalan el error específico en cuestión y pueden ser utilizados para localizar un problema dentro de una función específica de object o script, ayudando en gran medida con el proceso de debugging su juego. Sin embargo, no siempre es obvio el significado exacto del mensaje de error, por lo que las siguientes secciones enumeran todos los posibles mensajes y explican su significado para facilitarte la vida:

  • Errores de sintaxis
  • Errores del compilador
  • Errores del corredor

También puedes encontrar más información sobre cómo depurar tu juego en la sección El depurador.
