Funciones numéricas

Los números reales en GameMaker se consideran números de punto flotante de doble precisión -es decir, que tienen un punto decimal en ellos sin un número fijo de dígitos ni antes ni después del punto- o enteros -es decir, son números enteros sin valor de punto decimal-. El 2, por ejemplo, es un número entero, pero el 2,01 es un real de coma flotante.

NOTA: En el objetivo HTML5, todos los números reales son dobles.

Esta distinción entre enteros y flotantes es muy importante cuando se trata de desarrollo entre plataformas, ya que la precisión de los cálculos realizados en un PC Windows no es la misma que la de esos mismos cálculos cuando se realizan en un dispositivo móvil. Esto significa que hay que prestar especial atención al hacer comparaciones, por ejemplo:

if (image_index == 1)
{
    // Do something...
}

En el ejemplo anterior, si hemos establecido el image_speed a 0,25 - por ejemplo - entonces después de cuatro pasos se puede asumir que el valor de image_index sería 1, pero puede ser un valor como 1,0000002 debido a la forma en que las matemáticas de punto flotante funciona y por lo que la evaluación no será true. Este tipo de errores puede ser bastante difícil de depurar, por lo que siempre es bueno ser consciente de ellos y planificar con antelación y utilizar otros medios de comprobación de valores o utilizar las funciones de piso o redondeo apropiadas (enumeradas a continuación) para convertir el número a comprobar en un entero (para más información sobre las matemáticas de punto flotante y por qué esto es un problema, por favor vea aquí). Por ejemplo, el código anterior podría escribirse como

if (floor(image_index) == 1)
{
    // Do something...
}

También vale la pena señalar que cuando se utilizan los objetivos del Compilador YoYo, todas las expresiones y funciones se evalúan de izquierda a derecha, mientras que en todas las demás plataformas de destino se evalúan de derecha a izquierda, lo que significa que esto - por ejemplo - dará resultados diferentes dependiendo de la plataforma:

val = max(num, ++num, num++);

NOTA: Para más información, consulte la sección sobre la orden de evaluación.

También puede utilizar una función especial disponible en GameMaker para establecer el valor de épsilon para las matemáticas de punto flotante. Cuando un número real se redondea al número de punto flotante más cercano, el épsilon (también conocido como "épsilon de la máquina") forma un límite superior en el error relativo, y usted puede obtener y establecer el valor del épsilon utilizando las siguientes funciones:

  • math_set_epsilon
  • math_get_epsilon

NOTA Cuando se utilizan las funciones aleatorias, GameMaker mantiene la misma semilla aleatoria cada vez que se inicia el juego. Esto hace que debugging sea mucho más fácil, ya que se garantiza que las funciones aleatorias siempre devolverán inicialmente el mismo valor, sin embargo, si no desea que esto suceda, primero debe establecer una nueva semilla aleatoria en el inicio del juego, ya sea utilizando randomise() o random_set_seed().
