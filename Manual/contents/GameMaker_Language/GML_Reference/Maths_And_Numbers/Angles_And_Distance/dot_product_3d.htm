punto_producto_3d

El producto punto es un valor que expresa la relación angular entre dos vectores y se obtiene tomando dos vectores, multiplicándolos y sumando los resultados. El nombre "producto punto" deriva del punto centrado "-" que se utiliza a menudo para designar esta operación (el nombre alternativo "producto escalar" enfatiza la naturaleza escalar y no vectorial del resultado).

La fórmula matemática real se puede escribir así:

En 2D, el producto punto de los vectores a[x1,y1] y b[x2,y2] es x1x2 + y1y2, por lo que en 3D, el producto punto de los vectores a[x1,y1,z1] y b[x1,y1,z1] es x1x2 + y1y2 + z1z2. Por lo tanto, el dot_product_3d en GameMaker se calcula como:

a · b = (x1*x2) + (y1*y2) + (z1*z2);

Lo curioso del producto punto es la relación que tiene con el ángulo formado por los vectores de entrada que se puede expresar como:

a · b = (length of a) * (length of b) * cos(angle)

Es decir, el producto punto de dos vectores será igual al coseno del ángulo entre dichos vectores, multiplicado por las longitudes de cada uno de ellos. Aquí tienes una imagen para ilustrarlo:

Hay algunas cosas que podemos afirmar ahora de dos vectores cualesquiera en relación con el resultado de su producto punto:

  • Si a y b son perpendiculares (a 90 grados entre sí), el resultado del producto punto será cero, porque cos(ang) será cero
  • Si el ángulo entre a y b es menor que 90 grados, el producto punto será positivo (mayor que cero), ya que cos(ang) será positivo
  • Si el ángulo entre a y b es mayor que 90 grados, el producto punto será negativo (menor que cero), ya que cos(ang) será negativo

¿Qué significa esto para los que hacemos juegos? Bueno, esta relación matemática se puede utilizar en bastantes circunstancias, pero la mejor manera de verlo es construir un escenario real y ver por uno mismo lo que está pasando. Una de las formas más sencillas Una de las formas más sencillas de hacerlo es generar una simple comprobación de "altura" para un enemigo en, digamos, un juego de plataformas, de forma que el enemigo "vea" al jugador si está por encima del plano formado por el vector normal del enemigo y el suelo en 3D.

Básicamente, estamos obteniendo el vector normal del enemigo perpendicular al suelo y luego estamos obtener el vector del jugador al enemigo. Luego obtendremos el punto_producto de estos vectores, y si el resultado es positivo el jugador está "por encima" del plano del suelo del enemigo y si es negativo está por debajo. El código real para que esto funcione se suministra en el ejemplo siguiente.

Sintaxis:

dot_product_3d(x1, y1, z1, x2, y2, z2)

ArgumentoTipo Descripción
x1 The x coordinate of the first vector.
y1 The y coordinate of the first vector.
z1 The z coordinate of the first vector.
x2 The x coordinate of the second vector.
y2 The y coordinate of the second vector.
z2 The z coordinate of the second vector.

Devuelve:

Ejemplo:

var x1, y1, x2, y2;
x1 = 0;
y1 = 1;
z1 = 0;
x2 = o_Player.x - x;
y2 = o_Player.y - y;
z2 = o_Player.z - z; if dot_product_3d(x1, y1, z1, x2, y2, z2) > 0 above=true else above=false;

El código anterior crea un vector normal a lo largo del eje y (arriba) de las instancias, y luego obtiene el vector del jugador object "o_Player" a sí mismo. Finalmente calcula el producto punto de estos dos vectores y si es mayor que 0 pone la variable "arriba" a true, y si es menor o igual que 0 la pone a "false".
