Web y HTML5

GameMaker ofrece varias funciones para interactuar con Internet o con server, y pueden utilizarse en todas las plataformas de destino disponibles. Estas funciones son:

  • url_open
  • url_open_ext
  • url_open_full
  • url_get_domain
  • http_request
  • http_get
  • http_get_file
  • http_post_string

Aparte de las funciones generales enumeradas anteriormente, el objetivo HTML5 tiene algunas funciones web adicionales que están diseñadas específicamente para ser utilizadas por los juegos que se dirigen a él:

  • browser_height
  • browser_width
  • browser_input_capture
  • webgl_enabled
  • clickable_exists
  • clickable_add
  • clickable_add_ext
  • clickable_change
  • clickable_change_ext
  • clickable_set_style
  • clickable_delete
  • analytics_event
  • analytics_event_ext
  • os_browser
  • http_get_request_crossorigin
  • http_set_request_crossorigin

Además de estas funciones para el módulo de destino de HTML5, también hay una variable especial de JavaScript que se puede utilizar en esas raras ocasiones en las que el contenido de la carpeta html5game se almacena en un lugar completamente diferente (URL). Normalmente esta path está codificada en el archivo JavaScript y es relativa a la ubicación desde la que se llama al JavaScript, pero se puede establecer la variable g_GameMakerHTML5Dir antes de la llamada al juego script y el juego se ejecutará desde la nueva ubicación definida. Por ejemplo:

<script>var g_GameMakerHTML5Dir = "http://mygamealternatedirectory.com/";</script>

También vale la pena señalar que puedes flag script funciones utilizando el prefijo gmcallback_ lo que significa que no serán ofuscadas cuando compiles el proyecto para HTML5. Esto es muy útil cuando se utilizan las funciones de clickable_* mencionadas anteriormente (para más información, consulte aquí).
