Cuerdas

En algún momento, al crear tu juego, necesitarás utilizar texto. El texto en los juegos se trata utilizando las funciones de cadena (un string es sólo otra forma de decir una línea de texto) y GameMaker tiene un conjunto completo de funciones que le permiten manipular strings de muchas maneras, incluyendo la inserción de un string en otro, la copia de strings y la capacidad de analizar strings para los dígitos o las letras que contienen. En general, un string sólo puede crearse añadiendo texto entre comillas dobles " " y no se aceptan las comillas simples strings , ni se puede dividir el string en varias líneas y esperar que GameMaker lo represente como si los saltos de línea fueran nuevas líneas (a menos que se utilice un identificador literal de cadena @, como se explica más adelante).

Vale la pena señalar que hay ciertas convenciones que puede utilizar al crear strings, sobre todo relacionadas con el uso de caracteres de escape. Se trata de caracteres que van precedidos de un símbolo "\". Así, por ejemplo, si quieres poner comillas dentro de un string tendrías algo así:

str = "Hello\"World\"";

GameMaker también tiene soporte completo de caracteres Unicode de cuatro bytes de ancho, lo que le permite decodificar y codificar caracteres Unicode en los límites superiores de la norma (incluyendo - pero no limitado a - emoji). Para trabajar con caracteres Unicode, puede utilizar el "\" para preceder cualquier literal Unicode - dígitos de hexágono precedidos de un "u", por ejemplo "\u00e2" para "á"- donde los dígitos son el número del carácter Unicode. Cuando se trabaja con Unicode de esta manera, es necesario tener en cuenta el hecho de que GameMaker interpretará todos los dígitos que siguen a la "u", por lo que si quería escribir "áa", por ejemplo, debe utilizar:

"\u00e2\a"

o

"\u00e2\u61"

o

"\u00e2" + "a"

ya que el uso de "\u00e2a" daría como resultado el carácter Unicode "ส" (que se convertiría en "\ue2a").

GameMaker también puede manejar cualquier literal hexadecimal - normalmente escrito como dígitos de hex tras "0x", por ejemplo "0xff", donde los dígitos son el número del carácter a utilizar. En GameMaker se escriben con "\x" y luego el valor hexadecimal. Estos y otros caracteres de escape predefinidos se enumeran en la siguiente tabla:

Constante Descripción
\n Newline
\r Retorno del carro
\b Retroceso (0x08)
\f Alimentación del formulario (0x0c)
\t Ficha horizontal (0x09)
\v Ficha vertical (0x0b)
\\ La propia barra invertida (0x5c)
\a Alerta (0x07)
\u[Hex Digits] Insertar un carácter Unicode
\x[Hex Digits] Insertar carácter literal hexadecimal
\[Octal Digits] Insertar carácter octal Unicode

NOTA: Strings admite el avance de forma, el tabulador vertical, etc., pero esto no quiere decir que el renderizado lo haga, y al dibujar strings estos caracteres pueden ser ignorados.

También puede crear literales de cadenas de texto precediendo a la totalidad de string con el carácter @:

var test = @"
Line breaks
over multiple
lines
";

El código anterior mostrará el string en múltiples líneas como si hubiera un carácter de escape de salto de línea incluido. Un literal string es similar a los literales de la versión anterior GameMaker string , pero también utilizan comillas dobles o simples y deben ir precedidas de un símbolo @, pueden romperse en varias líneas del archivo de código y NO admiten caracteres de escape, es decir, @"Hello\World" no intentará escapar la W de World y se almacenará literalmente. Tenga en cuenta, sin embargo, que cuando se utilizan literales string como este, tendrá que romper el string si desea incluir comillas como parte del string, es decir:

var test = @"Hello " + "\"" + @"World" + "\""

Otra cosa que hay que tener en cuenta es que el carácter Unicode 9647 (▯) se utiliza para sustituir cualquier glifo que pueda faltar en su font designado al renderizarlo en el evento de dibujo. Así que si su font no tiene, por ejemplo, el símbolo °, entonces escribir 90° producirá realmente 90▯.
