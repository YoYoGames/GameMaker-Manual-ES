Modo de compatibilidad de colisiones

En las Opciones Generales de Juego de su proyecto, puede habilitar el "Modo de Compatibilidad de Colisiones" para switch al sistema de colisiones heredado. Sólo necesitaría hacer esto si notara un comportamiento extraño con las colisiones en su(s) proyecto(s) existente(s) después de actualizar a GameMaker 2022.1, que cambió el funcionamiento de las colisiones.

Cambios en las colisiones

En el sistema de colisión heredado, todas las coordenadas de los cuadros delimitadores se redondeaban a números enteros y eran "exclusivas", es decir, excluían los bordes inferiores y derechos de una máscara de colisión. Por ejemplo, el cuadro delimitador de una máscara de colisión de 16x16 se generaría desde (0, 0) hasta (15, 15) (relativo). Esto funcionaría bien para las instancias que utilizan sólo valores enteros para el posicionamiento y la escala, sin embargo, el uso de valores de punto flotante causaría un comportamiento inesperado y una ligera desconexión entre el renderizado y la comprobación de colisiones, ya que GameMaker siempre redondearía los valores del cuadro delimitador para realizar las comprobaciones de colisiones.

En el actual sistema de colisiones, los valores de los cuadros delimitadores no se redondean y se utilizan tal cual, lo que significa que las colisiones son más precisas y se comportan como cabría esperar. Los cuadros delimitadores también son ahora "inclusivos", es decir, incluyen los bordes inferiores y derechos. Por ejemplo, el cuadro delimitador para una máscara de colisión de 16x16 se genera ahora desde (0,0, 0,0) hasta (16,0, 16,0) (relativo).

Modo de compatibilidad

Los cambios en la colisión explicados anteriormente pueden hacer que algunos proyectos existentes se rompan, lo que puede solucionarse fácilmente activando el Modo de Compatibilidad de Colisiones para dichos proyectos, de modo que se utilice el sistema de colisión heredado; sin embargo, se recomienda probar y actualizar su código de colisión para que funcione con el nuevo sistema, ya que debería permitir que su juego tenga colisiones más precisas y lógicas.

Para los proyectos nuevos o que no utilizan el código de colisión existente, no es necesario activar esta opción.
