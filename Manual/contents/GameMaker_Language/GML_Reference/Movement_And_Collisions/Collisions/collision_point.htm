punto_de_colisión

Collision_point comprueba si el punto especificado por los argumentos x1,y1 colisiona con cualquier instancia de la object especificada por el argumento "obj". esta comprobación puede ser precisa o no, pero para que las colisiones precisas estén habilitadas, la object o la instancia que se está comprobando debe tener también activadas las colisiones precisas para su sprite. Si no es así, la comprobación por defecto se basa en los cuadros delimitadores. La siguiente imagen ilustra cómo funciona esto:

Recuerde que para que se consideren colisiones precisas, tanto la object sprite como la función de colisión deben tener marcada la precisión como activada. También hay que tener en cuenta que el valor de retorno de la función puede ser el id de cualquiera de las instancias consideradas en colisión, por lo que si tres instancias se solapan en ese punto, cualquiera de sus ids podría ser el valor de retorno de la función.

Tenga en cuenta que en lugar de un índice object puede proporcionar una instancia id para buscar una instancia específica, o las palabras clave de instancia all, o other (dependiendo del evento y del ámbito del código actual).

Sintaxis:

collision_point(x, y, obj, prec, notme);

Argumento Tipo Descripción
x Real The x coordinate of the point to check.
y Real The y coordinate of the point to check.
obj Activo del objeto o ID de la instancia The object to check for instance collisions.
prec Booleano Whether the check is based on precise collisions (true, que es más lento) o su caja delimitadora en general (false, más rápido).
notme Booleano Whether the calling instance, if relevant, should be excluded (true) o no (false).

Devuelve:

Instance ID or noone

Ejemplo:

if collision_point(x, y, obj_Cursor, false, true)
{
    score += 10S;
}

Aquí estamos comprobando el punto en la posición del object que tiene el código para el object "obj_Cursor". Si hay uno, entonces añadimos 10 a la variable de puntuación.
