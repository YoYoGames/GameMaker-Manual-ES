collision_rectangle

Collision_rectangle utiliza los cuatro primeros argumentos (x1,y1,x2,y2) para definir un área dentro de la actual room y luego comprueba si algún object que esté definido por el argumento "obj" está en colisión con esa área. Esta colisión puede ser comprobada como precisa o no, y también puede elegir comprobar la instancia que ejecuta el código en sí misma o no. Considere esta imagen:

Aquí, la instancia del centro está usando un rectángulo de colisión para comprobar la bola objects. Ahora bien, las azules no tienen un cuadro delimitador preciso y, como puedes ver, aunque la sprite no esté tocando realmente el rectángulo, la colisión seguirá ocurriendo (incluso si estableces la opción precisa en la función como verdadera) ya que el cuadro delimitador de esa sprite sobrepasa el rectángulo de colisión. Por otro lado, las bolas verdes sólo se considerarán en colisión si el sprite real sobrepasa el rectángulo. Recuerde que para que se consideren las colisiones precisas, tanto el object sprite como la función de colisión deben tener marcada la precisión como activada. También hay que tener en cuenta que el valor de retorno de la función puede ser el id de cualquiera de las instancias consideradas en colisión.

Tenga en cuenta que en lugar de un índice object puede proporcionar una instancia id para buscar una instancia específica, o las palabras clave de instancia all, o other (dependiendo del evento y del ámbito del código actual).

Sintaxis:

collision_rectangle(x1, y1, x2, y2, obj, prec, notme);

Argumento Tipo Descripción
x1 Real The x coordinate of the left side of the rectangle to check.
y1 Real The y coordinate of the top side of the rectangle to check.
x2 Real The x coordinate of the right side of the rectangle to check.
y2 Real The y coordinate of the bottom side of the rectangle to check.
obj Activo del objeto o ID de la instancia The object to check for instance collisions.
prec Booleano Whether the check is based on precise collisions (true, que es más lento) o su caja delimitadora en general (false, más rápido).
notme Booleano Whether the calling instance, if relevant, should be excluded (true) o no (false).

Devuelve:

Instance ID or noone

Ejemplo:

var inst;
inst = collision_rectangle(50, 50, 200, 100, obj_Ball, false, true);
if (inst != noone)
{
    instance_destroy(inst);
}

Este código corto utiliza collision_rectangle para comprobar un área en el room desde 50x, 50y (parte superior izquierda del rectángulo) hasta 200x, 200y (parte inferior derecha del rectángulo) para una instancia de un object llamado "obj_ball". Almacena el valor de retorno en una variable temporal que luego se comprueba para ver si ese valor es un id de instancia, o la palabra clave nadie. Si no es nadie, utiliza el identificador de instancia almacenado para destruir el object.
