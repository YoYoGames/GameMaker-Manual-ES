Colisiones

Cuando se planifican movimientos o se deciden ciertas acciones, a menudo es importante ver si hay colisiones con otros objects en ciertos lugares del mundo del juego, y a menudo elegir la colisión adecuada para el trabajo es la tarea más importante de todas. GameMaker tiene una serie de funciones incorporadas para ayudarte a tratar las colisiones correctamente y de acuerdo con las necesidades de tu proyecto.

NOTA Las funciones de colisión solían utilizar valores enteros en versiones anteriores de GameMaker, sin embargo ahora utilizan valores de punto flotante, lo que significa que las funciones de colisión ya no redondearán los argumentos no enteros. Para más información, lea Modo de compatibilidad de colisiones.

Las siguientes funciones se ocupan de las distintas formas en que se puede hacer que una instancia compruebe las colisiones con otra instancia:

  • place_empty
  • place_free
  • place_meeting
  • position_empty
  • position_meeting
  • position_change
  • position_destroy
  • instance_place
  • instance_place_list
  • instance_position
  • instance_position_list

Comprobación avanzada de colisiones

Para ello, se pueden utilizar las siguientes rutinas (así como otras funciones detalladas en las secciones relativas a Moverse e Instancias). Estas funciones de colisión funcionan todas de forma ligeramente diferente, pero mantienen tres argumentos comunes que explicaremos aquí:

  • obj: Aquí se especifica el object con el que la función tiene que comprobar la colisión. Puede ser un id de instancia, un tipo object o la palabra clave especial GM all. También hay que tener en cuenta que si el tipo object elegido es el padre designado de otro objects, entonces todos los hijos de ese object se incluirán también en esta comprobación.
  • prec: Puede ser true o false y se refiere a la precisión de la comprobación de colisiones. Si se establece en true, entonces se calcularán las colisiones precisas (pero sólo si el object que se está comprobando tiene marcado "preciso" para su detección de colisiones en sus propiedades de máscara), lo cual es una tarea ligeramente más compleja para la CPU y, por tanto, debe utilizarse con precaución. Si false entonces las colisiones se calcularán en base a la caja delimitadora del object que se va a comprobar.
  • notme: Es posible que necesites comprobar las colisiones con un object que tenga el mismo índice que el object que ejecuta el código, así que para evitar que el código encuentre una colisión consigo mismo, puedes establecer esto como true. Si estás buscando específicamente la instancia del object que ejecuta el código, o no es importante, se puede dejar como false.

Las funciones básicas devolverán el ID de una instancia que se encuentre en colisión, o la palabra clave especial noone cuando no haya colisión, mientras que las funciones de lista devolverán el número de instancias que están en colisión y rellenarán una lista DS preelaborada con sus IDs. Tenga en cuenta que si hay múltiples colisiones con las áreas definidas por estas funciones y las instancias del object dado, sólo se devuelve el ID de una instancia, y puede ser cualquiera de las instancias en la colisión.

Existen las siguientes funciones que se ocupan de las colisiones avanzadas.

  • collision_circle
  • collision_circle_list
  • collision_ellipse
  • collision_ellipse_list
  • collision_line
  • collision_line_list
  • collision_point
  • collision_point_list
  • collision_rectangle
  • collision_rectangle_list

Comprobación de colisiones sin máscara

Todas las funciones anteriores están relacionadas con las instancias de comprobación de colisiones, y como tales dependen de la máscara de colisión que se define para la instancia. Sin embargo, hay muchos momentos en los que requieres comprobar las "colisiones" con un punto o un área, especialmente cuando tu instancia no tiene asignado un sprite, o cuando estás trabajando con el ratón, etc. Por lo tanto, GameMaker también proporciona las siguientes funciones para ayudarle en estas situaciones:

  • point_in_rectangle
  • point_in_triangle
  • point_in_circle
  • rectangle_in_rectangle
  • rectangle_in_triangle
  • rectangle_in_circle

Colisiones físicas

Cuando se utiliza la física incorporada, no se garantiza que las funciones de colisión anteriores funcionen para las instancias habilitadas para la física. Esto se debe principalmente al hecho de que estas instancias ya no utilizan la mayoría de las variables regulares incorporadas (en su lugar, las instancias habilitadas para la física tienen su propio conjunto de variables) y tampoco utilizan la máscara de colisión o la caja delimitadora, ya que utilizan accesorios en su lugar. Sin embargo, hay una función especial para comprobar las colisiones con la física activada que comprueba la superposición de accesorios:

  • physics_test_overlap
