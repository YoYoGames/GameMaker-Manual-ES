mp_potential_path

Esta función calcula un path para la instancia desde su posición y orientación actuales hasta la posición especificada por los argumentos xgoal,ygoal. Utiliza el tamaño de paso indicado mientras intenta evitar la colisión con todas las instancias o sólo con aquellas marcadas como sólidas. Como en la función mp_potential_step() utiliza pasos de campo potencial para conseguirlo y la forma en que lo hace puede modificarse mediante la función mp_potential_settings(). El path indicado debe existir ya y será sobrescrito por el nuevo path. La función devolverá si se ha encontrado un path (verdadero) o no (falso).

Para evitar que la función continúe calculando eternamente es necesario proporcionar un factor mayor que 1 - la función se detendrá e informará de un fallo si no puede encontrar un path más corto que este factor multiplicado por la distancia entre el inicio y la meta. Un factor de 4 suele ser suficiente, pero si esperas largos desvíos puedes hacerlo más largo. Si falla, se crea un path que va en la dirección aproximada de la meta, pero no la alcanzará.

NOTA: Esta función no mueve la instancia. Establece un path solamente, y usted debe usar las funciones Path para el movimiento.

Sintaxis:

mp_potential_path(path, xgoal, ygoal, stepsize, factor, checkall)

ArgumentoTipo Descripción
path index of the path that is to be used by the function
xgoal The target x position.
ygoal The target y position.
stepsize The speed the instance moves in pixels per step.
factor This number is used to prevent an infinite loop. It should be larger than 1. See the function description for more information.
checkall Whether to check all instances (true) or just solid ones (false).

Devuelve:

Ejemplo:

path = path_add();
mp_potential_path(path, obj_Player.x, obj_Player.y, 3, 4, 0);
path_start(path, 3, 0, 0);

El código anterior crea un nuevo recurso path y almacena su índice en la variable "path". A continuación, genera un path utilizando mp_potential_path entre el object con este código y el object "obj_Player", comprobando las colisiones con todas las instancias en el room. Finalmente inicia el object a lo largo de ese path, incluso si la función no ha podido devolver una ruta completa al "obj_Player".
