gc_get_stats

Con esta función puedes recuperar información sobre el estado actual del recolector de basura. La función devolverá una estructura que tendrá las siguientes variables miembro (nótese que "objetos" aquí se refiere a cualquier cosa que pueda ser recolectada por la basura y no a instancias generales de object como se define en el navegador Asset ):

Estructura de las estadísticas de GC
Variable Tipo Descripción
objects_touched Real Este es el número de objects activos que el recolector de basura encontró en el marco anterior. Esto variará en función de la generación que se haya recogido.
objects_collected Real El número de objects que el recolector de basura determinó que no estaban activos en el marco anterior, y que por lo tanto podrían ser eliminados.
traversal_time Real Este es el tiempo en microsegundos (en el hilo principal) que el recolector de basura tardó en averiguar qué objects estaba activo.
collection_time Real Este es el tiempo en microsegundos (en un hilo separado) que el recolector de basura tomó para limpiar el objects considerado inactivo.
gc_frame Real Este es un contador que se incrementa cada vez que se produce un pase de recolección de basura. Si la recolección de basura está deshabilitada, no se incrementará.
generation_collected Real Es el índice de la última generación recogida. El 0 es la generación más joven y el 3 es la más antigua actualmente.
num_generations Real Este es el número total de generaciones de recolección de basura.
num_objects_in_generation Matriz de Reals Es una matriz (de tamaño num_generations) que contiene el número de objects en cada generación.

NOTA: En la plataforma de destino HTML5 la recolección de basura es manejada por el motor de JavaScript y por lo tanto las variables miembro en el anterior struct devolverán todas 0 si esta función se utiliza en esa plataforma.

Cuando utilice esta función, tenga en cuenta que la información que se muestra en objects sólo se actualiza cuando se termina de procesar una generación completa, lo que puede llevar varios fotogramas dependiendo de la configuración del tiempo de los fotogramas (consulte aquí para obtener más información sobre el tiempo de los fotogramas).

Sintaxis:

gc_get_stats();

Devuelve:

GC Stats Struct

Ejemplo:

if (global.debug == true)
{
    var _s = gc_get_stats();
    var _t = _s.traversal_time;
    var _c = _s.collection_time;
    show_debug_message("Traversal time = " + string(_t))
    show_debug_message("Collection time = " + string(_c))
}

El código anterior comprueba una variable global y si es true obtiene información del colector de basura y la envía a la consola como mensajes de depuración.
