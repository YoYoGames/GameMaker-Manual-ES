Errores de retroceso

Cuando se trabaja con el sistema de Rollback hay ciertos errores que pueden ocurrir. Estos errores son Errores de Corredor y por eso se muestran mientras el juego está en marcha:

La siguiente tabla enumera los posibles errores que puede obtener al trabajar con el sistema Rollback y una descripción. También proporciona una solución para cada mensaje de error específico.

Error Descripción
No se pudo obtener la capa para crear jugadores. Debe pasar el nombre de la capa a la función rollback_define_player o crear la capa Instances. Si el jugador object se definió con la función rollback_define_player, el sistema de retroceso intentará crear el jugador object en la capa, cuyo nombre puede pasarse opcionalmente como segundo parámetro (o en la capa con nombre "Instancias" si no está definida). Si no se puede encontrar la capa por su nombre, se dispara este error. La solución es establecer explícitamente el nombre de la capa para crear los reproductores o crear una capa con nombre "Instancias" si no se establece el nombre de la capa explícitamente.
El jugador objects debe ser gestionado cuando se ejecutan juegos multijugador. Si el jugador object se definió con la función rollback_define_player y el jugador object no está marcado como gestionado se dispara este error. La solución es marcar el jugador object como gestionado en el Editor de Objetos.
No se ha podido obtener la información de usuario para player_id N. Si el sistema de retroceso no pudo obtener información de la server sobre el jugador, se dispara este error. Comprueba en la consola del desarrollador si se ha bloqueado alguna petición.
No se ha podido llamar a rollback_chat antes de estar completamente conectado al juego. Prueba a esperar el evento rollback_game_info. La función de chat no está disponible hasta que el jugador se haya unido a la room. Solución: Desactivar el chat hasta que se reciba el evento rollback_game_info (Ver Eventos Rollback).
rollback_use_manual_start debe ser llamado antes de rollback_create_game y rollback_join_game. Estos errores se disparan si una llamada a una función viola las restricciones internas de la máquina de estado. El mensaje de error sugiere una solución.
rollback_use_player_prefs debe ser llamado antes de rollback_create_game y rollback_join_game.
rollback_use_player_prefs debe ser llamado antes de rollback_set_player_prefs.
rollback_set_player_prefs tiene que ser llamado antes de rollback_start_game.
rollback_define_player debe ser llamado antes de rollback_create_game y rollback_join_game.
rollback_define_input debe ser llamado antes de rollback_create_game y rollback_join_game.
rollback_define_mock_input debe ser llamado antes de rollback_create_game y rollback_join_game.
No leer la información del jugador antes de que el juego haya comenzado. Llama a rollback_get_info().
No leer la entrada del jugador antes de que el juego haya comenzado. Llama a rollback_get_input().
rollback_define_input_frame_delay debe ser llamado antes de rollback_create_game y rollback_join_game
rollback_define_input_frame_delay debe ser llamado antes de rollback_create_game y rollback_join_game.
rollback_set_player_prefs no pudo establecer las preferencias La página server ha rechazado las preferencias del jugador. Normalmente esto ocurre en los siguientes casos: 1. Un usuario intenta restablecer las preferencias del jugador mientras el otro usuario ya ha iniciado el juego. En este caso, el juego debería utilizar los últimos prefs de usuario establecidos con éxito, garantizamos que cada jugador debe establecer algunos prefs de usuario antes de que el juego pueda ser iniciado. 2. El tamaño de los prefijos del jugador serializados excede el tamaño máximo de los prefijos del jugador (actualmente 800 bytes).
rollback_create_game, número de jugadores fijado en N pero debe estar en el rango de 1 a 4 La función rollback_create_game fue llamada con un número incorrecto de jugadores. La solución es arreglar el primer parámetro de rollback_create_game para que esté en el rango de 1 a 4.
rollback_create_game falló al establecer las preferencias por defecto del jugador. Ver la descripción de "rollback_set_player_prefs falló al establecer las preferencias"
El rollback multijugador sólo es compatible con el objetivo operagx. Se llama a la función rollback_create_game y el objetivo no es OperaGX. Actualmente, el multijugador sólo es compatible con la plataforma OperaGX. La solución es establecer el objetivo a OperaGX.
No se ha podido crear una partida (tipo de sesión desconocido). Se trata de un error interno del sistema de reversión. Informa de un problema al equipo de GameMaker.
No se ha podido encontrar player_id en esta instancia, tienes que pasarlo a rollback_get_input(). La funciónrollback_get_input puede aceptar el id del jugador como parámetro. Si no se proporcionan parámetros a la función, ésta tomará el campo player_id de un jugador object para obtener entradas. Si el parámetro player_id no está establecido, se dispara este error. Si el jugador object fue creado con rollback_define_player debería tener definido el player_id. Solución: 1. Pasar explícitamente el parámetro player_id a la función 2. Cree el jugador con rollback_define_player.
Información player_id N está fuera de rango. rollback_get_info se llama con un player_id incorrecto (o el jugador object tiene un valor incorrecto en el campo player_id). Solución: Corregir el parámetro player_id para que esté en el rango de 1 a 4.
No se ha podido encontrar la instancia <id> de object <id> (<nombre>) a la que se hace referencia en los datos de reversión. Error interno del sistema de retroceso. Informa de unproblema al equipo de GameMaker.
Ni idea de cómo crear OBJECT_KIND=<kind>.
RefID <id> no tiene object (¿entrada malformada?).
ReferenceKind <id> no válido (¿entrada malformada?).
El mapeo de instancias está habilitado pero se ha intentado deserializar un índice fuera de rango. <id> no está en [0, <id>].
RValue inválido en la deserialización.
Intentando resucitar una instancia que ha sido eliminada.
No se puede serializar RValue con kind=<id> (<nombre>).
Definición de entrada vacía. rollback_define_input fue llamado con una entrada vacía.
La definición de la entrada contiene la constante <código de clave> dos veces, lo que no está permitido. Un código de tecla sólo puede asignarse a una única entrada. Comprueba si algún código de tecla se establece más de una vez en la definición de la entrada.
La definición de entrada con la etiqueta <label> no es un valor real o una matriz de reales. Todos los valores que describen los valores de entrada deben tener el tipo Real.
Definición vacía de la entrada falsa para player_id N. Ver "Definición de entrada vacía".
La definición simulada contiene un valor no real. Véase "La definición de entrada con etiqueta <etiqueta> no es un valor real o una matriz de reales".
