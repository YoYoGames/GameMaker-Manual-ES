rollback_sync_on_frame

Esta función comprueba si todos los jugadores están sincronizados en el fotograma actual, lo que significa que sus estados no se basan en predicciones sino en los datos reales recibidos de ellos. En tal caso, devuelve true.

Si los jugadores no están sincronizados en el fotograma actual, devuelve false, e intenta sincronizar a todos los jugadores.

Intentar sincronizar los reproductores con esta función puede provocar una congelación. Debe asegurarse de que esta función no se llame continuamente en un evento de Paso.

En su lugar, utilícelo como un conmutador. Vea el ejemplo al final de la página.

La variable rollback_confirmed_frame se actualiza automáticamente cuando todos los jugadores están sincronizados.

Sintaxis:

rollback_sync_on_frame();

Devuelve:

Boolean

Ejemplo:

if (obj_game.winning_player == -1 && points >= obj_game.points_required_for_win)
{
    if (rollback_sync_on_frame())
    {
        obj_game.winning_player = player_id;
    }
}

Este código se ejecuta en el evento Step de un objeto jugador.

Comprueba si no hay ningún jugador ganador y si el jugador actual tiene los puntos necesarios para ganar.

Si ambos son verdaderos, significa que el jugador debería ganar. Sin embargo, antes de hacerlo, comprueba si todos los jugadores están sincronizados, para asegurarse de que el estado es correcto.

Si todos los jugadores están sincronizados, establece la variable winning_player. Esto hace que la primera condición se convierta en falsa, asegurando que rollback_sync_on_frame() no se llame de nuevo.

Esto significa que la función se utiliza como un conmutador, en lugar de ejecutarse continuamente en cada fotograma.
