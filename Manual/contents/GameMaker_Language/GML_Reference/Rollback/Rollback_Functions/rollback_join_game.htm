rollback_join_game

Esta función intenta unirse a una partida. Debes estar en una URL que contenga el ID de la room virtual para unirte.

Si se unió un room, devuelve true, y si no se unió ningún juego, devuelve false.

Cuando un nuevo jugador se une a una partida en la que ya estás presente, se activa el evento Rollback. Consulta los eventos de retroceso para obtener más información.

El argumento dry_run es opcional, y cuando se establece en true, permite ejecutar la función sin unirse a un juego. Se utiliza para comprobar si se puede entrar en una partida sin entrar en ella.

Sintaxis:

rollback_join_game([dry_run]);

Argumento Tipo Descripción
dry_run Booleano Disabled by default (false). Cuando está activada(true), la función no se une al juego, sino que simplemente devuelve true si se puede unir uno, o false en caso contrario.

De este modo, puedes utilizar esta función para comprobar únicamente si el jugador ha sido invitado.

Devuelve:

Boolean

Ejemplo:

rollback_define_player(obj_player);
var _joined = rollback_join_game();

if (!_joined)
{
    rollback_create_game(2, false);
}

El código anterior define un jugador object, y luego intenta unirse a una partida. Si no se unió, crea una nueva partida en su lugar, con un máximo de 2 jugadores y la prueba de sincronización desactivada.

Para ver un ejemplo ampliado, consulte Crear una partida multijugador.
