rollback_set_player_prefs

Después de habilitar las preferencias, esta función se utiliza para establecer las preferencias del reproductor local. Esto significa que la función debe depender de la entrada local y no de la entrada del Rollback.

Las preferencias pueden ser de cualquier tipo. Si se establecen preferencias por defecto, asegúrese de utilizar la misma data type y la misma estructura.

Cuando cualquier jugador establece sus preferencias, todos en el juego reciben un evento de retroceso.

Si se utiliza, esta función debe ejecutarse antes de la función join/create.

Por favor, lea: Preferencias del jugador

Sintaxis:

rollback_set_player_prefs(preferences);

Argumento Tipo Descripción
preferences Cualquier The preferences to set for the local player.

Devuelve:

N/A

Ejemplo:

rollback_set_player_prefs({
carácter: obj_customizer.character_selected,
color: obj_customizer.color_selected
});

El código anterior establece las preferencias para el jugador, leyendo los valores de un "personalizador" object, que sería no administrado (local).
