rollback_define_input_frame_delay

Esta función define el retardo de los fotogramas fijos para las entradas del reproductor local.

Por defecto, tu juego utilizará un retardo de entrada adaptativo, que se basa en la latencia entre jugadores. Esto funcionará para casi todos los juegos, sin embargo, algunos tipos específicos de juegos (por ejemplo, los juegos de lucha) pueden preferir utilizar un retraso de entrada fijo, en lugar de un retraso adaptativo que puede cambiar en cualquier momento.

No se recomienda cambiar esto a menos que sepas exactamente lo que estás haciendo. Puedes llamar a esta función en cualquier momento antes y durante una partida multijugador.

El valor de retardo especificado debe estar en número de fotogramas. Si especifica -1, el juego volverá a utilizar un retardo adaptativo. El retardo máximo que se puede establecer es de 10 fotogramas.

Sintaxis:

rollback_define_input_frame_delay(frames);

Argumento Tipo Descripción
frames Real The number of frames to set as the fixed input delay, or -1 para restablecer el valor por defecto.

Devuelve:

N/A

Ejemplo:

// 'Options' script
global.frame_delay = 5;

// obj_game: Create event
rollback_define_input_frame_delay(global.frame_delay);
var _joined = rollback_join_game();

if (!_joined)
{
    rollback_start_game(2, false);
}

Puedes definir tu valor de retardo de fotogramas como una variable global, y usarla cuando llames a rollback_define_input_frame_delay() antes de que se inicie una partida de rollback.
