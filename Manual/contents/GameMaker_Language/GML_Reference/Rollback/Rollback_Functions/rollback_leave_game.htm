rollback_leave_game

Esta función abandona la sesión multijugador actual, permitiendo al jugador unirse o crear una nueva partida de nuevo.

Cuando el anfitrión de un room se va, la propiedad de ese room se transfiere a otro jugador, por lo que el juego puede continuar. El juego sólo termina cuando todos los jugadores se han ido.

Evitar errores

Después de que el jugador abandone una sesión multijugador, GameMaker continúa ejecutando todo el código del juego, sin embargo, vuelve al estado de "preinicio" en el que no se pueden crear ni modificar instancias gestionadas.

Esto significa que tienes que destruir manualmente tus instancias de jugador al salir de una partida, y pausar cualquier otra lógica del juego usando rollback_game_running en una condición, para evitar errores.

Ver: Estado antes del inicio del rollback

Evento desencadenado

Cuando un jugador abandona una partida en la que estás presente, se activa el evento Rollback. Consulta los eventos de retroceso para obtener más información.

Sintaxis:

rollback_leave_game();

Devuelve:

N/A

Ejemplo:

if (keyboard_check_pressed(vk_escape))
{
    rollback_leave_game();
}

El código anterior hace que el jugador salga del juego cuando se presiona la tecla Escape.
