rollback_define_player

Esta función define el object que debe ser usado para los jugadores en el multijugador. Esto es opcional, y cuando se utiliza, el sistema creará automáticamente instancias para los jugadores que se conectan, y destruirá las instancias para los jugadores que se desconectan.

Por defecto, esto crea instancias de jugadores en la capa "Instancias ", que debe existir en su juego room. Si deseas utilizar otra capa, pasa su nombre como segundo argumento.

Para información sobre la creación de un juego con o sin esta función, ver Definir un objeto jugador. Si se utiliza, esta función debe ejecutarse antes de la función join/create.

Cada instancia creada con esta función obtiene las variables de instancia listadas en esta página.

Sintaxis:

rollback_define_player(object, [layer_name]);

Argumento Tipo Descripción
object Activo de objeto The object that will be used for creating player instances.
layer_name Cadena OPTIONAL El nombre de la capa donde se crearán las instancias de los jugadores. "Instancias" por defecto.

Devuelve:

N/A

Ejemplo:

rollback_define_player(obj_player);
var _joined = rollback_join_game();

if (!_joined)
{
    rollback_start_game(2, false);
}

El código anterior define un jugador object, y luego intenta unirse a una partida. Si no se unió, inicia una nueva partida en su lugar, con un máximo de 2 jugadores y la prueba de sincronización desactivada.
