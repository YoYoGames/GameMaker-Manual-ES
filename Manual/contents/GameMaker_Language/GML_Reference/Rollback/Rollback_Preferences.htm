Preferencias de retroceso

Es posible que quieras permitir a cada jugador seleccionar un personaje, o personalizar sus propiedades. Estas propiedades se pueden configurar como "preferencias del jugador".

¿Cómo utilizar las preferencias?

En primer lugar, active las preferencias llamando a rollback_use_player_prefs() / Use Player Prefs. Con esta función puedes establecer opcionalmente las preferencias por defecto para todos los jugadores.

Una vez que llames a esa función, el "auto-inicio" se desactiva, lo que significa que el juego no se iniciará incluso cuando todos los jugadores se hayan unido, y tendrás que iniciarlo manualmente con rollback_start_game() / Iniciar partida.

Antes de que el juego haya comenzado, puedes establecer las preferencias de un jugador llamando a rollback_set_player_prefs() / Set Player Prefs en el cliente de ese jugador. Estos pueden ser leídos más tarde a través de rollback_get_player_prefs() / Get Player Prefs, o la variable player_prefs en una instancia de jugador.

Evento de retroceso

Cuando un jugador cambia sus preferencias, todos en el juego reciben un evento rollback_player_prefs Rollback. El jugador debe estar completamente conectado para que se envíe el evento.

Si las preferencias se establecieron antes de conectarse, al finalizar la conexión se envían a todos los jugadores las últimas preferencias establecidas antes de conectarse.

Condición de inicio

Si has activado las preferencias y no hay preferencias por defecto, el juego no se iniciará a menos que se hayan establecido las preferencias para todos los jugadores.

En tal caso, puedes llamar a rollback_start_game() repetidamente en una alarma para intentar iniciar el juego, o llamarlo cada vez que se reciba el evento rollback_player_prefs Rollback.

¿Qué tipos puedo utilizar?

Una preferencia puede ser de cualquier tipo, por ejemplo, un número, string, array, etc., pero se recomienda utilizar una estructura, que es la forma más fácil de almacenar una colección de variables.
