player_id

Cuando se utiliza rollback_define_player() para definir un jugador object, el sistema crea automáticamente una instancia de ese object para cada jugador. Cada instancia de ese object obtiene la variable player_id, que almacena el ID de ese jugador.

Este ID comienza en 0, que inicialmente es el propio anfitrión. Todos los jugadores de 1 en adelante son "compañeros" que se han unido al juego.

Si no utiliza rollback_define_player(), esta variable no se creará para ningún objects. En ese caso, las instancias de los jugadores y sus IDs deben ser gestionadas manualmente, como se explica en Definir un objeto jugador.

Sintaxis:

player_id

Devuelve:

Real

Ejemplo:

// Create event
if (player_id == 0)
{
    x = 300;
}
else if (player_id == 1)
{
    x = 500;
}

El código anterior cambia la posición X de la instancia del jugador basándose en su ID.

También puedes utilizar este ID como índice de una matriz, por ejemplo, para almacenar las puntuaciones de todos los jugadores:

// Step event
var _coin = instance_place(x, y, obj_coin);
if (_coin != noone)
{
    obj_game.scores[player_id]++;
    instance_destroy(_coin);
}

// Draw event
var _my_score = obj_game.scores[player_id];
draw_text(x, y, _my_score);

El código del evento Step de arriba aumenta la puntuación del jugador en un array en obj_game cuando colisiona con una moneda. Esa instancia de moneda es entonces destruida.

El código del evento Draw anterior recupera la puntuación del jugador del mismo array, y dibuja su valor.
