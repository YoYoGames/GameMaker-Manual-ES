player_local

Cuando se utiliza rollback_define_player() para definir un jugador object, el sistema crea automáticamente una instancia de ese object para cada jugador. Cada instancia de ese object obtiene la variable player_local, que indica si esa instancia de jugador es local o no.

Si no utiliza rollback_define_player(), esta variable no se creará para ningún objects.

Puedes usar esta variable para mostrar qué jugador es local. No se recomienda utilizar esta variable de forma que afecte a la lógica del juego, que debe ser la misma para los jugadores locales y remotos.

Sintaxis:

player_local;

Devuelve:

Boolean

Ejemplo:

// Draw event
if (player_local)
{
    draw_sprite(spr_arrow, 0, x, y - 60);
}

El código anterior se ejecuta en el evento Draw, y comprueba si la instancia del jugador es local. En ese caso, dibuja una flecha sprite sobre la instancia, indicando al jugador qué instancia está controlando.
