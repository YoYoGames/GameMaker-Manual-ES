buffer_async_group_end

Esta función finaliza la definición de un grupo buffer async. Debe haber llamado previamente a la función buffer_async_group_begin() para iniciar el grupo, y luego llamar a la función buffer_save_async() para cada archivo que desee guardar (o buffer_load_async() para cargar buffers). Finalmente se llama a esta función, que iniciará el guardado de los archivos. La función devolverá un valor de identificación único para el guardado, que puede ser utilizado en el evento asíncrono Save / Load para analizar los resultados del async_load mapa DS.

Sintaxis:

buffer_async_group_end();

Devuelve:

Async Request ID

Ejemplo ampliado:

La función buffer_async_group_end() puede ser llamada desde cualquier evento, y como es asíncrona la llamada de retorno puede ser casi instantánea o puede tardar varios segundos. Llamar a la función es sencillo y sería algo así:

buffer_async_group_begin("SaveGame");
save1 = buffer_save_async(buff1, "Player_Save1.sav", 0, 16384);
save2 = buffer_save_async(buff2, "Player_Save2.sav", 0, 16384);
save3 = buffer_save_async(buff3, "Player_Save3.sav", 0, 16384);
save4 = buffer_save_async(buff4, "Player_Save4.sav", 0, 16384);
save_id = buffer_async_group_end();

El código anterior inicia un grupo buffer y lo configura para guardar 4 archivos de forma asíncrona. La definición del grupo es entonces terminada (momento en el que comenzará el guardado), almacenando el ID de la llamada a la función en la variable "save_id". Cuando se complete el guardado, el evento asíncrono Save/Load se activará y podrás analizar el mapa async_load para encontrar el ID correcto de la función, así:

if ds_map_find_value(async_load, "id") == saveid
{
    if (ds_map_find_value(async_load, "status") == false)
    {
        show_debug_message("Save failed!");
    }
}

El código anterior comprobará primero el id del mapa DS que se ha creado, y luego comprobará el estado de la llamada de retorno, publicando un mensaje de depuración si ha habido algún problema.
