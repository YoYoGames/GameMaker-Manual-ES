Búferes

Un buffer es una región de memoria física que se utiliza para mantener temporalmente los datos mientras se mueven de un lugar a otro o se manipulan de alguna manera, por ejemplo se puede utilizar un buffer para mantener los datos de los paquetes a la espera de la transmisión a través de una red o para almacenar un paquete que se ha recibido a través de una red. GameMaker tiene una serie de funciones específicas relacionadas con la creación, modificación y carga/guardado de buffers, pero antes de utilizarlas tenga en cuenta los siguientes puntos:

  • Cuando se crea un buffer, se borra automáticamente y se llena a 0.
  • El argumento "size" de las funciones de buffer se refiere siempre al tamaño en bytes de buffer.
  • El argumento "Alineación" se refiere a la forma en que se almacenan los datos dentro de buffer. Si su alineación está configurada, por ejemplo, a 4, y escribe un solo dato de 1 byte de tamaño y luego hace un buffer_tell, obtendrá un desplazamiento de 1. Sin embargo, si escribe otro dato, también de 1 byte de tamaño, y luego hace un buffer_tell, obtendrá un desplazamiento de 5, ya que la alineación ha "rellenado" los datos a ese tamaño. La alineación sólo afecta al lugar donde se escriben las cosas, así que si haces un buffer_tell después de escribir algo, devolverá la posición de escritura actual que sigue inmediatamente a los datos que has escrito. Sin embargo, ten en cuenta que si luego escribes otra pieza de datos, internamente el buffer moverá la posición de escritura al siguiente múltiplo del tamaño de alineación antes de escribir realmente la pieza de datos.
  • Cuando se trata de "offset", este es el valor en bytes para desplazar los datos dentro del buffer dado. Así que si su buffer está alineado en 2 bytes y quiere saltarse los primeros 4 lugares alineados, tendría un offset de 2 * 4 bytes.
  • Algunas de las funciones de buffer también crean un nuevo buffer (como buffer_load, por ejemplo). Recuerde que estos buffers también tendrán que ser eliminados de la memoria cuando no estén en uso utilizando la función buffer_delete.
  • La memoria utilizada para crear buffers es la memoria del sistema, por lo que incluso cuando el juego no tiene foco, cualquier dato almacenado en un buffer debería estar a salvo, sin embargo, si la aplicación se cierra o se reinicia entonces se perderá.

¡IMPORTANTE! Cuando se crea un buffer, el valor del índice que se utiliza para identificarlo es un valor entero que comienza en 0. Estos índices son reutilizados por GameMaker, por lo que un valor de índice destruido de buffer puede ser utilizado por uno recién creado posteriormente, por lo que se recomienda establecer cualquier variable que contenga un índice de buffer a -1 después de haber destruido el buffer.
