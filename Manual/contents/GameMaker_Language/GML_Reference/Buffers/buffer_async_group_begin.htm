buffer_async_group_begin

Esta función se llama cuando se quiere empezar a guardar varios buffers en varios archivos (o cargar varios buffers). El nombre del grupo es un string y se utilizará como el nombre del directorio donde se guardan los archivos, y debe utilizarse como parte del archivo path cuando se carguen los archivos de nuevo en el IDE más tarde (utilizando cualquiera de las funciones buffer_load() funciones). Esta función sólo se puede utilizar con las funciones buffer_save_async() y buffer_load_async() y también debe terminar el grupo asíncrono llamando a la función buffer_async_group_end() de lo contrario los archivos no se guardarán/cargarán.

Ten en cuenta que para las plataformas de consola (como PS4, por ejemplo), el nombre del grupo se utilizará como descripción de la ranura de guardado, y el uso de esta función puede ayudarte a evitar que se muestre la interfaz de usuario para cada archivo que se está guardando. También ten en cuenta que cuando usas UWP puedes mover tus partidas guardadas entre las ventanas nativas de Xbox y de la tienda de Microsoft creando un grupo que sea el mismo que un grupo de ERA Xbox, por ejemplo:

if os_type == os_uwp
{
    buffer_async_group_begin("root/yourgroupname");
}
else
{
    buffer_async_group_begin("yourgroupname");
}

Sintaxis:

buffer_async_group_begin(groupname);

Argumento Tipo Descripción
groupname Cadena The name of the group (as a string).

Devuelve:

N/A

Ejemplo:

buffer_async_group_begin("SaveGame");
save1 = buffer_save_async(buff1, "Player_Save1.sav", 0, 16384);
save2 = buffer_save_async(buff2, "Player_Save2.sav", 0, 16384);
save3 = buffer_save_async(buff3, "Player_Save3.sav", 0, 16384);
save4 = buffer_save_async(buff4, "Player_Save4.sav", 0, 16384);
buffer_async_group_end();

El código anterior inicia un grupo buffer y lo configura para guardar 4 archivos de forma asíncrona. A continuación, se finaliza la definición del grupo (momento en el que comenzará el guardado).
