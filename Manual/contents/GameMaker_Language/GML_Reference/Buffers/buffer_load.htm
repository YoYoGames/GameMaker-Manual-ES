carga_del_buffer

Esta función se utiliza para cargar un buffer que se haya guardado previamente utilizando las funciones buffer_save() así como cualquier archivo incluido o cargado externamente. Devolverá el ID de un nuevo buffer que es creado por la función como un "grow" buffer (ver aquí) con una alineación de bytes de 1. Este ID debe ser almacenado en una variable y utilizado en todas las llamadas posteriores a esta función buffer. Si la carga falla por cualquier razón, la función devolverá -1 y se mostrará un mensaje en la ventana de salida del compilador diciendo que la carga ha fallado.

NOTA Es importante que elimines de la memoria cualquier recurso creado dinámicamente como éste cuando ya no lo necesites para evitar fugas de memoria, así que cuando termines con el buffer que has creado debes liberarlo de nuevo usando buffer_delete().

Notas específicas de la plataforma

  • En HTML5, cualquier buffers cargado desde el almacenamiento local será decodificado usando base64 (ya que se guardan como base64 strings), sin embargo, usando esta función en Archivos Incluidos o cualquier archivo externo no se usará la decodificación base64, ya que no se espera que estén en un formato base64.
  • En HTML5, todos los archivos incluidos se almacenan como texto codificado en UTF-8, lo que puede dar lugar a un aumento del tamaño del buffer cargado en comparación con el archivo original, ya que la versión codificada en UTF-8 del archivo puede tener un tamaño diferente al original (debido al cambio de formato).

Sintaxis:

buffer_load(filename);

ArgumentoTipo Descripción
filename The name of the file to load from.

Devuelve:

Buffer ID

Ejemplo:

player_buffer = buffer_load("Player_Save.sav");

El código anterior cargará en memoria un buffer previamente guardado, creando un nuevo buffer. El índice de este nuevo buffer se almacena en la variable "player_buffer".
