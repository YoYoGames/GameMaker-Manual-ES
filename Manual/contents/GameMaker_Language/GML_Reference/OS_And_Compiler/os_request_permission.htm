os_check_permission

Con esta función puedes solicitar un permiso específico. El permiso a solicitar se suministra como string utilizando el formato "android.permission.<permission>", por lo que para solicitar el permiso RECORD_AUDIO (por ejemplo) se llamaría

os_request_permission("android.permission.RECORD_AUDIO");

La función devolverá la activación de un Evento Asíncrono del Sistema en el que el mapa de async_load DS contendrá los siguientes pares clave/valor:

  • "tipo": Es el tipo de evento que se disparó y el valor será el string "permission_request_result".
  • "<permiso>": Esto será el permiso solicitado por la función como un string, que contendrá una de las siguientes constantes:
Constante Descripción
os_permission_granted Esto indica que el permiso ha sido concedido
os_permission_denied Esto indica que el permiso no ha sido concedido

Cabe destacar que los siguientes permisos son soportados de forma nativa por GameMaker, pero son considerados "peligrosos" por Google y como tal deben ser solicitados explícitamente (tenga en cuenta también que algunos permisos pueden ser solicitados utilizando las Opciones de Juego de Android sin necesidad de esta función):

  • android.permission.WRITE_EXTERNAL_STORAGE
  • android.permission.READ_PHONE_STATE
  • android.permission.RECORD_AUDIO

Para más información sobre los permisos de las aplicaciones, consulte la documentación de Android.

¡IMPORTANTE! Esta función es sólo para el objetivo Android.

Sintaxis:

os_request_permission(permission)

Argumento Tipo Descripción
permission Cadena The permission to request (a string)

Devuelve:

N/A

Ejemplo:

if (os_type == os_android)
{
    if (os_check_permission("android.permission.INTERNET") == os_permission_denied)
    {
        os_request_permission("android.permission.INTERNET");
    }
}

El código anterior comprueba el tipo de sistema operativo y si es Android, realiza una comprobación de los permisos y si el permiso "INTERNET" aún no ha sido concedido, lo solicita.
