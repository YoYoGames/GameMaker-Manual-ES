os_is_paused

Cuando se desarrolla para dispositivos móviles, es necesario tener una forma de comprobar que el dispositivo está todavía centrado en su juego o no, sobre todo porque si su juego está en un dispositivo iOS, Android o Windows Phone entonces puede ser interrumpido por las llamadas telefónicas y otras aplicaciones. Puedes utilizar esta función para comprobar si el dispositivo está en pausa o no.

Vale la pena señalar que esta función se ejecutará cualquier código que está contenida en el bloque de código en los dispositivos, pero entonces GameMaker se pausará inmediatamente después como el dispositivo caches el juego como un proceso de fondo. Así que si usted tiene algo como el código de ejemplo proporcionado a continuación, la instancia que se creará en realidad no aparecerá o ejecutar cualquiera de sus eventos hasta después de que el juego se ha reiniciado desde el estado de caché. Esto significa que puedes preferir que la función os_is_paused llame a una alarma y luego en el evento de la alarma (que se activará cuando el dispositivo esté en pausa pero no se ejecutará hasta que se desactive) crear un menú de pausa, reiniciar la música, etc.

Si está utilizando esta función en Windows, Ubuntu (Linux), MacOS o plataformas HTML5, entonces la función sólo se cambia por un paso a true cuando el juego pierde el foco, y luego volverá false de nuevo hasta que el juego vuelva a estar enfocado (y lo pierda de nuevo). Esto se debe a que estas plataformas no cache aplicaciones y juegos, por lo que estos seguirán ejecutándose en segundo plano.

NOTA: En los dispositivos móviles, los sonidos y la música se detendrán cuando el sistema operativo esté en pausa. No se reiniciarán de nuevo a menos que los inicies específicamente con la función correspondiente.

Sintaxis:

os_is_paused()

Devuelve:

Ejemplo:

if os_is_paused()
{
    if (!instance_exists(obj_PauseMenu))
    {
        instance_create_layer(0, 0, "Controllers", obj_PauseMenu);
    }
}

El código anterior comprueba si el juego ha sido pausado, y si lo ha sido entonces crea una instancia de la object "obj_PauseMenu" si no existe ya ninguna.
