os_check_permission

Con esta función puedes comprobar si un permiso específico ha sido concedido al juego por el usuario. El permiso a comprobar se suministra como string utilizando el formato "android.permission.<permission>", por lo que para comprobar el permiso RECORD_AUDIO (por ejemplo) se llamaría a

os_check_permission("android.permission.RECORD_AUDIO");

La función devolverá una de las tres constantes - mostradas a continuación - para indicar al juego el estado del permiso. Para más información sobre los permisos de las aplicaciones, consulta la documentación de Android.

¡IMPORTANTE! Esta función es sólo para el objetivo Android.

Sintaxis:

os_check_permission(permission)

Argumento Tipo Descripción
permission Cadena The permission to check (a string)

Devuelve:

Permission State Constant

Constante del Estado de Permiso
Constante Descripción
os_permission_granted Esto indica que el permiso ha sido concedido
os_permission_denied Esto indica que el permiso no ha sido concedido
os_permission_denied_dont_request Esto indica que el permiso ha sido bloqueado por la configuración del teléfono, o que el usuario ha rechazado previamente la solicitud y ha seleccionado "No volver a preguntar".

Ejemplo:

if (os_type == os_android)
{
    if (os_check_permission("android.permission.INTERNET") == os_permission_denied)
    {
        os_request_permission("android.permission.INTERNET");
    }
}

El código anterior comprueba el tipo de sistema operativo y si es Android, realiza una comprobación de los permisos y si el permiso "INTERNET" aún no ha sido concedido, lo solicita.
