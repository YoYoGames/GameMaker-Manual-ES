os_get_info

Esta función devuelve un DS Map con información detallada sobre el sistema operativo en el que se está ejecutando el juego. La información exacta devuelta dependerá del sistema operativo y del dispositivo, por lo que se recomienda probar esta función en todas las plataformas a las que quieras dirigirte y dar salida a los valores devueltos utilizando una función como show_debug_message(). Tenga en cuenta que el mapa DS no se borra automáticamente de la memoria y debe utilizar la función ds_map_destroy() cuando ya no lo necesite.

A continuación puedes encontrar algunos detalles sobre los datos extra que se pondrán a disposición del mapa de DS específicos para las siguientes plataformas, y hay que tener en cuenta que en todas las plataformas , excepto en HTML5, el mapa tendrá la clave "is64bit", que será true o false dependiendo de si el corredor se ejecuta en modo de 64 bits o no.

Windows / UWP

En Windows Desktop y Windows UWP, obtendrá algunos datos extra de DX11 con las siguientes teclas disponibles en el mapa DS (útil para extensions):

  • udid: Un identificador que es único para la máquina
  • video_d3d11_device: puntero al dispositivo DX11
  • video_d3d11_context: puntero al contexto DX11
  • video_d3d11_spawnchain: puntero a la cadena de intercambio DX11
  • video_adapter_vendorid string con el identificador de proveedor del adaptador
  • video_adapter_deviceid string con el identificador de dispositivo del adaptador
  • video_adapter_subsysid string con el identificador del subsistema del adaptador
  • video_adapter_revision string con el número de revisión del adaptador
  • video_adapter_description string con la descripción del adaptador de vídeo
  • video_adapter_dedicatedvideomemory string con el tamaño de la memoria de vídeo
  • video_adapter_dedicatedsystemmemory string con el tamaño de la memoria del sistema (utilizada por el adaptador DX11)
  • video_adapter_sharedsystemmemory string con el tamaño de la memoria compartida del sistema

macOS

La función devuelve alguna información limitada de OpenGL en macOS:

  • udid: Un identificador que es único para la máquina
  • gl_vendor_string GL_VENDOR de OpenGL, ejemplo - "Intel Inc."
  • gl_version_string GL_VERSION de OpenGL, ejemplo - "2.1 INTEL-16.2.16"
  • gl_renderer_string GL_RENDERER de OpenGL, ejemplo - "Intel(R) Iris(TM) Graphics 6100"

Ubuntu

Contiene la misma información que en macOS.

Android

En la plataforma Android obtendrá algunos datos extra de varias APIs diferentes con las siguientes claves disponibles en el mapa de DS:

  • android_tv: esto devolverá true si se detecta que el dispositivo es un dispositivo de TV Android, o false en caso contrario
  • GL_VERSION: la versión de OpenGL según lo informado por el controlador
  • GL_VENDOR: el proveedor de OpenGL según lo informado por el controlador
  • GL_RENDERER: el renderizador de OpenGL según lo informado por el controlador
  • GL_EXTENSIONS: los extensions de OpenGL que están disponibles según lo informado por el controlador
  • GL_SHADING_LANGUAGE_VERSION: la versión de GLSL que soporta este controlador OpenGL
  • GL_MAX_TEXTURE_SIZE: el tamaño máximo de la textura soportado por este controlador OpenGL
  • SDK_INT: valor de android.os.Build.VERSION.SDK_INT (véase la nota siguiente)
  • RELEASE: valor de android.os.Build.VERSION.RELEASE
  • MODEL: valor de android.os.Build.MODEL
  • DEVICE: valor de android.os.Build.DEVICE
  • MANUFACTURER: valor de android.os.Build.MANUFACTURER
  • CPU_ABI: valor de android.os.Build.CPU_ABI
  • CPU_ABI2: valor de android.os.Build.CPU_ABI2
  • BOOTLOADER: valor de android.os.Build.BOOTLOADER
  • BOARD: valor de android.os.Build.BOARD
  • VERSION: valor de os.version del método Android System.getProperty.
  • REGION: valores de user.region del método Android System.getProperty
  • VERSION_NAME: valor de versionName de este paquete - vea aquí para más información.
  • PHYSICAL_KEYBOARDSi pensamos que hay un teclado físico, entonces el string "TRUE" de lo contrario "FALSE"

NOTA: Para obtener información sobre cualquier variable de android.os.Build, consulte la documentación de Android Developer.

iOS y tvOS

Estas son las claves devueltas en iOS y tvOS (para información detallada, consulte la documentación para desarrolladores de Apple):

  • name: string, según informa [[UIDevice currentDevice] name]
  • systemName: string, según informa [[UIDevice currentDevice] systemName]
  • systemVersion: cadena según lo informado por [[UIDevice currentDevice] systemVersion]
  • model: cadena según lo informado por [[UIDevice currentDevice] model]
  • localisedModel: cadena según lo informado por [[UIDevice currentDevice] localizedModel]
  • uiIdiom: cadena según lo informado por [[UIDevice currentDevice] userInterfaceIdiom]
  • platform: hw.machine string recogido de sysctlbyname
  • hwModel: hw.model string recogido de sysctlbyname
  • freeDiskSpace: El espacio libre en el disco
  • totalDiskSpace: El espacio total del disco
  • cpuFrequency: La frecuencia de la CPU
  • busFrequency: La frecuencia del bus
  • cpuCount: El recuento de la CPU
  • totalMemory: La memoria total del dispositivo
  • userMemory: La memoria del usuario
  • maxSocketBufferSize: El tamaño máximo de la toma buffer
  • Claves adicionales que contienen información gráfica de OpenGL

Opera GX

Las siguientes teclas adicionales se incluirán en el mapa de DS para Opera GX:

  • mobile: Esto será 1. 0 si el juego se ejecuta en un navegador móvil, y 0.0 si no lo es.

HTML5

Devuelve -1.

Nintendo Switch

Devuelve -1.

PlayStation 4

Consulte la documentación de PS4 para obtener información sobre las funciones a las que se hace referencia aquí.

La función devuelve las siguientes teclas en una consola PlayStation 4:

  • display_safe_area_ratio: Un valor de punto flotante devuelto por sceSystemServiceGetDisplaySafeAreaInfo().
  • is_neo_mode: Será 1 si el juego se ejecuta en una Neo (PS4 Pro) y 0 si se ejecuta en una PS4 normal.
  • enter_button_assign: Un valor entero como el devuelto por sceSystemServiceParamGetInt(SCE_SYSTEM_SERVICE_PARAM_ID_ENTER_BUTTON_ASSIGN, &enterassign); un valor de 0 significa que el botón Círculo se utiliza como "asignación" y 1 significa que el botón X se utiliza en su lugar.

PlayStation 5

La función devuelve las siguientes teclas en una consola PlayStation 5:

  • display_safe_area_ratio: Igual que en PS4 (ver arriba).
  • display_resolution: "HD", "4K" o "desconocido"
  • display_dynamic_range: "HDR", "SDR" o "desconocido"
  • display_refresh_rate: "120Hz", "60Hz" o "Desconocido"

Xbox One

La información devuelta en Xbox One es mayormente la misma que Windows, sin embargo las claves video_adapter_* y udid son 0 (excepto video_adapter_description que es un string "" vacío). Tampoco tiene la clave video_d3d11_swapchain.

Esta plataforma también devuelve la siguiente información:

  • is_xbox_one_x: Será 1 si el juego se ejecuta en una Xbox One X y 0 si se ejecuta en una Xbox One básica.

Xbox Serie X/S

La información devuelta en las consolas Xbox Series X/S es mayoritariamente la misma que la de Xbox One, sin embargo las teclas video_d3d11_* han sido sustituidas por video_d3d12_*.

Estas son algunas de las claves que devuelve esta plataforma, además de las que aparecen en el apartado de Xbox One:

  • video_d3d12_cmdqueue: Apunta a la cola de comandos DX12
  • video_d3d12_cmdlist: Señala la lista de comandos DX12
  • video_d3d12_currentrt: Señala el objetivo de renderizado DX12 actual

Sintaxis:

os_get_info()

Devuelve:

DS Map ID

Ejemplo:

if (os_type == os_android)
{
    var _info = os_get_info();
    
    if (_info[? "android_tv"])
    {
        global.android_tv = true;
    }
}

El código anterior comprueba si el sistema operativo actual es Android, obtiene la información del sistema operativo y lee el mapa devuelto para comprobar si el juego se está ejecutando en un televisor Android; en ese caso establece la variable personalizada global.android_tv a true.
