Sistema operativo y compilador

Al crear juegos multiplataforma, suele ser de vital importancia obtener los detalles del sistema operativo que utiliza el dispositivo que ejecuta el juego. Cosas como el idioma, la versión o el estado de la red pueden servir para adaptar tu juego al sistema que lo ejecuta y hacer que la experiencia del usuario final sea la mejor posible, por lo que GameMaker tiene una serie de funciones especiales que pueden utilizarse para obtener la información necesaria.

Las siguientes funciones y variables existen para que puedas utilizarlas para obtener ciertos detalles sobre el sistema operativo o el navegador que está ejecutando tu juego:

  • os_browser
  • os_device
  • os_type
  • os_version
  • os_is_paused
  • os_is_network_connected
  • os_get_config
  • os_get_language
  • os_get_region
  • os_get_info
  • os_powersave_enable
  • os_lock_orientation
  • os_check_permission
  • os_request_permission

Algunas funciones se precompilan cuando se ejecuta el juego (es decir, se utilizan para definir cómo se compilará el juego final), y otras se utilizan también cuando se ha compilado el juego para obtener detalles específicos sobre el entorno runtime. Aquí puedes encontrar la lista completa de estas funciones:

  • GM_build_date
  • GM_build_type
  • GM_version
  • GM_runtime_version
  • GM_project_filename
  • gml_release_mode
  • gml_pragma
  • parameter_count
  • parameter_string
  • environment_get_variable

En Windows, la resolución del programador de hilos de Windows se establece por defecto en 1ms cuando se ejecuta el juego. Las siguientes funciones están disponibles para recuperar y cambiar la resolución del programador de hilos de Windows en tiempo de ejecución:

  • scheduler_resolution_get
  • scheduler_resolution_set

Por último, cuando se trabaja con extensiones puede ser necesario poder definir y llamar a funciones externas en runtime, por lo que se utilizaría lo siguiente:

NOTA: Estas son funciones heredadas y debería definir cualquier función de extension dentro del propio Editor de Extension.

NOTA: Estas funciones son sólo para las plataformas de destino Windows y macOS.

  • external_define
  • external_call
  • external_free
