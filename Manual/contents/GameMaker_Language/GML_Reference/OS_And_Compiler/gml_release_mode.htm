gml_release_mode

Cuando tu juego se compila, también incluye ciertas funciones internas para comprobar los errores mientras se ejecuta. Estas comprobaciones internas cubren muchos aspectos del código de un juego, incluyendo rangos, parámetros y algunos aspectos generales del compilador interno de GML y son lo que genera los mensajes de error finales en muchos casos. Sin embargo, esta función se puede utilizar para switch estas comprobaciones internas (es decir, eliminarlas completamente del juego compilado), dando un impulso de velocidad a su proyecto final.

Por favor, ten en cuenta que esto significa que si haces algo en tu código de juego que sea incorrecto, el juego compilado puede simplemente bloquearse, o puede experimentar un comportamiento muy inesperado. Por ejemplo, si se accede a un DS Grid en una coordenada fuera del tamaño definido, en el modo de desarrollador recibirás un mensaje de error en la ventana del compilador, pero las comprobaciones realizadas permitirán que el juego siga funcionando. Sin embargo, en el modo de lanzamiento, estas comprobaciones no existirán, por lo que el código podría devolver un puntero NULL, o un puntero malo que a su vez apuntará a alguna memoria aleatoria. Así que, en el mejor de los casos, podrías obtener los datos erróneos devueltos, o en el peor, podrías sobrescribir algo en la memoria - datos, gráficos, o incluso el propio código. Otro ejemplo sería cuando se construyen las primitivas - si proporcionas los datos de color, posición, textura, etc... en un orden diferente al que definiste cuando creaste el formato vertex, entonces el juego puede fallar o producir resultados inesperados cuando el modo de liberación esté activado.

Sintaxis:

gml_release_mode(flag);

ArgumentoTipo Descripción
flag Either true para establecer el modo de liberación o false para el desarrollo general.

Devuelve:

Ejemplo:

gml_release_mode(true);

El código de ejemplo anterior hará que el juego utilice el modo de liberación cuando se compile.
