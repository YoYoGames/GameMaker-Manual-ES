font_set_cache_size

Al realizar la mezcla y el coloreado de imágenes, HTML5 no puede hacerlo dinámicamente de la misma manera que un ejecutable. Por lo tanto, GameMaker tiene que guardar temporalmente una copia mezclada de las imágenes y cargarlas cuando sea necesario. Esta función establece cuántas copias mezcladas de la font dada pueden ser almacenadas en caché antes de que las antiguas sean sobrescritas. El valor por defecto es 4.

NOTA para sprite fonts debería utilizar la función equivalente para sprites, sprite_set_cache_size().

Sintaxis:

font_set_cache_size(ind, max);

ArgumentoTipo Descripción
ind The index of the font to change the cache size of.
max The maximum number of cached copies of the font that can be stored.

Devuelve:

Ejemplo:

font_set_cache_size(fnt_MainMenu, 2);

Esto establecerá el font cache del font indexado en la variable "fnt_MainMenu" a 2 copias.
