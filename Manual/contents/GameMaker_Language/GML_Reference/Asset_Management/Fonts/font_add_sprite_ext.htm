font_add_sprite_ext

Con esta función puedes utilizar una "tira de sprites" (el propio sprite debe ser un sprite asset del Navegador Asset, o un sprite que hayas añadido a los recursos del juego utilizando sprite_add) para crear un nuevo font asset, donde cada sub-imagen sería un símbolo o letra individual. A diferencia del sistema normal font_add_sprite() que tiene un orden específico para las sub-imágenes del sprite, esta función mapeará las sub-imágenes de sprite basándose en el argumento "string_map" de la función. Este argumento es un string que puedes usar para decirle a GameMaker qué orden tienen las sub-imágenes del sprite font y las mapeará automáticamente al escribir el texto. Así, por ejemplo, si tienes un mapa de cadenas de "AaBbCcDdEeFfGgHh", tu sprite font debe tener las sub-imágenes ordenadas de esta manera. Puede definir "espacio" como cualquier carácter que desee, por ejemplo una sola línea del tamaño que desee que tenga el espacio, y todos los espacios en el texto se renderizarán con esa anchura (pero la imagen elegida nunca se renderizará), o si no suministra un sprite para el espacio, entonces se utilizará en su lugar la anchura del carácter más ancho en el sprite font.

También puede establecer que el espaciado de font sea proporcional (true) o no (false), donde un font proporcional se espacia en base a la anchura real de las letras (por lo que la letra "i" ocupa menos room que la letra "w", por ejemplo) mientras que un font no proporcional se espaciará en base a la anchura de la subimagen, como un font monoespaciado. Por último, se puede definir el espacio que se dejará entre cada letra al escribir, y éste puede ser cualquier valor entero, siendo 0 sin espacio (las letras se "tocarán" si son proporcionales).

La función devuelve un valor de índice que debe almacenarse en una variable, ya que se necesitará en todos los códigos posteriores que hagan referencia a este font, o devolverá -1 si la función falla por cualquier motivo.

Cabe señalar que para que las funciones de alineación de font funcionen (como draw_set_halign()), font sprite debe tener su origen en la esquina superior izquierda. Si utiliza otros valores, tendrá que tener en cuenta el desplazamiento del origen al dibujar el texto con font.

NOTA Cuando cargue un font en GameMaker debe recordar eliminarlo de nuevo (con font_delete()) cuando ya no lo necesites, de lo contrario existe el riesgo de que se produzca una fuga de memoria que ralentizará y acabará por colapsar tu juego.

Sintaxis:

font_add_sprite_ext(spr, string_map, prop, sep);

Argumento Tipo Descripción
spr Activo Sprite The sprite to add a font based on.
string_map Cadena String from which sprite sub-image order is taken.
prop Booleano Set as proportional font or not.
sep Real The space to leave between each letter.

Devuelve:

Font Asset

Ejemplo:

global.Font = font_add_sprite_ext(spr_CalcFont, "0123456789+-*/=", true, 2);

El código anterior creará un nuevo font asset a partir del sprite indexado en la variable "spr_CalcFont" y almacenará el índice del nuevo font en la variable "global.Font". Las sub-imágenes del sprite font serán mapeadas al string especificado.
