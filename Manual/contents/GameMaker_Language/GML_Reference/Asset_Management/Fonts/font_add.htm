font_add

Esta función puede utilizarse para añadir un font a su juego a partir de un archivo font presente en los Archivos Incluidos. Puede definir el tamaño de font (en puntos), así como si el font debe estar en negrita o en cursiva, y también puede definir el rango de caracteres a incluir. La función devuelve un valor de índice que debe ser almacenado en una variable, ya que será necesario en todo el código posterior que haga referencia a este font, o devolverá -1 si la función falla por cualquier motivo.

Tenga en cuenta que el archivo font debe ser un archivo *.ttf o *.otf, que también puede ser útil para añadir archivos no estándar fonts como los asiáticos o los árabes fonts.

ADVERTENCIA!: Si incluyes un font de este tipo en los Archivos Incluidos, debe tener licencia para su distribución con el juego.

Para el módulo de destino HTML5, esta función sólo puede utilizarse para añadir fuentes web, y sólo funcionará si WebGL no está habilitado - si WebGL está habilitado, entonces debe utilizar un recurso general font (es decir: utilizando draw_set_font() junto con un font del navegador Asset ) o un mapa de bits fonts (véase font_add_sprite()).

¡IMPORTANTE! Cuando se dirige a HTML5, el texto en cursiva y en negrita sólo se admite si el propio font asset o web font tiene incluidos estos caracteres. No se sintetizarán a partir de la base font.

Cuando se carga un font desde un archivo, el tamaño del font está en píxeles y los valores primero y último se ignoran, lo que significa que se añadirán todos los glifos en el font. Los Fonts añadidos de esta manera se asignan a su propia página de textura, por lo que se debe tener cuidado al utilizar esta función, ya que incrementará el número de intercambios de textura al dibujar. También hay que tener en cuenta que fonts puede parecer un poco más grande cuando se dibuja, ya que los glifos pueden tener partes que se dibujan fuera de la caja delimitadora definida para el font. También hay que tener en cuenta que hay una limitación de alrededor de 200 glifos que se pueden renderizar en un solo cuadro para un font en particular.

Por defecto, los fonts añadidos tendrán el antialiasing activado. Sin embargo, puede comprobar el estado del AA en cualquier momento utilizando la función font_add_get_enable_aa() y puede activar/desactivar el AA utilizando la función font_add_enable_aa().

NOTA Cuando cargue un font en GameMaker debe recordar eliminarlo de nuevo (con font_delete()) cuando ya no lo necesites, de lo contrario existe el riesgo de que se produzca una fuga de memoria que ralentizará y acabará por colapsar tu juego.

Sintaxis:

font_add(name, size, bold, italic, first, last);

Argumento Tipo Descripción
name Cadena The path of the font file in the Included Files (*.ttf o *.otf).
size Real The size of the font - points for Web Fonts, pixels for file fonts.
bold Booleano Whether the font is bold (true) o no (false).
italic Booleano Whether the font is italic (true) o no (false).
first Real The first character to include (if you're unsure, go for 32).
last Real The last character to include (if you're unsure, go for 128).

Devuelve:

Font Asset

Ejemplo:

new_font = font_add("gamefont.ttf", 24, true, true, 32, 128);

Esto cargará el archivo "gamefont.ttf" presente en los Archivos Incluidos del juego con un tamaño de 24pt. El font estará en negrita y cursiva, y el rango de fuentes incluirá letras mayúsculas y minúsculas, números y toda la puntuación común. Una vez cargado el font, su ID se almacenará en la variable new_font.
