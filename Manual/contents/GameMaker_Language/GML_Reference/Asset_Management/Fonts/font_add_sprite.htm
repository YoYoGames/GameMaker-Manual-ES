font_add_sprite

Con esta función puedes usar una "tira de sprites" para crear un nuevo font asset, donde cada sub-imagen sería un símbolo o letra individual. El índice es el número de mapa UTF8 para el primer carácter a utilizar y los rangos a utilizar son los mismos que para un font normal, empezando por "!" pero recuerde que sólo debe incluir lo que necesita y que debe utilizar la función ord() para obtener el valor UTF8 correcto para la letra inicial. El sprite font creado tendrá entonces las sub-imágenes ajustadas a un glifo individual del font, donde el "primer" glifo tendrá el image_index de 0. Puede definir el "espacio" como cualquier carácter que desee, por ejemplo una sola línea del tamaño que desee que tenga el espacio, y todos los espacios en el texto se renderizarán con esa anchura (pero la imagen elegida nunca se renderizará), o si no proporciona un sprite para el espacio entonces se utilizará la anchura del carácter más ancho en el sprite font en su lugar.

También puede establecer que el espaciado de font sea proporcional (true) o no (false), donde un font proporcional se espacia en base a la anchura real de las letras (por lo que la letra "i" ocupa menos room que la letra "w", por ejemplo) mientras que un font no proporcional se espaciará en base a la anchura de la subimagen, como un font monoespaciado. Por último, puede definir el espacio que se dejará entre cada letra al escribir, y éste puede ser cualquier valor entero, siendo 0 sin espacio (las letras se "tocarán" si son proporcionales).

La función devuelve un valor de índice que debe almacenarse en una variable, ya que se necesitará en todos los códigos posteriores que hagan referencia a este font, o devolverá -1 si la función falla por cualquier motivo.

También hay que tener en cuenta que no puede haber espacios en el mapa de caracteres, de modo que si, por ejemplo, utiliza '.' y '0' pero no '/', deberá incluir '/' de todos modos o dejar una subimagen en blanco entre los dos caracteres utilizados.

El propio sprite puede ser un sprite asset del navegador Asset o uno cargado desde un archivo utilizando primero sprite_add para obtener el índice del sprite cargado (que luego se utiliza en la función). Es importante tener en cuenta que para que las funciones de alineación de font (como draw_set_halign()) funcionen como se espera cuando se utiliza un sprite font, el sprite debe tener su origen fijado en la esquina superior izquierda. Si utiliza otros valores, tendrá que tener en cuenta el desplazamiento del origen al dibujar el texto con font.

NOTA Cuando cargues un font en GameMaker debes recordar eliminarlo de nuevo (con font_delete()) cuando ya no lo necesites, de lo contrario existe el riesgo de que se produzca una fuga de memoria que ralentizará y acabará por colapsar tu juego.

Sintaxis:

font_add_sprite(spr, first, prop, sep);

Argumento Tipo Descripción
spr Activo Sprite The sprite to add a font based on.
first Real The index of the first available character.
prop Booleano Set as proportional font or not.
sep Real The space to leave between each letter.

Devuelve:

Font Asset

Ejemplo:

global.Font = font_add_sprite(spr_HUD_Font, ord("!"), true, 2);

El código anterior creará un nuevo font asset a partir de un sprite asset y almacenará el índice del nuevo font en la variable "global.Font" para su uso posterior.
