La estructura de la pista

La pista struct puede utilizarse tanto para laspistas deactivosde nivel superior como para laspistas de parámetros de subpistas , y el comportamiento de la pista está definido por dos cosas, su nombre y su tipo. Los structs de pista se crean con la función sequence_track_new() y pueden recuperarse de sequence assets utilizando la propiedad activeTracks de la Estruc de la Instancia de la Secuencia o la propiedad tracks de la Estruc de la Secuencia Object .

Las propiedades disponibles en la pista struct son:

Estructura de la pista de secuencia
Variable Tipo Descripción
type Secuencia Tipo de pista Constante Contiene un Constante del tipo de pista de la secuencia que describe el tipo de pista que es. Puede ser cualquiera de las constantes indicadas en la sección "Tipo" más abajo.
name Cadena Al crear una pista de "nivel superior" asset, el nombre que le dé puede ser cualquier string que necesite para identificar la pista. Sin embargo, para las pistas de parámetros, necesita especificar strings para decirle a GameMaker qué tipo de pista de parámetros está creando. Esto se amplía en la sección "Nombre" más adelante.
tracks Matriz de Estructura de la pista de la secuencias Esta propiedad permite acceder a la lista de pistas que son hijas de esta pista. Al obtener esta propiedad se devuelve un array de Estructura de la pista de secuencias, y cuando se establece esta propiedad se debe especificar una matriz de Estructura de la pista de la secuenciadebe especificarse un array de s.
visible Booleano Indica si esta pista es visible (el valor es true) o no (el valor es false). Puedes obtener o establecer este valor y si una pista no es visible entonces ninguna de sus pistas hijas se dibujará tampoco.
keyframes Matriz de Secuencia Keyframe Structs Esta propiedad permite acceder a la lista de structs de fotogramas clave de la pista. Cuando se obtiene esta propiedad se devuelve un array de keyframe structs, y cuando se establece esta propiedad se debe especificar un array de keyframe structs.

Nombre

La propiedad name puede ser cualquiera de las siguientes cadenas:

  • "posición" - cuando se establece en una pista que es de tipo seqtracktype_real indica que el contenido de esta pista debe ser utilizado como datos de posición.
  • "scale" - cuando se establece en una pista que es del tipo seqtracktype_real esto indica que el contenido de esta pista debe ser utilizado como datos de escala.
  • "rotación" - cuando se establece en una pista que es de tipo seqtracktype_real esto indica que el contenido de esta pista debe ser utilizado como datos de rotación.
  • "blend_multiply" o "image_blend" - cuando se establece en una pista de tipo seqtracktype_colour indica que el contenido de esta pista debe utilizarse como datos de multiplicación del color. Los datos de keyframe para dicha pista deben ser una matriz de cuatro valores con el formato [A, R, G, B]. Tenga en cuenta que los valores de cada componente se expresan entre 0 y 1, donde 0 corresponde al valor HEX #00 y 1 corresponde al valor HEX #FF (0 - 255 como se muestra en el selector de color para las pistas de mezcla de imágenes en el Editor Sequences ).
  • "Velocidad_de_la_imagen" - cuando se establece en una pista del tipo seqtracktype_real, indica que el contenido de esta pista debe utilizarse como valor de la velocidad de la imagen.
  • "image_index" - cuando se establece en una pista que es de tipo seqtracktype_real indica que el contenido de esta pista debe ser utilizado como el valor del índice de la imagen.
  • "Ángulo_de_imagen" - cuando se establece en una pista que es del tipo seqtracktype_real indica que el contenido de esta pista debe ser utilizado como el valor del ángulo de la imagen.


    Nombres de parámetros de pistas de texto (sólo se utilizan en las pistas de texto):
  • "frameSize" - cuando se establece en una pista seqtracktype_real, indica que el contenido de esta pista debe ser utilizado como el tamaño del marco para el texto.Cada keyframe para esta pista debe tener dos canales, 0 y 1, donde 0 almacena el tamaño X y 1 el tamaño Y del marco.
  • "characterSpacing" - también se utiliza con las pistas de seqtracktype_real, se utiliza como el espaciado de caracteres para el texto. Cada keyframe para esta pista debe tener un único canal que contenga el valor de espaciado.
  • "lineSpacing" - también se utiliza con las pistas de seqtracktype_real, se utiliza como el interlineado para el texto. Cada keyframe para esta pista debe tener un único canal que contenga el valor del espaciado.
  • "paragraphSpacing" - también se utiliza con las pistas de seqtracktype_real, se utiliza como el espaciado de párrafo para el texto. Cada keyframe para esta pista debe tener un único canal que contenga el valor de espaciado.
