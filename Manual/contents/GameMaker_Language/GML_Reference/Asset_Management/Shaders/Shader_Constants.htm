Constantes de sombreado

Aparte de las funciones y constantes de shader definidas en las páginas de referencia del lenguaje de sombreado OpenGL ES (GLSL ES), también hay una serie de constantes de shader disponibles que son exclusivas de GameMaker.

Las siguientes constantes de la matriz de visualización pueden utilizarse como índices de la matriz cuando se utiliza la constante de la matriz shader gm_Matrices :

Constante Descripción
MATRIX_VIEW Esta constante de índice de matriz mantiene el índice de la matriz de la vista actual. El índice devuelto se utilizaría como un valor de matriz al llamar a la constante gm_Matrices dentro del código shader.
MATRIX_PROJECTION Esta constante de índice de matriz mantiene el índice de la matriz de proyección actual. El índice devuelto se utilizaría como un valor de matriz al llamar a la constante gm_Matrices dentro del código shader.
MATRIX_WORLD Esta constante de índice de matriz mantiene el índice de la matriz mundial actual. Esto puede ser usado para cosas como la iluminación si tienes información de luz en el espacio-mundo. El índice devuelto se utilizaría como un valor de matriz al llamar a la constante gm_Matrices dentro del código shader.
MATRIX_WORLD_VIEW Esta constante de índice de matriz mantiene el índice del resultado de las matrices de mundo y vista multiplicadas juntas. Esto se utiliza a menudo para cosas como la niebla. El índice devuelto se utilizaría como un valor de matriz al llamar a la constante gm_Matrices dentro del código shader.
MATRIX_WORLD_VIEW_PROJECTION Esta constante de índice de matriz mantiene el índice del resultado de las matrices de mundo, vista y proyección multiplicadas juntas. Esta es la matriz de transformación normal utilizada para las posiciones de vertex. El índice devuelto se utilizaría como un valor de matriz al llamar a la constante gm_Matrices dentro del código shader.
MATRIX_MAX No se trata de un índice de matriz, sino que devuelve el tamaño de la matriz en vertex shader.
MAX_VS_LIGHTS No se trata de un índice de matriz, sino que devuelve el número de las luces en el vertex shader.

La siguiente constante también está disponible para la iluminación:

Constante Descripción
MAX_VS_LIGHTS El número máximo de luces puntuales y direccionales disponibles en el shader

Los siguientes uniformes y constantes de matriz predefinidos pueden utilizarse en su shader para acceder a valores específicos de GameMaker:

Constante Descripción
gm_Matrices[matrix] Esta constante de matriz devuelve una matriz de transformación y es uno de los uniformes predefinidos disponibles que GameMaker crea para que usted lo utilice dentro del editor de código shader. El índice de la matriz se elige de una de las constantes enumeradas anteriormente.
gm_BaseTexture Esta es una constante de muestreo 2D que devuelve la textura del object actual, tal y como lo establece GameMaker. Así que sería la actual sprite, superficie o textura que normalmente se utilizaría al dibujar sin que se llame a la shader.
gm_LightingEnabled Esto se puede utilizar para obtener o establecer la iluminación de GameMaker cuando se utiliza el 3D.
gm_FogStart Esto se puede utilizar para obtener la distancia en la que los polígonos comienzan a mezclarse con el color de la niebla.
gm_RcpFogRange Esto se puede utilizar para obtener la distancia a la que la niebla es máxima y ya no se ve nada.
gm_PS_FogEnabled Esto devolverá true o false si la GPU tiene activada la niebla de píxeles o no.
gm_FogColour Esto se puede utilizar para obtener el color de la niebla utilizado por GameMaker.
gm_VS_FogEnabled Esto devolverá true o false si la GPU tiene la niebla vertex activada o no.
gm_AlphaTestEnabled Esto se puede utilizar para obtener la prueba alfa en el shader. Ver gpu_set_alphatestenable() para más información sobre las pruebas alfa.
gm_AlphaRefValue Se puede utilizar para obtener el valor de referencia de la prueba alfa actual. Consulte gpu_set_alphatestref() para más información sobre la referencia de prueba alfa.
