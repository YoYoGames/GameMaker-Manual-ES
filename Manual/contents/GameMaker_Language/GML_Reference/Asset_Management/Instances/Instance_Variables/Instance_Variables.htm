Variables de instancia

Cuando creas un nuevo object, éste vendrá con ciertas variables ya inicializadas con valores por defecto. Estas variables las llamamos variables de instancia, ya que se crearán para cada instancia que coloques del object en tu juego rooms, y una vez que se crea una instancia los valores de estas variables son únicos para esa instancia y sólo para esa instancia. Algunas de estas variables las usarás mucho, como x y y, mientras que otras se requieren menos dependiendo de lo que quieras hacer, pero en general son muy útiles y siempre que sea posible deberías intentar usarlas en lugar de crear tus propias variables personalizadas.

La mayoría de las variables de instancia pueden establecerse y leerse, lo que te permite cambiar las propiedades y el comportamiento de una instancia simplemente ajustando el valor de una determinada variable - puedes evitar que una instancia dibuje, por ejemplo, simplemente estableciendo la variable incorporada visible en false. A continuación puedes encontrar las diferentes variables que se inicializan para todas las instancias de todos los objects en tu juego.

IMPORTANTE Si una instancia está en un sequence entonces algunas de estas variables - como image_xscale / image_yscale / image_angle / x / y - se sobrescribirán cuando el sequence actualice cada paso después de comenzar a ser reproducido.

Estas variables se ocupan de las propiedades generales de la instancia:

  • id
  • visible
  • solid
  • persistent
  • depth
  • layer
  • alarm

Estas variables se ocupan de la posición y el movimiento de la instancia:

  • direction
  • friction
  • gravity
  • gravity_direction
  • hspeed
  • vspeed
  • speed
  • xstart
  • ystart
  • x
  • y
  • xprevious
  • yprevious

Propiedades de los objetos

La siguiente variable contiene el ID de índice único de la object desde la que se creó la instancia:

  • object_index

Todas estas variables están relacionadas con la sprite asignada a la instancia y pueden utilizarse para cambiar lo que se dibuja y cómo:

  • sprite_index
  • sprite_width
  • sprite_height
  • sprite_xoffset
  • sprite_yoffset
  • image_alpha
  • image_angle
  • image_blend
  • image_index
  • image_number
  • image_speed
  • image_xscale
  • image_yscale

Estas variables se ocupan de la máscara de colisión:

  • mask_index
  • bbox_bottom
  • bbox_left
  • bbox_right
  • bbox_top

Estas variables están relacionadas con paths y con la forma en que la instancia interactúa con una, si se le asigna:

  • path_index
  • path_position
  • path_positionprevious
  • path_speed
  • path_scale
  • path_orientation
  • path_endaction

Estas variables sirven para configurar una instancia para utilizar una línea de tiempo:

  • timeline_index
  • timeline_running
  • timeline_speed
  • timeline_position
  • timeline_loop

Secuencias

Las siguientes variables almacenan información sobre la Sequence que puede estar controlando la instancia en un momento dado:

  • in_sequence
  • sequence_instance

Hay un gran número de variables incorporadas para su uso con las funciones de física de GameMaker, por lo que para mantener las cosas más claras, se pueden encontrar en la sección del manual que cubre todo lo relacionado con la simulación de la física:

  • Física
