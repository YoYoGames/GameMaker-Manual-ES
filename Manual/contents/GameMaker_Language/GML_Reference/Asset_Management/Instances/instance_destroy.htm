instance_destroy

Se llama a esta función cada vez que se desea "destruir" una instancia, normalmente activando un Evento de Destrucción y también un Evento de Limpieza. Esto la eliminará de room hasta que se reinicie room (a menos que room sea persistente). Llamar a la función sin argumentos simplemente destruirá la instancia que está actualmente en el ámbito y ejecutando el código, pero puedes proporcionar un argumento opcional "id" y apuntar a una instancia específica usando el valor de la instancia id, o puedes apuntar a todas las instancias de un object particular usando un argumento object_index. Por ejemplo:

instance_destroy(other);      // destroy the "other" instance in a Collision Event
instance_destroy(obj_Bullet); // destroy ALL instances of the object "obj_Bullet"

La segunda opción flag permite "desactivar" el evento de destrucción de la instancia que se está destruyendo. Por defecto, el Evento de Destrucción siempre se llevará a cabo, pero si estableces este flag a false entonces puedes hacer que la instancia se destruya a sí misma y omitir la realización de ese evento.

¡IMPORTANTE! Esto omitirá el evento de destrucción, pero el evento de limpieza seguirá siendo llamado.

Cabe destacar que cuando se destruye una instancia, su evento de destrucción es llamado inmediatamente después del código o acción que llama a la función de destrucción. Además, aunque se ejecute el evento de destrucción, la instancia no se elimina inmediatamente del juego, y seguirá ejecutando el código contenido en el evento actual. Sólo cuando el evento actual termine será eliminado del juego.

Así, si tienes, por ejemplo, este código:

if (hp <= 0) instance_destroy();
score += 10;

La variable "score" se incrementará aunque la función instance_destroy haya sido llamada, y la instancia será finalmente eliminada de tu juego al final del evento. Ten cuidado con esto, ya que si (por ejemplo) has creado un recurso dinámico para la instancia, como una estructura de datos, y luego has destruido ese recurso en el evento de destrucción, si hay alguna referencia a él después de que se haya realizado la función o acción de destrucción, entonces obtendrás un error de "recurso desconocido", ya que el evento de destrucción lo eliminó del juego.

Sintaxis:

instance_destroy([id, execute_event_flag]);

Argumento Tipo Descripción
id ID de la instancia o activo del objeto OPTIONAL El ID de la instancia a destruir, o el object asset para destruir instancias (opcional, por defecto es la instancia que llama)
execute_event_flag Booleano OPTIONAL Establezca true o false para realizar o no el evento Destroy (opcional, por defecto es true)

Devuelve:

N/A

Ejemplo:

if (bbox_right < 0 || bbox_left > room_width || bbox_bottom < 0 || bbox_top > room_height)
{
    instance_destroy(id, false);
}

El código anterior destruirá la instancia que ejecuta el código sin llamar al evento Destroy si la instancia está fuera de los límites de room.
