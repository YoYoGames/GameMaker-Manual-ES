audio_play_sound_on

Con esta función puedes reproducir cualquier recurso sonoro a través de un emisor, con cualquier cambio en la ganancia, posición, tono o velocidad del emisor que afecte a cómo el usuario oye el sonido final que se reproduce. Se proporciona el índice del emisor que se va a utilizar, el índice del sonido que se va a reproducir y, a continuación, se especifica si el sonido va a loop o no. A continuación, puede asignar una prioridad, que se utiliza para determinar cómo se tratan los sonidos cuando el número de sonidos que se reproducen supera el límite establecido por la función audio_channel_num(). Los sonidos de menor prioridad se detendrán en favor de los de mayor prioridad, y el valor de la prioridad puede ser cualquier número real (el valor real es arbitrario, y puede ser de 0 a 1 o de 0 a 100, ya que GameMaker les dará la misma prioridad). Tenga en cuenta que cuando se trata de la prioridad, cuanto mayor sea el número, mayor será la prioridad, de tal manera que un sonido con prioridad 100 será favorecido sobre un sonido con prioridad 1.

Esta función también devolverá un número de índice único para el sonido que se está reproduciendo, el cual puede ser almacenado en una variable para que puedas pausarlo o detenerlo con las funciones apropiadas. Esto significa que si tienes varias instancias del mismo sonido reproduciéndose en un momento dado, puedes apuntar a una sola instancia de ese sonido para tratarla con las funciones de audio.

Sintaxis:

audio_play_sound_on(emitter, sound, loop, priority, [gain], [offset], [pitch], [listener_mask]);

Argumento Tipo Descripción
emitter Identificación del emisor de audio The index of the emitter to use.
sound Activo sonoro The index of the sound to use.
loop Booleano Flags the sound as looping or not.
priority Real Set the channel priority for the sound.
gain Real OPTIONAL Valor de la ganancia.
offset Real OPTIONAL El tiempo (en segundos) en el que se establece el punto de inicio. Los valores más largos que la longitud del sonido dado se ignoran.
pitch Real OPTIONAL El multiplicador de paso (por defecto 1).
listener_mask Real OPTIONAL Los datos de la máscara de bits a establecer para el sonido. En el objetivo HTML5 esto no tendrá ningún efecto ya que el objetivo no admite más de un oyente.

Devuelve:

Sound Instance ID

Ejemplo 1:

if global.SFX
{
    audio_play_sound_on(s_emit, snd_Explode, false, 1);
}

El código anterior comprueba la variable global "SFX" y si devuelve true entonces el sonido indexado en la variable "snd_Explode" se reproducirá a través del emisor indexado en la variable "s_emit" con una prioridad baja y sin bucle.

Ejemplo 2:

if hit == true
{
    audio_play_sound_on(s_emit, snd_Hit, false, 1, 1.3);
}

El código anterior comprueba si la instancia que ejecuta el código fue golpeada. Si true reproduce el sonido "snd_Hit" a través del emisor "s_emit" con una ganancia ligeramente superior de 1,3. La ganancia establecida aquí se multiplica por la ganancia del emisor para obtener la ganancia final con la que se reproduce el sonido.
