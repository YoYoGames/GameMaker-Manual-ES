audio_debug

Esta función puede utilizarse para mostrar información de depuración sobre el sistema de audio, con true encendiéndola y false para switch apagándola. Cuando se debugging audio de esta manera, la superposición mostrará la siguiente información:

  • Handle - El handle de la fuente OpenAL (único para cada sonido reproducido)
  • Data Handle - El handle de OpenAL buffer (muestra el mismo valor para múltiples sonidos con el mismo handle de origen)
  • Bufferes procesados: cuántos buffers se han reproducido
  • Bufferes en cola: cuántos buffers se han puesto en cola (y todavía están esperando para jugar)
  • Ganancia - El valor de la ganancia para el audio (0 - 65536)
  • Play Offset - la posición de la muestra (posición actual de reproducción) en el buffer actual
  • Barras de Buffers en cola - un pequeño diagrama de caja de la buffers procesada/en cola con el desplazamiento de la reproducción (posición actual) dibujado como una línea azul

Para los grupos de audio sincronizados, también tiene la función audio_sync_group_debug().

NOTA Esta función no funciona en la plataforma de destino HTML5.

Sintaxis:

audio_debug(enable);

ArgumentoTipo Descripción
enable Enable (set to true) o desactivar (false) el audio debugging.

Devuelve:

Ejemplo:

if (debug_mode)
{
    audio_debug(true);
}
else
{
    audio_debug(false);
}

El código anterior switch activará o desactivará la superposición de la depuración de audio dependiendo de si el juego se está ejecutando en modo de depuración o no.
