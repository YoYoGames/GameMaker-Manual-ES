audio_play_sound_at

Con esta función puede reproducir cualquier recurso sonoro en una posición determinada dentro del espacio de audio. Usted proporciona el índice del sonido y luego le asigna una posición dentro del espacio 3D. La posición por defecto del oyente es (0, 0, 0), lo que significa que si el oyente no se ha movido y quiere que el sonido venga de la izquierda (por ejemplo), debe establecer la posición x en un valor negativo (para más información sobre la configuración de la posición del oyente vea audio_listener_position()).

También puedes establecer una distancia de caída (0 hará que el sonido sea silencioso, por defecto es 100) que hará que el sonido se desvanezca a medida que se aleja de la posición del oyente. La forma en que se escuche el desvanecimiento dependerá de la referencia de desvanecimiento (que es la distancia por debajo de la cual el volumen de la fuente normalmente bajaría a la mitad) y del factor de desvanecimiento (que afecta al sonido más allá de la distancia de referencia de desvanecimiento solamente). El factor por defecto es normalmente 1, y el efecto de los diferentes valores de decaimiento dependerá del modelo elegido (para una guía completa de los diferentes modelos de decaimiento y cómo se utilizan estos valores, consulte la función audio_falloff_set_model()).

Los siguientes dos argumentos son para establecer que el sonido sea o no loop y para asignar una prioridad al sonido. Esta prioridad se utiliza para determinar cómo se tratan los sonidos cuando el número de sonidos que se reproducen supera el límite establecido por la función audio_channel_num(). Los sonidos de menor prioridad se detendrán en favor de los de mayor prioridad, y el valor de la prioridad puede ser cualquier número real (el valor real es arbitrario, y puede ser de 0 a 1 o de 0 a 100, ya que GameMaker les dará la misma prioridad). Tenga en cuenta que cuando se trata de la prioridad, cuanto mayor sea el número, mayor será la prioridad, de tal manera que un sonido con prioridad 100 será favorecido sobre un sonido con prioridad 1.

Esta función devolverá un número de índice único para el sonido que se está reproduciendo, el cual puede ser almacenado en una variable para que puedas pausarlo o detenerlo con las funciones apropiadas. Esto significa que si tienes varias instancias del mismo sonido reproduciéndose en un momento dado, puedes apuntar a una sola instancia de ese sonido para tratarla con las funciones de audio.

Sintaxis:

audio_play_sound_at(index, x, y, z, falloff_ref, falloff_max, falloff_factor, loop, priority, [gain], [offset], [pitch], [listener_mask]);

Argumento Tipo Descripción
index Activo sonoro The index of the sound to play.
x Real The x position.
y Real The y position.
z Real The z position.
falloff_ref Real The falloff reference relative to the listener (clamp).
falloff_max Real The maximum falloff distance relative to the listener.
falloff_factor Real The falloff factor (default 1).
loop Booleano Flags the sound as looping or not.
priority Real Set the channel priority for the sound.
gain Real OPTIONAL Valor de la ganancia.
offset Real OPTIONAL El tiempo (en segundos) en el que se establece el punto de inicio. Los valores más largos que la longitud del sonido dado se ignoran.
pitch Real OPTIONAL El multiplicador de paso (por defecto 1).
listener_mask Real OPTIONAL Los datos de la máscara de bits a establecer para el sonido. En el objetivo HTML5 esto no tendrá ningún efecto ya que el objetivo no admite más de un oyente.

Devuelve:

Sound Instance ID

Ejemplo 1:

if global.SFX
{
    audio_play_sound_at(snd_Waterfall, x, y, 0, 100, 300, 1, true, 1);
}

El código anterior comprueba la variable global "SFX" y si devuelve true entonces el sonido indexado en la variable "snd_Waterfall" se pondrá en bucle en su posición room, con una referencia de caída de 100, una distancia de caída de 300, un factor de caída de 1 y una prioridad baja.

Ejemplo 2:

if global.SFX
{
    audio_play_sound_at(snd_Waterfall, x, y, 0, 100, 300, 1, true, 1, 1, 2);
}

El código anterior comprueba la variable global "SFX" y si devuelve true entonces el sonido indexado en la variable "snd_Waterfall" se reproducirá en bucle en su posición room utilizando los ajustes de decaimiento dados y sin cambiar la ganancia. El sonido comienza a reproducirse inmediatamente a partir del segundo 2.
