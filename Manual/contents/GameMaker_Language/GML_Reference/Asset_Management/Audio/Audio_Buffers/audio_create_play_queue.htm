audio_create_play_queue

Esta función prepara un buffer queue para el audio. Se pasa el formato de datos (actualmente sólo se admite buffer_u8 o buffer_s16 ), la frecuencia de muestreo (que puede estar entre 1000hz y 48000hz), y los canales que requiere el audio a partir de una de las constantes listadas a continuación:

Constante Descripción
audio_mono Audio mono (un solo canal).
audio_stereo Audio estéreo (doble canal).
audio_3D Audio 3D (5.1).


Tenga en cuenta que debe liberar cualquier audio queues creado con esta función cuando ya no sea necesario para evitar fugas de memoria utilizando la función audio_free_play_queue().

Esta función activará un Evento Asíncrono de Reproducción de Audio cuando el queue haya llegado al final, y en este evento se creará un mapa DS especial en la variable async_load con los siguientes pares clave/valor:

  • "queue_id" - el queue que ha terminado de reproducir datos
  • "buffer_id" - el buffer que ya no se reproduce desde
  • "queue_shutdown" - se pone a 0 durante la reproducción normal y a 1 cuando se recibe el evento porque audio_free_play_queue() ha sido llamado. Cuando esto se establece en 1, no quiere queue hasta más datos.

Sintaxis:

audio_create_play_queue(queueFormat, queueRate, queueChannels);

Argumento Tipo Descripción
queueFormat Tampón Data Type Constante The format of the buffer data to use (buffer_u8 o buffer_s16).
queueRate Real The sample rate of the data in the buffer queue.
queueChannels Tipo de canal de audio The channels to use from one of the constants listed above.

Devuelve:

Audio Queue ID

Ejemplo:

audio_queue = audio_create_play_queue(buffer_s16, 11052, audio_mono);

El código anterior crea un nuevo audio queue y asigna su ID a una variable para futuras referencias.
