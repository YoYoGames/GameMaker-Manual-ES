audio_play_sound

Con esta función puedes reproducir cualquier recurso sonoro en tu juego. Usted proporciona el índice del sonido y le asigna una prioridad, que luego se utiliza para determinar cómo se tratan los sonidos cuando el número de sonidos que se reproducen supera el límite establecido por la función audio_channel_num(). Los sonidos de menor prioridad se detendrán en favor de los sonidos de mayor prioridad, y el valor de la prioridad puede ser cualquier número real (el valor real es arbitrario, y puede ser de 0 a 1 o de 0 a 100, ya que GameMaker les dará la misma prioridad). Tenga en cuenta que cuando se trata de la prioridad, cuanto mayor sea el número, mayor será la prioridad, de tal manera que un sonido con prioridad 100 será favorecido sobre un sonido con prioridad 1. El tercer argumento es para hacer que el sonido sea loop y si se establece en true hará que el sonido sea loop hasta que se detenga y si se establece en false hará que el sonido se reproduzca una sola vez.

Esta función también devolverá un número de índice único para el sonido que se está reproduciendo, el cual puede ser almacenado en una variable para que puedas pausarlo o detenerlo con las funciones apropiadas. Esto significa que si tienes varias instancias del mismo sonido reproduciéndose en un momento dado, puedes apuntar a una sola instancia de ese sonido para tratarla con las funciones de audio.

Sintaxis:

audio_play_sound(index, priority, loop, [gain], [offset], [pitch], [listener_mask]);

Argumento Tipo Descripción
index Activo sonoro The index of the sound to play.
priority Real Set the channel priority for the sound.
loop Booleano Sets the sound to loop or not.
gain Real OPTIONAL Valor de la ganancia.
offset Real OPTIONAL El tiempo (en segundos) en el que se establece el punto de inicio. Los valores más largos que la longitud del sonido dado se ignoran.
pitch Real OPTIONAL El multiplicador de paso (por defecto 1).
listener_mask Real OPTIONAL Los datos de la máscara de bits a establecer para el sonido. En el objetivo HTML5 esto no tendrá ningún efecto ya que el objetivo no admite más de un oyente.

Devuelve:

Sound Instance ID

Ejemplo 1:

if health <= 0
{
    lives -= 1;
    audio_play_sound(snd_PlayerDead, 10, false);
}

El código anterior comprueba la variable global "salud" y si es menor o igual a 0, eliminará 1 de la variable global "vidas" y reproducirá un sonido.

Ejemplo 2:

if bbox_left > room_width
{
    audio_play_sound(snd_Goodbye, 10, false, 1.1, 0, 2);
}

El código anterior comprueba si la instancia pasa el límite derecho de la room y reproduce un sonido con una ganancia de 1,1 y un tono de 2 (el doble del tono normal).
