Habitaciones

Las salas se crean en el Editor de Salas de GameMaker y es donde todo sucede en su juego. Debes tener al menos un room en cualquier juego para que se ejecute, pero puedes necesitar muchos más dependiendo del tipo de juego y de cómo estructures tu proyecto. Esta sección tiene todas las funciones y variables generales de GML relacionadas con rooms y para obtener información sobre ellas, así como para establecer ciertas propiedades dentro de ellas.

Las siguientes variables y funciones globales pueden utilizarse para obtener información básica sobre cualquier sala:

  • room_first
  • room_last
  • room_next
  • room_previous
  • room
  • room_speed
  • room_height
  • room_width
  • room_persistent

NOTA Las variables enumeradas anteriormente no serán válidas en un script, ya que scripts se ejecuta antes de que se cargue el primer room.

Esto sólo se aplica al código escrito en la raíz de un script, pero no a las funciones creadas dentro de scripts, ya que esas funciones pueden ser ejecutadas en cualquier momento del juego.

También puede utilizar las siguientes funciones para obtener el nombre de la room actual (tal y como se define en el Navegador de Activos), así como para desplazarse entre salas:

  • room_get_name
  • room_goto
  • room_goto_next
  • room_goto_previous
  • room_restart

A continuación, tenemos una serie de funciones que se pueden utilizar para cambiar rooms y su contenido o incluso crear nuevos rooms y añadirlos. Hay que tener en cuenta que estas funciones no deben ejecutarse nunca desde el room que se desea modificar, sino que deben ejecutarse primero desde una instancia en otro room. También hay que tener en cuenta que si se cambia cualquier room asset de esta manera, el cambio es permanente y se mantendrá durante toda la duración del juego, incluso si se llama a la función room_restart() (aunque cerrar y reiniciar el juego restablecerá las propiedades de room ).

  • room_exists
  • room_add
  • room_duplicate
  • room_assign
  • room_instance_add
  • room_instance_clear
  • room_set_height
  • room_set_width
  • room_set_persistent
  • room_set_view_enabled
  • room_set_viewport
  • room_get_viewport
  • room_set_camera
  • room_get_camera

También hay una serie de funciones diferentes relacionadas con la creación y manipulación de los diferentes tipos de elementos que se pueden colocar en las capas en el room. Estas funciones se enumeran en las secciones apropiadas para cada tipo de elemento que se enumeran a continuación:

  • Funciones generales de la capa
  • Capas del mapa de azulejos
  • Capas de fondo
  • Capas de sprites
  • Capas de secuencia
  • Funciones de filtros y efectos

Una cosa importante a tener en cuenta al crear las capas de room es que puedes tener varios tipos de elementos diferentes en una capa. Así, por ejemplo, puedes tener una capa para los enemigos, y en esa capa tener las instancias del enemigo así como ciertos sprite assets que sólo están relacionados con ese enemigo. O podrías tener un elemento de fondo con un elemento de mapa tile para crear la apariencia de un room. Esto no está permitido en el editor room, pero al crear cosas dinámicamente puedes hacerlo, por lo que existen funciones como layer_get_all_elements() existen.
