Capas del mapa de azulejos

El editor de salas de GameMaker le permite añadir tiles en cualquier room utilizando capas. Como su nombre indica, una capa de mapa tile es simplemente una colección de tile set tiles que se consideran todas a la misma profundidad dentro de room, y apilando capas puedes hacer que algunas cosas se dibujen encima o debajo de otras. También puedes controlar ciertos aspectos de las capas desde el código, añadiendo o quitando cosas, o cambiando ciertas propiedades de la capa o lo que contiene en tiempo de ejecución. Tenga en cuenta que en el editor room está limitado a un mapa tile por capa (el Capa de azulejos), pero cuando se trabaja con ellos en código, se pueden tener múltiples mapas tile asignados a una sola capa, teniendo cada mapa su propio y único ID y propiedades, así:

  • El conjunto de baldosas es el recurso que se utiliza para crear las capas de tile y los mapas de tile
  • La capa de azulejos es la que se crea en el editor Room para añadir tile sets a
  • Se llama mapa de azulejos a la colección de tiles que se añaden a una capa, ya sea en el editor room o a través de código, como un único elemento en esa capa

Aparte de las funciones enumeradas anteriormente para establecer los mapas de tile, también hay una serie de funciones que se pueden utilizar para manipular los datos de cada tiles. Un "blob" de datos de tile se compone del índice de tile más unos cuantos bits adicionales para mostrar si el tile ha sido girado, volteado o reflejado. También tiene la posibilidad de añadir su propia máscara de bits a los datos de tile, aunque esto es sólo para usuarios avanzados ya que estará enmascarando bits que están reservados para el índice de tile - esencialmente hay 19 bits reservados para sus índices de tile, pero si sólo utiliza 8 de ellos, entonces le quedan once bits que pueden ser utilizados para crear una máscara personalizada. Esta máscara se puede utilizar y comprobar en el código para crear colisiones personalizadas tile o lo que sea. Sin embargo, normalmente no necesitarás crear tus propios blobs de datos de tile, y en su lugar utilizarás la función tilemap_get() para obtener los datos de tile, luego los manipularías usando las funciones de abajo, y luego los establecerías de nuevo usando tilemap_set().
