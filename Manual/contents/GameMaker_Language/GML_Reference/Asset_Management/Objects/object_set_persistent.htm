object_set_persistent

Con esta función puede establecer la persistencia de un object específico. Un object persistente se "traslada" de room a room y mantiene sus propiedades en todo momento (para más información sobre la persistencia, véase la sección sobre el Editor de objetos). Establecer la persistencia de object significa que todas las instancias de este object que se creen después de haber cambiado la persistencia se crearán persistentes, mientras que las instancias que ya están en el room no se verán afectadas.

Cuando se crea una instancia con persistencia habilitada en un room, se le asignará una capa o una profundidad (dependiendo de la función que se haya utilizado para crear la instancia del object). Cuando se cambie la room, y si la siguiente room no tiene una capa con el mismo nombre o profundidad que la asignada, entonces se creará una nueva capa para la instancia que persiste a través de la rooms. Si usted dio una capa con nombre cuando la instancia fue creada, entonces la nueva capa también se llamará igual que la capa original, mientras que si usted asignó una profundidad a la instancia entonces la nueva capa se llamará "_layer_XXX", donde "XXX" es un valor hexadecimal utilizado para dar a la capa un nombre único. Estas capas se eliminarán de la room en Room Fin. Otra cosa a tener en cuenta sobre las capas y las instancias persistentes es que si has asignado a la instancia una capa con nombre en la creación, y hay otra capa en la siguiente rooms con el mismo nombre, entonces la instancia persistente será asignada a la capa con el mismo nombre sin importar la profundidad de la capa. Por último, si una instancia persistente se mueve a una room con una capa con la misma profundidad en la que se creó la instancia, no se asignará a esta capa, sino que se creará una nueva capa con la misma profundidad (siguiendo la convención de nombres explicada anteriormente).

Tenga en cuenta que ésta no es una función de instancia. Puede establecer la persistencia flag de instancias individuales utilizando la variable persistent y así tener diez instancias marcadas como persistentes aunque el object en cuestión tenga el flag establecido como falso en el IDE, e incluso si cambia el persistente flag del object a verdadero usando esta función, todas las instancias que actualmente están en el room permanecerán como estaban, y sólo las instancias creadas después de llamar a la función comenzarán con el persistente marcado como verdadero.

Sintaxis:

object_set_persistent(index, pers);

Argumento Tipo Descripción
index Activo de objeto The index of the object to change.
pers Booleano Whether the object should be flagged as persistent (true) o no (false).

Devuelve:

N/A

Ejemplo:

if (room == rm_final)
{
    object_set_persistent(obj_Player, false);
}
instance_create_layer(32, 32, "Instances", obj_Player);

El código anterior comprueba en qué room se encuentra actualmente la instancia y si está en el "rm_final" especificado, entonces establece la persistencia de object "obj_Player" en false y luego crea la instancia.
