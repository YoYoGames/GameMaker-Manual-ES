evento_heredado

Esta función llamará al evento actual del padre object de la instancia. Normalmente, cuando una instancia tiene un padre object, automáticamente inherits todos los mismos eventos que el padre, pero si (por ejemplo) su padre object tiene un evento de creación y usted añade uno a su hijo object, todas las instancias del hijo object ejecutarán el nuevo evento de creación que usted ha añadido y no el que está en el padre object. Si necesitas utilizar tanto el evento padre object como el evento hijo object del mismo tipo, debes utilizar esta función ya que ejecutará el evento padre object antes de continuar con el resto del código o acciones que contenga el evento hijo.

Sintaxis:

event_inherited();

Devuelve:

Ejemplo:

event_inherited();
switch (sprite_index)
{
    case spr_Enemy_1: dmg += 2; break;
    case spr_Enemy_4: dmg -= 1; break;
    case spr_Enemy_10: dmg +=10; break;
}

El código anterior llama al evento padre heredado (en el que inicializamos la variable "dmg") y luego pasa a modificar la variable "dmg". Si no hay un padre especificado para la instancia que ejecuta este código, obtendríamos un error de "variable desconocida" ya que dmg no ha sido definida.
