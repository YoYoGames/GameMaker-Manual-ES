Generación de eventos Object

GameMaker está completamente orientado a los eventos, y todas las acciones ocurren como resultado de ciertos eventos que se desencadenan en un orden específico o por una ocurrencia específica dentro del juego. Hay un número de eventos diferentes, cada uno con una tarea o tiempo específico y algunos de ellos están incluso divididos en "sub" eventos separados. Normalmente estos eventos son ejecutados por GameMaker cuando detecta un disparador en el juego, o cada tick del juego (como es el caso de los eventos de paso y dibujo), pero también puedes aplicar un evento a una instancia en el actual room desde dentro de un trozo de código.

Existen las siguientes funciones que se encargan de generar eventos:

  • event_inherited
  • event_perform
  • event_perform_async
  • event_perform_object
  • event_user

También hay una serie de variables de sólo lectura que se pueden utilizar con las funciones mencionadas anteriormente y que son especialmente útiles para la depuración:

  • event_action
  • event_number
  • event_object
  • event_type

Para obtener más información sobre los eventos, consulte la sección Eventos de los objetos.
