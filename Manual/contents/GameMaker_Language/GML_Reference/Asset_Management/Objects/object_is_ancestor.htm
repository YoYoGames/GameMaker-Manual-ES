object_is_ancestor

Esta función se puede utilizar para comprobar un object y ver si es un ancestro o no de otro object. Por lo tanto, ambos argumentos deben ser índices de object y no los ids de instancia, y el primero es siempre el object que se quiere averiguar si es hijo del segundo argumento, que es siempre el object que se quiere comprobar como ancestro (padre).

Sintaxis:

object_is_ancestor(obj, par);

Argumento Tipo Descripción
obj Activo de objeto The object that is being checked as the child.
par Activo de objeto The object that is being checked as the ancestor (parent).

Devuelve:

Boolean

Ejemplo:

if object_is_ancestor(object_index, obj_Enemy)
{
    instance_destroy();
}

El código anterior comprueba si la instancia que ejecuta el código es hija de object "obj_Enemy", y si es así se destruye.
