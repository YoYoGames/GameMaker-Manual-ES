mezcla_de_imágenes

Esta variable controla el "tinte" de la instancia sprite y el valor por defecto es -1 (pero también puede ser c_white). Cualquier otro valor (incluyendo constantes de color internas como c_red, o c_aqua) mezclará el color especificado con el original sprite. Por favor, ten en cuenta que para que los cambios en esta variable sean visibles, la instancia debería no tener ningún evento de dibujo (y así GameMaker dibujará por defecto el sprite) o ser dibujada usando una de las funciones de dibujo extendidas como draw_self() o draw_sprite_ext().

Tenga en cuenta que debe tratar de limitar la mezcla en la plataforma HTML5 (a menos que utilice WebGL), ya que cada sprite mezclado tiene que ser almacenado en caché por separado y, por lo tanto, tener muchos sprites mezclados puede afectar negativamente al rendimiento (también puede establecer el tamaño de cache utilizando la función sprite_set_cache_size()).

Sintaxis:

image_blend;

Devuelve:

Colour Constant

Ejemplo:

image_blend = make_colour_hsv(255, 255, random(255));

El código anterior establecerá el image_blend del sprite definido para la instancia a un color aleatorio.
