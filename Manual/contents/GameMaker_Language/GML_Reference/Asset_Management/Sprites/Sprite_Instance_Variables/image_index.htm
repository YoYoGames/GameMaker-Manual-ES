image_index

Un sprite se compone de una o más sub-imágenes que pueden hacer que el sprite aparezca animado a medida que switch de uno a otro, o pueden ser cambiados en código para dar diferentes "estados", al igual que un botón tiene en las ventanas. Si el sprite está animado, entonces puede obtener el fotograma actual de la animación comprobando la variable image_index, o si quiere cambiar el estado de un sprite estático, puede seleccionar una nueva sub-imagen ajustando esta variable a la sub-imagen deseada del sprite. Por favor, tenga en cuenta que para que los cambios en esta variable sean visibles, la instancia no debe tener ningún evento de dibujo (y así GameMaker dibujará por defecto el sprite) o ser dibujado usando una de las funciones de dibujo como draw_self() o draw_sprite_ext() (suministrando el image_index en el argumento apropiado).

Tenga en cuenta que al establecer esta variable directamente en un fotograma no se activarán los mensajes de difusión que puedan estar presentes en ese fotograma.

También tenga en cuenta que mientras se usa la animación esquelética sprites, todavía puede obtener y establecer los valores de image_index - vea la función skeleton_animation_get_duration() para ver ejemplos de cómo hacerlo.

Sintaxis:

image_index

Devuelve:

(single precision floating point value)

Ejemplo:

if (image_speed > 0)
{
    if (image_index >= image_number - 1)
    {
        instance_destroy();
    }
}

El código anterior comprueba si el sprite está animando, y si lo está entonces comprueba si el image_index actual está en el último fotograma y en ese caso destruye la instancia.
