sprite_add

Con esta función puede añadir una imagen como sprite, cargándola desde una fuente externa donde el archivo de imagen a cargar debe estar siempre en formato *.png, *.gif, *.jpg/jpeg o *.json ( los archivos*json se utilizan para cargar sprites de animación de esqueleto hechos con Spine). La función devuelve el nuevo índice sprite que debe ser utilizado en todo el código posterior que se relacione con el sprite. Si utiliza esta función con HTML5 o está obteniendo una imagen desde una URL, esta función también generará un evento asíncrono de Imagen Cargada. Consulte esta página para obtener más información.

Todas las imágenes que se van a convertir en animadas sprites - excepto *.gif (ver más abajo) - deben tener un formato de "tira" (ver la imagen de abajo) y se dividirá en el número de subimágenes especificadas siguiendo la regla ancho del sprite = ancho de la tira / subimágenes.

Como puede ver en la imagen anterior, el sprite se ha colocado sobre un fondo púrpura oscuro, y esto puede eliminarse estableciendo el argumento "removeback" en true. Esto funciona comprobando el píxel inferior izquierdo del sprite para el color que hay allí y luego utiliza eso como el color a eliminar. Por ejemplo, en la imagen anterior, si tuviéramos el color del píxel inferior izquierdo como verde, todas las partes verdes de sprite se habrían eliminado y el resto del fondo morado se habría ignorado.

Si elige la opción "removeback", puede que también quiera que GameMaker suavice los bordes de sprite estableciendo el argumento "smooth" en true. Todo lo que esto hace es crear un borde semitransparente alrededor de los bordes de la sprite después de que se haya eliminado su fondo.

Por último, también puede especificar el origen x e y para sprite. Este es el punto en el que el sprite se "fija" en la instancia que lo utiliza, y siempre se calcula como relativo a la esquina superior izquierda 0,0 de una subimagen de sprite. Así, por ejemplo, un sprite de 32 x 32 píxeles con estos valores fijados en (16,16) tendrá su origen en el centro.

Si está añadiendo una imagen en formato *.gif, sólo necesita especificar el nombre del archivo y el número de la imagen (que debe ser 1 para el primer fotograma) y puede dejar todos los demás argumentos como 0, teniendo en cuenta que sólo se cargará la primera imagen del GIF. Para *.json sprites, el número de imagen debe ser 1, y los ajustes "removeback" y "smooth" serán ignorados.

Por defecto, todos los nuevos sprites tienen sus cuadros delimitadores calculados automáticamente (el cuadro delimitador exacto dependerá del tamaño y la transparencia del sprite), sin embargo, es posible que desee personalizarlo, en cuyo caso deberá utilizar también la función sprite_collision_mask().

Hay que tener en cuenta que la memoria utilizada al añadir un sprite de esta manera será mayor de lo que se puede esperar. Esto se debe a que GameMaker almacenará el sprite como una página de textura y también se almacenará en la memoria de la GPU, por lo que la memoria total será mayor de lo que cabría esperar para un archivo de imagen del mismo sprite.

NOTA: Cuando cargue un sprite en GameMaker debe recordar eliminarlo de nuevo (con sprite_delete()) cuando ya no lo necesites, de lo contrario existe el riesgo de que se produzca una fuga de memoria que ralentizará y acabará por colapsar tu juego.

NOTA: En iOS, si está cargando un archivo incluido desde una carpeta, es decir, con un path algo así como "Backgrounds/background1.png", NO es necesario incluir la carpeta como parte de path y basta con proporcionar el nombre del archivo. En el resto de las plataformas, debe proporcionar la(s) carpeta(s) como parte de path al archivo.

NOTA: En HTML5, si está cargando un sprite desde un server seguro, es posible que tenga que configurar el tipo de origen cruzado y utilizar un path relativo en lugar de un path absoluto. Consulte http_set_request_crossorigin() para más detalles.

Sintaxis:

sprite_add(fname, imgnumb, removeback, smooth, xorig, yorig);

Argumento Tipo Descripción
fname Cadena The name (a string file path) of the file to add.
imgnum Real Use to indicate the number of sub-images (1 for a single image or for a *.gif).
removeback Booleano Indicates whether to make all pixels with the background colour (left-bottom pixel) transparent.
smooth Booleano Indicates whether to smooth the edges if transparent.
xorig Real Indicate the x position of the origin in the sprite.
yorig Real Indicate the y position of the origin in the sprite.

Devuelve:

Sprite Asset

Ejemplo:

spr = sprite_add("player_5.png", 16, true, true, 0, 0);

El código anterior carga un sprite en el juego y almacena su índice en la variable "spr".
