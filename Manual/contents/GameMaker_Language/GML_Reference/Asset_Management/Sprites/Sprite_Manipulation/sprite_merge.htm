sprite_merge

Esta función fusionará la sprite indexada en el argumento 1 ("ind2") con la indexada en el argumento 0 ("ind1"). Las imágenes en sí NO se fusionan, sino que los índices de las imágenes se fusionan, y las subimágenes de sprite "ind2" se añaden a las de sprite "ind1", es decir, se añaden al final. Tenga en cuenta que si los sprites son de distinto tamaño, los sprites añadidos se estiran para ajustarse al tamaño de la imagen de "ind1".

Este cambio es permanente, y desde el momento en que se utiliza esta función hasta que se cierra el juego o se borra el sprite, el sprite en el que se está fusionando cambiará, sin embargo el sprite con el que se está fusionando seguirá siendo el mismo.

NOTA: Ni la imagen asset a la que se está fusionando, ni la imagen asset a la que se está fusionando, pueden ser uno de los recursos originales del juego. Debe utilizar sprite_duplicate() primero para crear dos nuevos recursos (si está fusionando el juego incluido assets) y luego fusionarlos.

NOTA: Esta función sólo es útil para el mapa de bits sprites y no funcionará con SWF o JSON (Spine) sprites.

Sintaxis:

sprite_merge(ind1, ind2);

ArgumentoTipo Descripción
ind1 The index of the sprite to merge.
ind2 The index of the sprite that ind1 is to be merged with.

Devuelve

Ejemplo:

var tspr;
tspr[0] = spr_add(working_directory + "Sprite Assets\explode1.png", 16, true, true, 0, 0);
tspr[1] = sprite_duplicate(spr_Explosion);
sprite_merge(tspr[0], tspr[1]);
sprite_index = t_spr[0];
sprite_delete(tspr[1]);

El código anterior carga un sprite en una variable local y luego lo fusiona con otro sprite que ha sido duplicado. Finalmente el sprite fusionado se asigna a la instancia sprite_index mientras que el sprite cargado se elimina para liberar la memoria que utiliza. Ten en cuenta que en algún momento después de esta operación, como en los eventos Destroy o Room End, también necesitarás usar sprite_delete() para eliminar el sprite fusionado de la memoria también.
