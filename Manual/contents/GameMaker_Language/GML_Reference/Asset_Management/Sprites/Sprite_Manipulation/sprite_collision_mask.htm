sprite_collision_mask

Con esta función puede establecer las propiedades de la máscara de colisión que debe tener un sprite. Si selecciona como modo de cuadro delimitador el automático (0) o el de imagen completa (1), los valores individuales del cuadro delimitador pueden establecerse en 0. Sin embargo, para una máscara definida por el usuario (2) tendrá que establecer estos valores. Los diferentes valores de los cuadros delimitadores son siempre relativos a la esquina superior izquierda de sprite (independientemente del origen x e y), que se considera la posición (0, 0).

La configuración del tipo de máscara establece la forma general de la propia máscara, pero tenga en cuenta que cualquier cosa que no sea una máscara rectangular requerirá más potencia de procesamiento a la hora de resolver las colisiones, con la consiguiente disminución del rendimiento. En general, sólo deberías utilizar tipos de máscara distintos a la rectangular cuando sea absolutamente necesario.

NOTA: Esta función no permite el tipo de máscara de colisión de rectángulo girado.

El tipo de máscara que se puede establecer será una de las siguientes constantes:

Tipo de caja delimitadora (forma) Constante
Constante Descripción
bboxkind_rectangular Una forma de máscara de colisión rectangular (no giratoria)
bboxkind_ellipse Una forma de máscara de colisión elíptica
bboxkind_diamond Una forma de máscara de colisión en forma de diamante
bboxkind_precise Una máscara de colisión precisa, en la que la máscara se ajustará a los píxeles no transparentes de la página sprite, basándose en el valor de tolerancia dado (véase más adelante))
bboxkind_spine Apart from rectangle, this is the only valid option for Spine sprites. It enables more precise collision checking against the mesh in the sprite.


Por último, la tolerancia se utiliza para definir la precisión de la máscara (cuando se utiliza con una máscara de "imagen completa", esto no tendrá ningún efecto), con una tolerancia de 0 significa que la máscara seguirá cada uno de los píxeles que tengan una transparencia superior a 0, mientras que otros valores desplazarán el perímetro de la máscara de colisión dependiendo de la transparencia de los píxeles.

NOTA: Esta función sólo es útil para el mapa de bits sprites y no funcionará con SWF o JSON (Spine) sprites.

NOTA: Esta función sólo funcionará en los recursos añadidos sprites o duplicados sprites y no directamente en los recursos prefabricados. Puede duplicar sprites utilizando la función sprite_duplicate().

Sintaxis:

sprite_collision_mask(ind, sepmasks, bboxmode, bbleft, bbtop, bbright, bbbottom, kind, tolerance);

Argumento Tipo Descripción
ind Activo Sprite The index of the sprite to set the bounding box of.
sepmasks Booleano Whether to create collision masks for each sub-image of the sprite (true), o una máscara para todos (false).
bboxmode Real What kind of bounding box to use. 0 = automatic, 1 = full image, 2 = user defined.
bbleft Real The maximum left position of the bounding box.
bbtop Real The maximum top position of the bounding box.
bbright Real The maximum right position of the bounding box.
bbbottom Real The maximum bottom position of the bounding box.
kind Tipo de caja delimitadora (forma) Constante The kind of mask, a constant (see the table in the description).
tolerance Real Indicates the tolerance in the transparency value (0=no tolerance, 255=full tolerance).

Devuelve

N/A

Ejemplo:

spr = sprite_add("player_5.png", 16, true, true, 0, 0);
sprite_collision_mask(spr, true, 1, 0, 0, 0, 0, 0, 0);

El código anterior carga un sprite desde una fuente externa y almacena el nuevo índice en la variable "spr". A continuación, el código configura el nuevo sprite para que tenga una máscara de colisión precisa para cada una de sus subimágenes.
