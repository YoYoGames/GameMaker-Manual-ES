sprite_add_from_surface

Esta función funciona exactamente igual que sprite_create_from_surface() sólo que en lugar de crear un nuevo sprite a partir del área de la superficie indexada que seleccione, añade el área definida de la superficie como una nueva subimagen a un sprite previamente creado (esto significa que no puede añadirlo directamente a un sprite del árbol de recursos, sino sólo a uno previamente creado a partir de una superficie, o a uno que haya sido duplicado desde un recurso sprite utilizando sprite_duplicate()).

Hay que tener en cuenta que la memoria utilizada al añadir un sprite de esta manera será mayor de lo que se puede esperar. Esto se debe a que GameMaker almacenará el sprite como una página de textura y también se almacenará en la memoria de la GPU, por lo que la memoria total será mayor de lo que cabría esperar para un archivo de imagen del mismo sprite.

Sintaxis:

sprite_add_from_surface(index, surface, x, y, w, h, removeback, smooth);

Argumento Tipo Descripción
index Activo Sprite The index of the sprite to add the new image to.
surface Identificación de la superficie The index of the surface from which we get the image.
x Real The x position to copy from.
y Real The y position to copy from.
w Real The width of the area to be copied (from the x position).
h Real The height of the area to be copied (from the y position).
removeback Booleano Indicates whether to make all pixels with the background colour (left-bottom pixel) transparent.
smooth Booleano Indicates whether to smooth the edges.

Devuelve:

N/A

Ejemplo:

spr_custom = sprite_create_from_surface(surf, 0, 0, 32, 32, true, true, 16, 16);
var i;
for (i = 1; i < 8; i += 1)
{
    sprite_add_from_surface(spr_custom, surf, i, 0, 32, 32, true, true);
}

El código anterior crea un sprite a partir de la superficie indexada en la variable "surf", asignando su índice a la variable "spr_Custom", y luego utiliza un for loop para desplazarse por la superficie y capturar varias secciones que se añaden en el sprite como subimágenes.
