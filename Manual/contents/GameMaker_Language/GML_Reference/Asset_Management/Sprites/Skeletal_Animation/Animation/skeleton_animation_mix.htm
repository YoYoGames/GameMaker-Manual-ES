skeleton_animation_mix

Puede switch conjuntos de animación fácilmente usando la función skeleton_animation_set() pero esto puede causar un salto o tartamudeo cuando una animación es intercambiada por otra. Para evitar esto, puede establecer el valor de mezcla entre dos conjuntos de animación y el sprite interpolará entre ellos. Normalmente usted querrá hacer esto en el evento Create de la instancia con la animación del esqueleto, ya que sólo necesita establecerse una vez, y GameMaker interpolará todos los cambios posteriores a la sprite utilizando los conjuntos de animación en esa instancia. Tenga en cuenta que el valor de la duración es de 0 a 1, donde un valor de 0,5 tendría una interpolación "mitad y mitad" de un conjunto a otro.

NOTA Spine 4.0 cambia el funcionamiento de la animación rotacional, ya que ya no interpola la rotación utilizando la ruta más corta posible, sino que utiliza los valores absolutos en su keyframes para la interpolación. Esto puede causar un comportamiento inesperado si está actualizando su animación Spine desde una versión anterior, y pueden ser necesarios cambios dentro de su proyecto Spine para que las rotaciones se comporten como es debido.

Sintaxis:

skeleton_animation_mix(animfrom, animto, duration);

Argumento Tipo Descripción
animfrom Cadena The name (a string) of the first animation set to interpolate from.
animto Cadena The name (a string) of the second animation set to interpolate to.
duration Real The duration of the interpolation (from 0 to 1)

Devuelve:

N/A

Ejemplo:

skeleton_animation_set("walk");
skeleton_animation_mix("walk", "jump", 0.2);
skeleton_animation_mix("jump", "walk", 0.4);

El código anterior iría en el evento Create de una instancia con una animación esquelética sprite y establece la duración de la mezcla de animación para interpolar entre los dos conjuntos de animaciones "caminar" y "saltar".
