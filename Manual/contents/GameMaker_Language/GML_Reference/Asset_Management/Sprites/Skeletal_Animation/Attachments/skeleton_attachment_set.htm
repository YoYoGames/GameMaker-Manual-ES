esqueleto_adherido_set

Una animación esquelética sprite puede tener otras imágenes añadidas como archivos adjuntos, y estas imágenes se añaden a una ranura con nombre (el nombre se da cuando se crea la ranura del archivo adjunto en su programa de animación) y se dibujarán junto con la animación del sprite actual. Con esta función se puede establecer un adjunto a una ranura determinada, donde se requiere dar los nombres (como strings) de la ranura y del adjunto. Estos nombres son definidos por el programa de animación utilizado, o (en el caso del adjunto) cuando se llama a skeleton_attachment_create().

Tenga en cuenta que también puede pasar un sprite_index como adjunto, y se utilizará ese sprite, o puede utilizar -1 para eliminar el adjunto de la ranura. Cuando se pasa un índice sprite como argumento, se creará una ranura de adjunto usando el nombre de sprite como string para nombrar la ranura (así que usando spr_sword, por ejemplo, se creará una ranura de adjunto "spr_sword"), y la ranura usará el primer índice de imagen (0) del sprite, así como su desplazamiento de origen x e y, con una escala de (1,1) y una rotación de 0.

Sintaxis:

skeleton_attachment_set(slot, attachment);

Argumento Tipo Descripción
slot Cadena The slot name (a string) to get the attachment of.
attachment Activo decadena o sprite The name (as a string or a sprite_index) of the attachment image.

Devuelve:

String

Ejemplo:

if skeleton_attachment_get("slot_leftHand") == ""
{
    skeleton_attachment_set("slot_leftHand", choose("sword", "spear", "knife"));
}

El código anterior comprobaría el nombre de archivo adjunto actualmente asignado para la ranura llamada "slot_leftHand" y si se devuelve un string vacío, se adjunta un nuevo sprite.
