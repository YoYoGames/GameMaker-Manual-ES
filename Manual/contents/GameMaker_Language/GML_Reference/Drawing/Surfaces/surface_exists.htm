superficie_existe

Esta función es esencial cuando se trabaja con superficies debido a su naturaleza volátil. Las superficies se mantienen siempre en la memoria de la textura, lo que significa que pueden ser destruidas de un momento a otro (por ejemplo, cuando se inicia un salvapantallas en las ventanas, o cuando se minimiza en un dispositivo Android ), por lo que siempre debe comprobar que una superficie existe antes de hacer nada con ella (esto incluye dibujarla en la pantalla). El código de ejemplo que sigue muestra un uso típico de este comando en el evento draw de una instancia para comprobar si existe una superficie y volver a crearla si ha sido eliminada (tenga en cuenta que la superficie habrá sido creada originalmente en el evento create de object).

Sintaxis:

surface_exists(surface_id);

ArgumentoTipo Descripción
surface_id The ID of the surface to check.

Devuelve:

Ejemplo:

if !surface_exists(surf)
{
    surf = surface_create(1024, 1024);
}
else
{
    if view_current == 1
    {
        draw_surface(surf, 0, 0);
    }
}

El código anterior comprobará si existe una superficie indexada en la variable "surf", y si no existe, la recreará. Si existe, entonces comprueba qué vista se está dibujando actualmente y si es la vista[1] dibuja la superficie.
