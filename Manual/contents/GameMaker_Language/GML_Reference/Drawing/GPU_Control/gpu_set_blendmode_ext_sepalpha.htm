gpu_set_blendmode_ext_sepalpha

Cuando GameMaker va a dibujar un píxel hay un color de origen (el color del píxel que vamos a dibujar) y un color de destino (el color que ya está en el píxel que estamos dibujando), por lo que al determinar el nuevo color del píxel, los colores de origen y destino se calculan de acuerdo con el modo de mezcla elegido. Cada componente de los colores se almacena como un valor de punto flotante entre 0 y 1, y el nuevo color se calcula multiplicando cada componente del color de origen por algún factor y multiplicando cada componente del color de destino por algún otro factor y luego sumando los resultados componente por componente.

Esta función le permite establecer las diferentes partes de los componentes que deben ser factorizados juntos para crear un modo de mezcla personalizado, seleccionando diferentes partes de diferentes factores de modo de mezcla para los componentes RGB y el componente Alfa. Así, los argumentos "src" y "dest" utilizarán los 3 primeros valores del modo de fusión (RGB) y los argumentos "alphasrc" y "alphadest" tomarán la última parte del factor de modo de fusión. Por ejemplo, si tiene la fuente RGB como bm_zero, entonces se utilizará (0, 0, 0). Si luego establece la fuente Alfa como bm_one, entonces se utilizará (1), dando un modo de mezcla final de la fuente de (0, 0, 0, 1).

La siguiente tabla muestra los valores RGBA para cada modo de mezcla, donde (Rs, Gs, Bs, As) se consideran los valores de origen y (Rd, Gd, Bd, Ad) son los valores de destino. Las once constantes que se pueden utilizar pueden ser de origen o de destino (o ambas) cuando se utilizan en esta función.

Constante Factor de mezcla (rojo, verde, azul, alfa)
bm_zero (0, 0, 0, 0)
bm_one (1, 1, 1, 1)
bm_src_colour (Rs, Gs, Bs, As)
bm_inv_src_colour (1-Rs, 1-Gs, 1-Bs, 1-As)
bm_src_alpha (As, As, As, As)
bm_inv_src_alpha (1-As, 1-As, 1-As, 1-As)
bm_dest_alpha (Ad, Ad, Ad, Ad)
bm_inv_dest_alpha (1-Ad, 1-Ad, 1-Ad, 1-Ad)
bm_dest_colour (Rd, Gd, Bd, Ad)
bm_inv_dest_colour (1-Rd, 1-Gd, 1-Bd, 1-Ad)
bm_src_alpha_sat (f, f, f, 1) donde f = min(As, 1-Ad)

¡IMPORTANTE! HTML5 sin WebGL activado no podrá mostrar correctamente los siguientes modos de uso:

  • bm_src_colour
  • bm_inv_src_colour
  • bm_dest_colour
  • bm_inv_dest_colour
  • bm_src_alpha_sat

Tenga en cuenta que puede suministrar cuatro argumentos individuales a esta función o puede suministrar una matriz de argumentos (como la devuelta por la función gpu_get_blendmode_ext_sepalpha() por ejemplo). Si se suministra una matriz, ésta debe tener los siguientes cuatro elementos:

  • [0] = Modo de fusión de la fuente menos el componente alfa (por defecto es bm_src_alpha (As, As, As))
  • [1] = Modo de fusión de destino menos el componente alfa (por defecto es bm_inv_src_alpha (1-As, 1-As, 1-As))
  • [2] = Componente alfa de la fuente (por defecto es bm_src_alpha (As))
  • [3] = Componente alfa de destino (por defecto es bm_inv_src_alpha (1-As))

Para ayudarte a sacar el máximo provecho de los modos de fusión y entender cómo funcionan y cómo afectan a la imagen final que se dibuja en la pantalla, te recomendamos que leas la siguiente guía:

  • Guía de uso de Blendmodes

Sintaxis:

gpu_set_blendmode(src, dest, alphasrc, alphadest);

Argumento Tipo Descripción
src Factor de mezcla constante Source blend mode factor for RGB (see constants above).
dest Factor de mezcla constante Destination blend mode factor for RGB (see constants above)
alphasrc Factor de mezcla constante Source blend mode factor Alpha (see constants above).
alphadest Factor de mezcla constante Destination blend mode factor Alpha (see constants above)

Devuelve:

N/A

Ejemplo:

var bm = gpu_get_blendmode_ext_sepalpha();
bm[0] = bm_src_alpha;
bm[1] = bm_inv_src_alpha;
gpu_set_blendmode_ext_sepalpha(bm);

El código anterior crea un array local y obtiene el modo de fusión actual extendido con valores alfa separados. Este array es entonces manipulado y devuelto para establecer el modo de mezcla.
