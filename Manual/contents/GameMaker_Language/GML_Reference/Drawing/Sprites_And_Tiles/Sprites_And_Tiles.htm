Sprites y baldosas

Aunque puedes configurar los fondos en el editor room y puedes asignar sprites a las instancias, a menudo esto no es lo suficientemente flexible para el juego que estás haciendo. Por ejemplo, si tienes una instancia que tiene que dibujar un menú, puede que quieras mostrar varios botones sprites a la vez, o si tienes un juego con fondos parallax de desplazamiento necesitarás poder configurarlos a través de código en una instancia de controlador de algún tipo. En estas situaciones deberías usar las siguientes funciones que pueden dibujar sprite y el fondo assets en cualquier parte de room con algunas de las funciones que también te permiten manipular la forma, el tamaño, el color y el alfa de sprite para lograr efectos especiales.

Usted puede asignar un sprite a una instancia y por defecto GameMaker lo dibujará con las propiedades dadas por las diversas variables del sprite, sin embargo si usted agrega cualquier cosa en el evento de dibujo, usted está "tomando el control" y debe decirle a GameMaker exactamente qué dibujar. Esto significa que si debes usar las siguientes funciones para definir exactamente qué es lo que la instancia debe dibujar y en qué lugar de la room debe hacerlo:

  • draw_self
  • draw_sprite
  • draw_sprite_ext
  • draw_sprite_general
  • draw_sprite_part
  • draw_sprite_part_ext
  • draw_sprite_stretched
  • draw_sprite_stretched_ext
  • draw_sprite_pos
  • draw_sprite_tiled
  • draw_sprite_tiled_ext

Si está utilizando el formato SWF sprites, también existen opciones adicionales para controlar el antialiasing utilizado al dibujar (consulte Importar Sprites sin mapa de bits para obtener más información sobre este tipo de sprites). Tenga en cuenta que estas funciones trabajan a nivel global y establecerán el AA en todos los SWF vectoriales sprites dibujados después de que hayan sido llamados.

  • draw_enable_swf_aa
  • draw_set_swf_aa_level
  • draw_get_swf_aa_level

Aparte de las funciones enumeradas anteriormente, también hay dos funciones especiales diseñadas para su uso con animaciones esqueléticas (ver Importar Sprites sin mapa de bits para más información sobre este tipo de sprites, y la sección Animaciones Esqueléticas para funciones específicas relacionadas con este tipo de sprites):

  • draw_skeleton
  • draw_skeleton_instance
  • draw_skeleton_collision
  • draw_skeleton_time

Los mosaicos se dibujan automáticamente en función de los datos que contienen, el tile set utilizado y las propiedades del mapa tile. Sin embargo, puede dibujar mapas individuales de tiles o tile utilizando las siguientes funciones:

  • draw_tile
  • draw_tilemap
