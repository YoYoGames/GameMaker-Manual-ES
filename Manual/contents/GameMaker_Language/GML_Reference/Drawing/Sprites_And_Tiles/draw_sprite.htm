draw_sprite

Esta función dibuja la sprite y la subimagen dadas en una posición dentro del juego room. Para el sprite puedes usar la variable de instancia sprite_index para obtener el sprite actual que está asignado a la instancia que ejecuta el código, o puedes utilizar cualquier otro sprite asset. Lo mismo ocurre con la sub-imagen, ya que también se puede establecer en la variable de instancia image_index que establecerá la sub-imagen a la seleccionada para la instancia actual sprite (tenga en cuenta que puede dibujar un sprite diferente y seguir utilizando el valor de la sub-imagen para la instancia actual), o puede utilizar cualquier otro valor para esto para dibujar una sub-imagen específica del sprite elegido. Si el valor es mayor que el número de sub-imágenes, entonces GameMaker automáticamente loop el número para seleccionar la imagen correspondiente (por ejemplo, si el sprite que se está dibujando tiene 5 sub-imágenes numeradas del 0 al 4 y establecemos el valor del índice en 7, entonces la función dibujará la sub-imagen 3, numerada 0). Por último, la posición x e y es la posición dentro de room en la que se dibujará la sprite, y está centrada en el desplazamiento x e y de sprite.

NOTA Esta función puede no funcionar como se espera cuando se utiliza la animación del esqueleto sprites, y puede encontrar que la función sólo dibuja el primer cuadro de la pose por defecto. Deberías utilizar las funciones de draw_skeleton_* en su lugar.

Sintaxis:

draw_sprite(sprite, subimg, x, y);

Argumento Tipo Descripción
sprite Activo Sprite The index of the sprite to draw.
subimg Real The sub-image (frame) of the sprite to draw (image_index or -1 correlate to the current frame of animation in the object).
x Real The x coordinate of where to draw the sprite.
y Real The y coordinate of where to draw the sprite.

Devuelve:

N/A

Ejemplo:

draw_sprite(sprite_index, image_index, x, y);
draw_sprite(spr_Halo, 0, x, y-32);

Esto dibujará las instancias asignadas sprite (sprite_index) con la sub-imagen actual en la posición x y y de la instancia dentro del room. Luego dibujará el primer cuadro del sprite indexado por "spr_Halo" en la misma posición x y y pero 32 pixeles 'arriba'.
