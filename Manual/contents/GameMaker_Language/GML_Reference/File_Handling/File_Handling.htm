Manejo de archivos

En algún momento, mientras creas tu juego, será necesario almacenar información externa y poder recuperarla. Por ejemplo, es posible que desee almacenar los datos de acceso, o las puntuaciones máximas, o tal vez la progresión del jugador a través de sus niveles. Cualquiera que sea la razón, necesitará saber cómo utilizar las diferentes funciones de archivo que están disponibles cuando se trabaja con GameMaker.

Es muy importante tener en cuenta que GameMaker es Sandboxed por defecto, lo que significa que hay una serie de límites y complicaciones menores cuando se utilizan las funciones del sistema de archivos que están disponibles, y vale la pena señalar también que el sistema de archivos funcionará de forma ligeramente diferente dependiendo de la plataforma a la que se dirige. Para más información sobre esto, por favor vea la siguiente sección:

  • El sistema de archivos

Puedes desactivar el sandboxing en los objetivos de escritorio (Windows, macOS y Ubuntu (Linux)) desde las Opciones de Juego para la plataforma elegida, lo que significa que puedes guardar y cargar desde cualquier lugar, dentro de los límites de los permisos del sistema operativo.

Si Steam te ha concedido permiso para utilizar su API dedicada con tu juego, encontrarás que también hay un nuevo conjunto de funciones de archivo disponibles para ti. Éstas se explican en la sección de la API de Steam de este manual.

Por último, también hay que tener en cuenta que hay algunas diferencias claras entre ciertas plataformas de destino que se tendrían en cuenta al utilizar las funciones de archivo. Estas son:

  • iOS: En iOS, si está cargando un archivo incluido desde una carpeta, es decir, con un path algo así como "Backgrounds/background1.png", NO necesita incluir la carpeta como parte del path y basta con suministrar el nombre del archivo. En todas las demás plataformas, debe proporcionar la(s) carpeta(s) como parte de path al archivo.
  • macOS y Ubuntu (Linux): En macOS y Ubuntu, los nombres de archivo y paths distinguen entre mayúsculas y minúsculas en estas plataformas, por lo que, para evitar posibles problemas, GameMaker hará que todos los nombres/rutas utilicen automáticamente las minúsculas por defecto.
  • HTML5: En HTML5, guardar y cargar desde y hacia el almacenamiento local funcionará como se explica dentro de las diferentes páginas de funciones, sin embargo, si está cargando un archivo desde un server, sólo debe utilizar la función buffer_load_async() ya que la carga de forma sincrónica ha sido desaprobada en la mayoría de los navegadores y eventualmente será obsoleta. También tenga en cuenta que no puede crear o destruir directorios en el almacenamiento local del navegador.

  • game_load
  • game_load_buffer
  • game_save
  • game_save_buffer
