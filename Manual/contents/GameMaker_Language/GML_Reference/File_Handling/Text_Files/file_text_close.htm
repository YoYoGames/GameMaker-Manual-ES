file_text_close

Una vez que hayas terminado de trabajar con un archivo determinado (ya sea leyendo de él o escribiendo en él), debes cerrar el archivo de nuevo, o de lo contrario te arriesgas a perder la información que contiene. Esto también evita las fugas de memoria y asegura que nunca se sobrepase el límite de archivos al tener más de 32 archivos abiertos.

Sintaxis:

file_text_close(fileid);

ArgumentoTipo Descripción
fileid The id of the file to close.

Devuelve:

Ejemplo:

var file = file_text_open_write(working_directory + "Game_Data.txt");
while (!file_text_eof(file))
{
    file_text_readln(file);
}
file_text_write_string(file, level_data);
file_text_close(file);

El código anterior abre un archivo para escribir y luego loops a través de las líneas de texto ya escritas en el archivo hasta que llega al final. En este punto escribe un string y luego cierra el archivo de nuevo.
