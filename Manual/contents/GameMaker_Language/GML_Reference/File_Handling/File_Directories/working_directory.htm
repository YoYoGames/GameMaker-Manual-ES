directorio_de_trabajo

Working_directory puede en realidad devolver dos valores diferentes dependiendo de para qué se utilice.

Si estás escribiendo un archivo en el disco, working_directory apunta al área de almacenamiento local que está reservada para tu juego en el dispositivo de destino (en Windows esto es "%LOCALAPPDATA%/gamedir/" donde el "gamedir" es el directorio con el nombre de tu juego).

Sin embargo, cuando se lee del disco, working_directory puede ser el almacenamiento local (game_save_id) o el área donde se almacenan los Archivos Incluidos.

NOTA La variable working_directory devolverá el path incluyendo la barra invertida final.

Por ejemplo, si usted tiene un archivo *.txt por defecto incluido con su juego y lo lee en la memoria usando working_directory, entonces ese archivo será tomado del área donde se almacenan los Archivos Incluidos. Si luego elige escribir esa información en un archivo de texto usando working_directory, esto escribirá el archivo en el almacenamiento local de la plataforma de destino, desde donde se leerá la próxima vez a menos que le indique a GameMaker que borre el archivo, en cuyo caso working_directory volverá a apuntar al área donde se almacenan los Archivos Incluidos.

Cabe señalar que , en general, no es necesario utilizarlo y basta con especificar el propio nombre del archivo (más cualquier información adicional de path ), que es suficiente. Por ejemplo, esto:

file_text_open_read(working_directory + "my_file.txt");

sería mejor hacerlo así:

file_text_open_read("my_file.txt");

ADVERTENCIA ¡Esta función puede no funcionar como usted espera debido a que GameMaker está en una caja de arena! Por favor, vea la sección sobre el Sistema de Archivos para más información.

Sintaxis:

working_directory

Devuelve:

String

Ejemplo:

ini_open(working_directory + "temp_ini.ini");

Esto abrirá un archivo ini desde el directorio de trabajo del juego (creándolo si no existe ya). Esto podría ser el almacenamiento local, o el área donde están los archivos incluidos, dependiendo de si el archivo que se busca existe en cualquiera de los dos lugares.
