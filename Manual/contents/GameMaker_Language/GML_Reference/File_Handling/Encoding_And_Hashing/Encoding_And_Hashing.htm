Codificación y Hashing

Cuando se trata de archivos externos, siempre existe la posibilidad de que el usuario final pueda abrir y cambiar la información que contienen, y así modificar tu juego. Esto puede dar lugar a elementos de juego rotos o a puntuaciones fraudulentas en línea (por ejemplo), por lo que GameMaker le proporciona algunas funciones básicas de codificación, así como funciones para realizar comprobaciones de hashing en strings y archivos para asegurarse de que han mantenido su integridad antes de ser utilizados. También se suministran algunas funciones para codificar y descodificar el formato JSON strings.

NOTA La codificación NO es una encriptación. Una codificación base64 hace que el archivo sea ilegible a simple vista y requerirá un esfuerzo por parte del usuario para descifrarlo, pero no es seguro frente a la piratería. Se recomienda mezclar esas funciones con su propia codificación (hay muchas formas de codificación y las funciones de script para GameMaker están disponibles en Internet).

  • json_encode
  • json_decode
  • json_stringify
  • json_parse
  • base64_encode
  • base64_decode
  • md5_string_utf8
  • md5_string_unicode
  • md5_file
  • sha1_string_utf8
  • sha1_string_unicode
  • sha1_file
  • zip_unzip
  • load_csv

Cabe destacar que también hay una serie de funciones relacionadas con la codificación y el hashing de los Buffers, que puedes encontrar en los siguientes enlaces:

  • buffer_md5
  • buffer_sha1
  • buffer_crc32
  • buffer_base64_encode
  • buffer_base64_decode
  • buffer_base64_decode_ext
