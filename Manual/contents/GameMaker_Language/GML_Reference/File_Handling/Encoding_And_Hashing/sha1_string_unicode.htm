sha1_string_unicode

En criptografía, SHA-1 es una función hash criptográfica diseñada por la Agencia de Seguridad Nacional de los Estados Unidos y se emplea en varias aplicaciones y protocolos ampliamente utilizados, como el popular Git, donde se utiliza para comprobar los cambios en los archivos. Esta función toma una entrada unicode string y devuelve un compendio de mensajes de 160 bits en formato ASCII, por ejemplo:

sha1 input string = "The quick brown fox jumps over the lazy dog"
sha1 output hash = "2fd4e1c6 7a2d28fc ed849ee1 bb76e739 1b93eb12"

De este modo, se puede generar una clave segura que se puede almacenar y utilizar para comprobar la integridad de la información que se envía a (o se recibe de) un server externo (por ejemplo).

NOTA: Hay dos formatos para la codificación sha1, UTF-8 y unicode. Ambos se proporcionan para facilitar la comunicación con diferentes configuraciones de server, pero el más común de usar es unicode.

NOTA: SHA-1 no es completamente seguro y puede romperse. Consulte esta página para obtener más información.

Sintaxis:

sha1_string_unicode(string)

ArgumentoTipo Descripción
string The string to hash.

Devuelve:

Ejemplo:

var hash, str;
str = base64_encode(game_data);
hash = sha1_string_unicode(str);
http_get("http://www.MacSweeneyGames.com/CatchTheHaggis/gamedata?hash=" + hash); http_get("http://www.MacSweeneyGames.com/CatchTheHaggis/gamedata?data=" + str);

El código anterior codificará en base64 un string y luego generará un hash sha1. Finalmente, tanto el hash como el string codificado se envían a un server.
