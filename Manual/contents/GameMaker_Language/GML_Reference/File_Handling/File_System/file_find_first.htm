file_find_first

Esta función devolverá el nombre del primer archivo que satisfaga la máscara y los atributos. Si no existe tal archivo, se devuelve un string vacío.

La máscara puede contener un path con comodines, por ejemplo C:\temp\*.doc.

Los atributos dan los archivos adicionales que quieres ver, así que los archivos normales siempre se devuelven cuando satisfacen la máscara. Puede sumar las siguientes constantes para ver el tipo de archivos que desea (usando |). Si no desea añadir ningún atributo, utilice 0.

Constante de atributos del archivo
Constante Descripción
fa_readonly Archivos de sólo lectura
fa_hidden Archivos ocultos
fa_sysfile Archivos del sistema
fa_volumeid Archivos de volumen-identificación
fa_directory Directorios
fa_archive Ficheros archivados

Los atributos sólo pueden utilizarse en Windows. Para todas las demás plataformas, utilice 0.

ADVERTENCIA Esta función puede no funcionar como usted espera debido a que GameMaker está en una zona de seguridad. Por favor, vea la sección sobre el Sistema de Archivos para más información.

NOTA Esta función no funcionará en absoluto en el objetivo HTML5.

Sintaxis:

file_find_first(mask, attr);

Argumento Tipo Descripción
mask Cadena The mask to use for searching.
attr Constante de atributos del archivo(s) The specific file attribute to look for.

Devuelve:

String

Ejemplo:

if directory_exists("\User Content")
{
    fileA = file_find_first("/User Content/*.doc", fa_readonly);
    fileB = file_find_next();
    fileC = file_find_next();
    file_find_close();
}

Este código comprueba si el directorio especificado existe y, si lo hace, va allí y devuelve el primer archivo de "sólo lectura" .doc encontrado. Luego busca dos archivos más y cierra el buscador de archivos.

Puede buscar cualquier número de archivos utilizando un while bucle:

var files = [];
var file_name = file_find_first("/User Content/*.doc", fa_readonly);

while (file_name != "")
{
    array_push(files, file_name);

    file_name = file_find_next();
}

file_find_close();

El código anterior crea una matriz vacía para almacenar todos los nombres de archivos encontrados, y comienza a buscar archivos de sólo lectura .doc. Si ese nombre de archivo no es un string vacío, lo empuja a la matriz files, y luego intenta encontrar el siguiente archivo. El loop se repetirá hasta que se encuentre un string vacío (lo que significa que no hay más archivos que coincidan), que es cuando termina el loop y cierra el buscador de archivos al final.
