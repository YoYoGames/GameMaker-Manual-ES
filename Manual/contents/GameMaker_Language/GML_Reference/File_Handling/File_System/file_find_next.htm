file_find_next

Esta función devuelve el nombre del siguiente archivo que satisface la máscara dada previamente y los atributos (definidos por file_find_first()). Si no existe tal archivo, se devuelve el string vacío.

ADVERTENCIA Esta función puede no funcionar como usted espera debido a que GameMaker está en una zona de seguridad. Por favor, vea la sección sobre el Sistema de Archivos para más información.

NOTA Esta función funciona en todas las plataformas de destino C++ (Windows, Mac, iOS, Android), PERO el filtro flags sólo funciona en Windows.

Sintaxis:

file_find_next();

Devuelve:

Ejemplo:

if directory_exists("\User Content")
{
    fileA = file_find_first("/User Content/*.doc", fa_readonly);
    fileB = file_find_next();
    fileC = file_find_next();
    file_find_close();
}

Este código comprueba si el directorio especificado existe y, si lo hace, va allí y devuelve el primer archivo de "sólo lectura" .doc encontrado. Luego busca dos archivos más y cierra el buscador de archivos.

Puede buscar cualquier número de archivos utilizando un while bucle:

var files = [];
var file_name = file_find_first("/User Content/*.doc", fa_readonly);

while (file_name != "")
{
    array_push(files, file_name);

    file_name = file_find_next();
}

file_find_close();

El código anterior crea una matriz vacía para almacenar todos los nombres de archivos encontrados, y comienza a buscar archivos de sólo lectura .doc. Si ese nombre de archivo no es un string vacío, lo empuja a la matriz files, y luego intenta encontrar el siguiente archivo. El loop se repetirá hasta que se encuentre un string vacío (lo que significa que no hay más archivos que coincidan), que es cuando termina el loop y cierra el buscador de archivos al final.
