get_save_filename_ext

Esta función abre un diálogo y pide al jugador un nombre de archivo para guardar con el filtro dado. El filtro tiene la forma "nombre1|máscara1|nombre2|máscara2|...", donde la máscara puede contener las diferentes opciones con un punto y coma entre ellas y también se puede utilizar un "*" para significar cualquier string. Por ejemplo: "bitmaps|*.bmp;*.wmf", pediría al usuario que seleccionara el nombre de un archivo de mapa de bits del sistema, pero sólo aquellos que tengan los dos extensions especificados (bmp y png). ¡esta función no guarda el archivo en sí! Sólo devuelve un string con el nombre completo y path del archivo. Si el usuario pulsa "Cancelar" se devuelve un string vacío. También puede añadir un directorio desde el que iniciar la búsqueda, y una leyenda que se mostrará en la parte superior de la ventana en lugar de la predeterminada.

Una cosa importante a tener en cuenta cuando se utiliza esta función es que le otorga ciertos permisos para ese archivo, durante la duración de su juego. Así, una vez que obtengas el archivo path del usuario, podrás acceder a él una y otra vez sin tener que pedirlo, y el permiso se aplica a la lectura y a la escritura por igual.

NOTA: Esta función sólo es válida en los objetivos Windows y macOS.

Sintaxis:

get_save_filename_ext(filter, fname, directory, caption);

ArgumentoTipo Descripción
filter The file type filter and/or name.
fname The suggested filename to use.
directory The directory to start the search from.
caption The caption for the window.

Devuelve:

Ejemplo:

var file;
file = get_save_filename_ext("screenshot|*.png", "", working_directory, "Save a level here");
if (file != "")
{
    screen_save(file);
}

El código anterior pedirá al usuario que seleccione un nombre de archivo de imagen para guardarlo en el directorio de trabajo y, a continuación, comprobará el nombre de archivo devuelto y path. Si no es un string vacío, tomará una captura de pantalla y la guardará utilizando el nombre dado.
