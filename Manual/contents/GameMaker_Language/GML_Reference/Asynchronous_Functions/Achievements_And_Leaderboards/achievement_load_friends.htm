achievement_load_friends

Esta función enviará una solicitud a server para obtener información sobre todos los amigos de los usuarios conectados y activará una llamada de retorno Social Asynchronous Event que contiene el mapa async_load con los pares clave/valor relevantes. La clave de identificación de este mapa DS se utiliza para identificar el callback correcto (puede haber puede haber más de una función de activación para cualquier evento asíncrono), y se emparejará con la constante achievement_friends_info, así como con una serie de otros pares clave/valor para cada amigo. El contenido exacto del mapa es se muestra a continuación:

NOTA: Esta función es sólo para iOS.

  • "id" - Para esta función debe ser achievement_friends_info
  • "FriendN" - El nombre del amigo, donde "N" es un valor entero correspondiente a su posición dentro de la lista de amigos.
  • "FriendidN" - El id de usuario único del amigo, "N".

Sintaxis:

achievement_load_friends()

Devuelve:

N/A

Ejemplo ampliado:

El siguiente código se llamaría probablemente después de que el jugador haya entrado en su cuenta de juego usando achievement_login() para obtener una lista de todos los amigos de ese usuario:

achievement_friends_info();

Esto enviará una solicitud de información sobre los amigos de los usuarios y generará una devolución de llamada asíncrona con el mapa especial async_load DS que contiene los siguientes datos:

var ident = ds_map_find_value(async_load, "id");
if ident == achievement_friends_info
    {
    var numfriends = ds_map_find_value(async_load, "numfriends");
    global.numfriends = numfriends;
    for(var i=0; i < numfriends; i++;)
        {
        global.friendname[i] = ds_map_find_value(async_load, "Friend" + string(i));
        global.friendid[i] = ds_map_find_value(async_load, "Friendid" + string(i));
        achievement_get_pic(global.friendid[i]);
        }
    }

El código anterior comprueba el mapa DS devuelto en el evento social asíncrono y si su "id" coincide con la constante que se está comprobando, entonces loops a través del mapa almacenando todos los diferentes valores en una serie de arrays y solicita una imagen para cada una de las entradas del mapa (ver achievement_get_pic()).
