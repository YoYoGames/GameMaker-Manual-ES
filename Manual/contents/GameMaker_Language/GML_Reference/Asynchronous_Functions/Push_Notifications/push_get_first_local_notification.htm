push_get_first_local_notification

Esta función rellenará un mapa DS prefabricado con una serie de pares clave/valor para la primera notificación push local en el queue que se mostrará. Devuelve -1 si no hay notificaciones en cola, o un valor real en caso contrario que representa el ID de la notificación. Este ID puede utilizarse para cancelar la notificación mediante la función push_cancel_local_notification().

NOTA: Esta función está limitada a los módulos de destino iOS y Android.

El mapa DS contendrá los siguientes pares clave/valor:

  • "title" - El título de la notificación (sólo estará disponible en la plataforma Android, ya que iOS no almacena la información del título)
  • "mensaje" - El texto del cuerpo del mensaje
  • "data" - La cadena del paquete de datos

Sintaxis:

push_get_first_local_notification(map);

ArgumentoTipo Descripción
map The DS Map Identificación a utilizar.

Devuelve:

Push Notification ID or -1

Ejemplo :

var map = ds_map_create();
var ntf = push_get_first_local_notification(map);
while(ntf >= 0)
{
    var data = ds_map_find_value(map, "data");
    if data == "Daily_Reward"
    {
        push_cancel_local_notification(ntf);
    }
    ntf = push_get_next_local_notification(map);
}
ds_map_destroy(map);

El código anterior crea un DS Map y, a continuación, solicita datos para la primera notificación disponible en el queue. Si existe una notificación, el mapa se rellena y el código comprueba entonces la clave "data" para un string específico. Si se encuentra el string, se cancela la notificación y se pasa a comprobar la siguiente notificación hasta que se hayan comprobado todas en el queue.
