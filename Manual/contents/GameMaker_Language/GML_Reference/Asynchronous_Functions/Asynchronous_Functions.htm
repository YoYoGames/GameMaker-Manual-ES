Funciones asíncronas

En el contexto de GameMaker, una función asíncrona es aquella que envía a una web server, o pide la entrada del usuario, o incluso transmite datos, mientras deja que GameMaker siga ejecutándose sin bloquear el juego que se está ejecutando. Cuando se recibe el callback de información se dispara un Evento As íncrono especial que, a su vez, permite que cualquier instancia con un Evento Asíncrono asignado pueda ejecutar otras funciones (aunque no tiene por qué ser así). Así que, básicamente, es una forma de comunicarse con algún recurso externo (server, usuario o dispositivo) sin que el juego o el proyecto se detenga mientras se espera una respuesta.

¿Por qué es esto importante? Bueno, porque permite que GameMaker siga funcionando mientras envía o recibe datos, y esto significa que puedes hacer muchas cosas sin bloquear el juego loop como:

  • Transmitir los datos del dispositivo al juego sin que el jugador tenga que esperar a que se cargue.
  • tener eventos de devolución de llamada para hacer cosas sólo cuando se recibe la información correcta
  • comunicar e intercambiar datos con un servidor web

Esto hace que todo funcione de una manera mucho más fluida y discreta. Hay que tener en cuenta que los Eventos Asíncronos se dispararán para todas las instancias que los tengan, por lo que puedes usar una función asíncrona en una instancia, y tener el Evento Asíncrono que se ocupa de la respuesta en otra, o incluso varias más.
