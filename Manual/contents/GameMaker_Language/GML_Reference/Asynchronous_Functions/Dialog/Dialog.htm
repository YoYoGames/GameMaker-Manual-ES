Diálogo

A la mayoría de los ordenadores y dispositivos móviles modernos no les gusta que un juego se detenga mientras se muestra una ventana de diálogo. Tienden a interpretar esto como que el corredor del juego se ha detenido, y pueden considerarlo un error y cerrar la aplicación prematuramente. Para evitar esto GameMaker tiene funciones de diálogo asíncronas que le permiten solicitar ciertas formas de retroalimentación o entrada del usuario, pero sin bloquear el corredor de juego en ningún momento, lo que significa que - cuando se llama - el juego se ejecutará con normalidad en el fondo mientras el usuario responde a la ventana de diálogo mostrada. Una vez que el usuario haya respondido, se llamará a un Evento Asíncrono de Diálogo para que puedas tratar los resultados.

NOTA: Con la excepción de shop_leave_rating()estas funciones son sólo para uso de depuración y pruebasy no deben ser utilizadas en juegos lanzados. Para ello, debes crear una interfaz de usuario personalizada para recibir las entradas de los jugadores, de modo que tengas un control total sobre el aspecto y el comportamiento de los diálogos.

A continuación se muestra una lista de todas las funciones de diálogo asíncronas disponibles:

  • get_login_async
  • get_string_async
  • get_integer_async
  • show_message_async
  • show_question_async
  • shop_leave_rating
