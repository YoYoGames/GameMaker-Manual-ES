get_string_async

Esta función abre una ventana y muestra un mensaje, así como un espacio para que el usuario introduzca un string (que contendrá el string suministrado por defecto para empezar). Esta es una función asíncrona, y como tal GameMaker no bloquea el dispositivo en el que se está ejecutando mientras espera una respuesta, sino que sigue ejecutando los eventos de forma normal. Una vez que el usuario ha escrito su string y ha pulsado el botón "Okay", se dispara un evento asíncrono de Diálogo que, sólo durante la duración de ese evento, tendrá un mapa DS almacenado en la variable async_load.

Este mapa contendrá las tres claves, "id", "status" y "result". "id" es el valor que devolvió la función cuando fue llamada, el "status" será true para el botón "Okay" que fue presionado, o false si el mensaje fue cancelado (cuando sea aplicable ya que no todas las plataformas de destino permiten cancelar el mensaje). Por último, "resultado" devolverá el string que el usuario introdujo (o un string "" vacío si no se suministró ninguno).

NOTA Esta función es sólo para uso de depuración y pruebasy no debe ser utilizada en juegos lanzados. Para ello, debe crear una interfaz de usuario personalizada para recibir las entradas de los jugadores, de modo que tenga un control completo sobre el aspecto y el comportamiento de los diálogos.

Sintaxis:

get_string_async(string, default);

Argumento Tipo Descripción
string Cadena The message to show in the dialog.
default Cadena The default string.

Devuelve:

Async Request ID

Ejemplo ampliado:

El evento de pulsación del botón izquierdo del ratón (por ejemplo) del object que está mostrando el mensaje tendría el siguiente código:

msg = get_string_async("What's your name?","Anon");

Lo anterior mostrará un mensaje solicitando que el usuario ingrese un string y presione "Okay". El id de la función se almacena en la variable "msg" y se utilizará en el evento asíncrono Dialogs como se muestra a continuación:

var i_d = ds_map_find_value(async_load, "id");
if i_d == msg
{
    if ds_map_find_value(async_load, "status")
    {
        if ds_map_find_value(async_load, "result") != ""
        {
            global.Name = ds_map_find_value(async_load, "result");
        }
    }
}

El código anterior comprueba la clave "id" del DS Map devuelto con el valor almacenado en la variable "msg". Si son iguales, entonces comprueba si se ha pulsado "Okay" (en lugar de cerrar/cancelar la ventana) y si devuelve true entonces comprueba el "resultado" de un string para asegurarse de que no se ha devuelto ningún strings vacío antes de establecer una variable global.
