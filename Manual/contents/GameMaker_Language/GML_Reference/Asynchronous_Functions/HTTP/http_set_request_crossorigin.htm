http_set_request_crossorigin

Con esta función se puede establecer el tipo de origen cruzado a utilizar cuando se cargan imágenes desde un archivo. La función es exclusiva para la plataforma HTML5 y sólo es útil cuando se carga sprites desde un archivo utilizando la función sprite_add(). Tenga en cuenta que cuando se establece como "use-credentials" ya no es necesario dar un path absoluto al sprite que se está cargando, sino que se daría un path relativo (como se muestra en el ejemplo siguiente). Por defecto, el tipo de origen cruzado se establece como "anónimo".

Sintaxis:

http_set_request_crossorigin(origin_type);

ArgumentoTipo Descripción
origin_type The cross origin type to use (a string)

Devuelve:

Ejemplo:

if string_lower(http_get_request_crossorigin()) != "use-credentials"
{
    http_set_request_crossorigin("use-credentials");
}
sprite_add("sprites/player_outfit_1.png", 0, false, false, 0, 0);

El código anterior comprobará primero el tipo de origen cruzado actualmente establecido y, si no es "use-credentials", se establecerá como "use-credentials" y, a continuación, se añadirá un sprite desde un archivo.
