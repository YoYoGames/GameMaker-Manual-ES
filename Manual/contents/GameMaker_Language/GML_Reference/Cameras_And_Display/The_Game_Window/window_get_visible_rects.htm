window_get_visible_rects

Con esta función puedes encontrar la región de superposición del rectángulo definido por (x1,y1) a (x2,y2) en cada una de las pantallas adjuntas. La función devolverá un array con 8 valores por pantalla (es decir, si tienes dos pantallas, el array tendrá una longitud de 16 índices), donde los valores [0 ... 3] corresponden a los valores overlapx1, overlapy1, overlapx2, overlapy2 -que definen la región de solapamiento en esta pantalla y se establecerán en 0,0,0,0 si no hay solapamiento- y los valores [4 ... 7] corresponden a los valores monitorx1, monitory1, monitorx2, monitory2 - las coordenadas de la pantalla en el espacio de visualización virtual. Esto puede utilizarse para comprobar si una posición de ventana guardada va a ser visible o no (el usuario puede haber desconectado un monitor externo o movido la ventana fuera de la pantalla, lo que ha dejado la posición de la ventana guardada como no válida), por ejemplo.

Sintaxis:

window_get_visible_rects(x1, y1, x2, y2);

ArgumentoTipo Descripción
x1 The left edge of the rectangle to check
y1 The top edge of the rectangle to check.
x2 The right edge of the rectangle to check
y2 The bottom edge of the rectangle to check.

Devuelve:

Ejemplo:

var wx = window_get_x();
var wy = window_get_y();
var ww = window_get_width();
var wh = window_get_height();
display_data = window_get_visible_rects(wx, wy, wx + ww, wy + wh);
display_num = array_length(display_data) / 8;

El código anterior generará una matriz 1D contenida en la variable display_data que contiene la información sobre las pantallas, así como creará la variable display_num para mantener el número de pantallas activas encontradas.
