mostrar_método_de_tiempo_de_presentación

Esta función se puede utilizar para establecer el método de sincronización que se utilizará para renderizar su juego. El método puede ser una de las siguientes constantes:

Constante Descripción
tm_sleep Utilizar el valor del margen de sueño como método principal de temporización
tm_countvsyncs Utilizar la sincronización vsync como método principal de sincronización (por defecto para todas las plataformas soportadas)

El método de sincronización vsync utiliza el soporte de las plataformas de destino para la sincronización vertical con el fin de proporcionar un ancla para los cálculos de tiempo de renderizado de los juegos, mientras que al establecer el modo de margen de sueño sólo se tratará de garantizar que cada fotograma dure la cantidad correcta de tiempo (digamos 1/30 o 1/60 de segundo) esperando o durmiendo. En general, la sincronización vsync por defecto dará los resultados más suaves, pero ten en cuenta que cuando se utiliza el método vsync, los valores del margen de sueño siguen siendo relevantes, aunque tendrán un impacto reducido y recomendamos mantenerlo en su valor por defecto cuando se utiliza este método.

Por defecto, en todas las plataformas , excepto en PS4, Ubuntu y HTML5, GameMaker utilizará el método de temporización vsync, mientras que en las plataformas no soportadas, sólo se podrá utilizar la temporización del margen de sueño.

Si desea establecer el margen de reposo, puede hacerlo con la función display_set_sleep_margin() y puede encontrar qué método de temporización se está utilizando actualmente con la función display_get_timing_method().

Sintaxis:

display_set_timing_method(method);

Argumento Tipo Descripción
method Método de temporización de la pantalla Constante The timing method to use (see the list of constants, above)

Devuelve:

N/A

Ejemplo:

if display_get_timing_method() != tm_sleep
{
    display_set_timing_method(tm_sleep);
    if display_get_sleep_margin() != 20
    {
        display_set_sleep_margin(20);
    }
}

El código anterior comprobará el método de temporización y, si no está establecido en tm_sleep, se establecerá en ese valor y el margen de reposo se establecerá en 20.
