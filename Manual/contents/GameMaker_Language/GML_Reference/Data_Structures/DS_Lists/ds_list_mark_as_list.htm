ds_list_mark_as_list

Esta función "marcará" (o "marcará") una posición dada dentro de una lista DS previamente creada como si tuviera otra lista DS. Esta funcionalidad es necesaria cuando se codifica JSON strings (que se puede crear con json_encode()), pero también puede ser útil cuando se anidan estructuras de datos, ya que los elementos marcados de esta manera serán automáticamente recogidos de la basura (destruidos) cuando la lista DS padre sea destruida. Esto significa que no tienes que recorrer manualmente el contenido de la lista y destruir los data structures marcados individualmente antes de destruir la lista "padre". Sin embargo, si borras la posición de la lista individualmente, la lista DS que contenía no será recogida de la basura y deberás llamar a la función de destrucción de la lista DS apropiada antes de borrar el elemento de la posición de la lista padre. Ten en cuenta también que si llamas a la función ds_list_clear() en una lista, cualquier elemento marcado como lista será destruido también cuando se borre la lista.

También hay que tener en cuenta que si la posición que se ha marcado cambia dentro de la lista, la "marca" se moverá con ella, por lo que si se ha marcado la posición 3 de la lista (por ejemplo) y luego se insertan 2 elementos más por debajo de ella para que suba a la posición 5, seguirá marcada como lista.

Sintaxis:

ds_list_mark_as_list(id, pos);

ArgumentoTipo Descripción
id The id of the list to mark.
pos The position within the list to mark.

Devuelve:

DS List ID or -1 (if it fails)

Ejemplo:

var j_list = ds_list_create();
var sub_list = ds_list_create();
ds_list_add(sub_list, health);
ds_list_add(sub_list, lives);
ds_list_add(sub_list, score);
ds_list_add(j_list, sub_list);
ds_list_mark_as_list(j_list, 0);

El código anterior crea dos listas DS, y luego rellena una con varios valores de las variables globales. Esta lista se añade entonces a la segunda lista, y la posición se "marca" como tal para poder codificarla correctamente más tarde.
