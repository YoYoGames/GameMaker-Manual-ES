Listas DS

Una lista DS es una estructura de datos que almacena información de forma secuencial a medida que se va añadiendo (de forma muy parecida a una matriz). Las listas DS son muy flexibles data structures, ya que permiten añadir valores al final o insertarlos en algún punto de la lista, así como barajar el orden (para aleatorizar los valores) u ordenarlos de forma ascendente o descendente.

NOTA: Se recomienda utilizar arrays en lugar de listas DS, ya que tienen características similares, son más fáciles de usar y se recolectan automáticamente.

Cuando acceda a la lista DS data structures, siempre debe tratar de usar valores enteros para la posición de la lista, y todos los índices no enteros serán redondeados por GameMaker si no lo hace. Si esto no es lo que necesita, entonces tendrá que hacer el redondeo usted mismo previamente antes de pasar el índice que desea comprobar.

NOTA: Como todos los recursos dinámicos, data structures ocupa memoria y, por tanto, debe destruirse siempre que ya no se necesite para evitar fugas de memoria que ralenticen y acaben por colapsar el juego.

Aparte de las funciones específicas que se enumeran a continuación, todos pueden utilizar una expresión (llamada accesor) para añadir o modificar el contenido de su lista DS. Este accesor se parece a un array 1D con la siguiente sintaxis:

list_index[| index]

Puede encontrar más información, además de ejemplos, en la página GML Overview en Accessors.

Existen las siguientes funciones que se ocupan de las listas DS:

  • ds_list_create
  • ds_list_destroy
  • ds_list_clear
  • ds_list_empty
  • ds_list_size
  • ds_list_add
  • ds_list_set
  • ds_list_delete
  • ds_list_find_index
  • ds_list_find_value
  • ds_list_insert
  • ds_list_replace
  • ds_list_shuffle
  • ds_list_sort
  • ds_list_copy
  • ds_list_read
  • ds_list_write

Las listas Ds pueden ser usadas para contener mapas DS y otras listas DS también, y por ello tenemos algunas funciones especiales que se usan para "marcar" posiciones dentro de la lista como contenedoras de estos data structures. En general, esto no es necesario, pero tiene la ventaja de recoger automáticamente la basura (destruir) los data structures marcados cuando se destruye la lista principal, y también es necesario cuando se utiliza junto con la función json_encode() para que la lista y sus datos se codifiquen correctamente. Las funciones para esto son:

  • ds_list_mark_as_list
  • ds_list_mark_as_map
  • ds_list_is_list
  • ds_list_is_map
