Rejillas DS

Una cuadrícula DS es básicamente un tipo de matriz con 2 dimensiones. Se define la rejilla estableciendo su anchura y altura enteras, lo que a su vez establece el número total de "celdas" dentro de la rejilla (la anchura y la altura multiplicadas) y estas celdas se utilizan para contener diferentes valores de datos (números reales o strings).

Este tipo de estructura te permite establecer y recuperar el valor de las celdas de la rejilla dando el índice x e y de la misma (que empieza por 0 tanto para el eje x como para el eje y y llega hasta el ancho - 1 y el alto -1). Pero también puedes establecer el valor en regiones específicas de la rejilla, añadir valores individuales y recuperar la suma, el máximo, el mínimo y el valor medio sobre una región determinada, lo que hace que esta estructura de datos sea la más versátil de todas las que tienes a tu disposición.

Cuando acceda a la rejilla DS data structures, siempre debe tratar de utilizar valores enteros para la posición de la celda, y todos los índices no enteros serán redondeados por GameMaker si no lo hace. Si esto no es lo que necesita, entonces tendrá que hacer el redondeo usted mismo previamente antes de pasar el índice que desea comprobar.

NOTA Como todos los recursos dinámicos, data structures ocupa memoria, por lo que debe destruirse siempre que no se necesite para evitar fugas de memoria que ralenticen y acaben por colapsar el juego.

NOTA si necesita comprobar si una estructura de datos existe, puede utilizar la función ds_exists() función.

Aparte de las funciones específicas que se enumeran a continuación, todos pueden utilizar una expresión (llamada accesorio) para añadir o modificar el contenido de su cuadrícula DS. Este accesorio se parece a un array 2D con la siguiente sintaxis:

grid_index[# xpos, ypos]

Puede encontrar más información, además de ejemplos, en la página GML Overview en Accessors.
