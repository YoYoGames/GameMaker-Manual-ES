Colas DS

Un DS queue es algo similar a una pila DS, pero funciona sobre la base de "primero en entrar, primero en salir" (FIFO). El valor que se introduce primero en el queue es también el primero que se retira de él. Puedes pensar en ello como en un queue en una tienda para pagar, donde la persona que está primero en el queue paga primero y luego sale de la tienda. Queues se utilizan típicamente para almacenar acciones o comportamientos (en forma de scripts, por ejemplo) que todavía necesitan ser realizados por una instancia, pero hay muchos otros usos.

NOTA: Como todos los recursos dinámicos, data structures ocupa memoria y, por tanto, debe destruirse siempre que ya no se necesite para evitar fugas de memoria que ralenticen y acaben por colapsar el juego.

NOTA: si necesita comprobar si una estructura de datos existe, puede utilizar la función ds_exists() función.

Existen las siguientes funciones que se ocupan de las colas DS:

  • ds_queue_create
  • ds_queue_destroy
  • ds_queue_clear
  • ds_queue_empty
  • ds_queue_size
  • ds_queue_dequeue
  • ds_queue_enqueue
  • ds_queue_head
  • ds_queue_tail
  • ds_queue_copy
  • ds_queue_read
  • ds_queue_write
