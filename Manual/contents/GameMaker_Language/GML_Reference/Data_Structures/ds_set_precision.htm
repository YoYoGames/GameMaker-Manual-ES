ds_set_precision

Al comparar valores, por ejemplo al buscar en un mapa u ordenar una lista, GameMaker debe decidir cuándo dos valores son iguales. Para strings y los valores enteros esto es claro, pero para los números reales, debido a los errores de redondeo en coma flotante, los números aparentemente iguales pueden convertirse fácilmente en desiguales. ¡Por ejemplo, es posible que (5 / 3) * 3 no sea igual a 5! Para evitar esto, se utiliza un valor de precisión en todas las funciones de números reales, y cuando la diferencia entre dos números es menor que esta precisión se consideran iguales. Por defecto se utiliza una precisión de 0.0000001 para todas las funciones de estructura de datos a menos que se cambie por esta función.

¡NOTA: Esta precisión se utiliza en todos los data structures pero no en otras comparaciones en GML!

Sintaxis:

ds_set_precision(prec);

Argumento Tipo Descripción
prec Real The precision value (default 0.0000001)

Devuelve:

N/A

Ejemplo:

ds_set_precision(0.0001);

El código anterior cambiará la configuración de precisión por defecto para todas las funciones de estructura de datos.
