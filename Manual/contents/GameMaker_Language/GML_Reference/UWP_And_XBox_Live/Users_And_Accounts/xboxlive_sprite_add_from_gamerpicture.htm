xboxlive_sprite_add_from_gamerpicture

Con esta función puedes obtener la imagen del jugador para un puntero de ID de usuario dado. La función trabaja de forma asíncrona, y lanzará un evento asíncrono Image Loaded para informarle de que la función ha terminado (como con las llamadas HTML5/URL a sprite_add()). El evento asíncrono Image Loaded tendrá un mapa DS almacenado en la variable incorporada async_load. El mapa contiene la siguiente información:

  • "nombre del archivo": La dirección completa path al archivo solicitado.
  • "id": El ID del recurso que has cargado. Será el mismo que la variable a la que has asignado el recurso.
  • "estado": Devuelve un valor inferior a 0 para un error.

NOTA: Cuando cargue dinámicamente un sprite en su juego en runtime, debe recordar eliminarlo de nuevo usando sprite_delete() cuando ya no lo necesites, de lo contrario existe el riesgo de que se produzca una fuga de memoria que ralentizará y acabará por colapsar tu juego.

Sintaxis:

xboxlive_sprite_add_from_gamerpicture(user_id, imagesize, xorig, yorig);

ArgumentoTipo Descripción
user_id The user ID (a pointer) to get the gamer picture for
imagesize Size in pixels of the sprite to be returned
xorig Indicate the x position of the origin in the sprite
yorig Indicate the y position of the origin in the sprite

Devuelve:

Async Request ID

Ejemplo:

var _a = 0;
var _num = xboxlive_get_user_count();
for (var i = 0; i < _num; ++i;)
{
    var _uid = xboxlive_get_user(i);
    if _uid != pointer_null
    {
        global.UserName[_a] = xboxlive_gamedisplayname_for_user(_uid);
        global.UserScore[_a] = xboxlive_gamerscore_for_user(_uid);
        global.UserRep[_a] = xboxlive_reputation_for_user(_uid);
        global.UserAvatar[_a] = xboxlive_sprite_add_from_gamerpicture(_uid, 256, 0, 0);
        ++a;
    }
}

El código anterior loops a través de los usuarios registrados y almacena sus datos de jugador en varias matrices globales.
