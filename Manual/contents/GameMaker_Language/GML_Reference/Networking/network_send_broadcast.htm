network_send_broadcast

Con esta función se pueden difundir los datos de un buffer localmente a un rango de direcciones IP (para más información sobre buffers ver Referencia - Bufferes). El rango está limitado al del dispositivo que ejecuta el server, de tal manera que si el dispositivo tiene una IP de 92.168.11.130, entonces los datos se difundirán sobre el rango 92.168.11.*. La función devolverá el número de bytes de datos enviados, o un número menor que 0 si el envío ha fallado.

Esta función sólo funcionará cuando se utilice con UDP - su server necesita ser TCP y su cliente necesita tener un socket de cliente UDP creado con network_create_socket_ext() para poder recibir cualquier emisión enviada desde server.

Esta función no funcionará cuando se utilice en un proyecto que se ejecute en el objetivo HTML5.

Sintaxis:

network_send_broadcast(socket, port, bufferid, size);

Argumento Tipo Descripción
socket ID de la toma de red The id of the socket to use.
port Real The port that the server will use.
bufferid ID del búfer The id of the buffer to get the data from.
size Real The size (in bytes) of the data.

Devuelve:

Real

Ejemplo:

buffer_seek(broadcast_buffer, buffer_seek_start, 0);
buffer_write(broadcast_buffer, buffer_string, global.ServerName);
network_send_broadcast(server, 6511, broadcast_buffer, buffer_tell(broadcast_buffer));

El código anterior escribe el nombre string del actual server (almacenado en "global.ServerName"), luego lo escribe en un binario buffer con el id "broadcast_buffer". Estos datos son entonces difundidos localmente a un rango de IPs (la IP del dispositivo está actualmente implicada como el rango base de difusión) al puerto 6511.
