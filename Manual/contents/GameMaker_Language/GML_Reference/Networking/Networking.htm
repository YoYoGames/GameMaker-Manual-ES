Conexión a la red

Las funciones de red de GameMaker se basan en sockets, que es un estándar de la industria en todas las plataformas. Con los sockets, tienes la posibilidad de crear tanto clientes como servers usando un simple GML que te permite crear incluso un juego de un solo jugador usando el modelo cliente/servidor (algo que permitirá añadir fácilmente el multijugador más adelante).

Básicamente, un socket es un object que puede enviar, recibir, conectar y escuchar puertos en la red. Estas funciones sólo se ocupan de las conexiones TCP/IP, que es en lo que se basa Internet (IPv4 para ser más precisos). En esencia, esto te permite conectar un socket usando una dirección IP (digamos 192.168.1.10) a otro socket en otra dirección IP (digamos 192.168.1.11). Una vez conectados, estos dos sockets pueden enviar datos de ida y vuelta utilizando las funciones de network_send_* (como network_send_packet()) y el evento asíncrono dedicado que es disparado por ellos. Este evento asíncrono generará un mapa DS que contendrá los datos recibidos así como otros detalles, y puedes encontrar los detalles completos en la sección sobre el Evento Asíncrono de Red.

En lugar de que los programas tengan que leer y ocuparse de todo el tráfico de red que llega a una máquina, las direcciones IP también se ocupan de los puertos, que están numerados del 0 al 65535. Esto permite que cada programa obtenga sólo los paquetes de datos que le interesan, y no todo lo que ha sido enviado por cada programa. Esto también ahorra un tiempo considerable de la CPU, ya que reduce seriamente el procesamiento de datos, por lo que con GameMaker también puedes "escuchar" los puertos. Así, un server creará un socket, y luego lo vinculará a un puerto específico para escuchar. Entonces recibirá notificaciones de conexiones, desconexiones y transferencias de datos desde ese socket.

Los propios "paquetes" de datos están formados por datos binarios extraídos de un buffer. Este buffer puede crearse y manipularse utilizando las funciones del Buffer de GameMaker.

Vale la pena señalar que si estás creando un proyecto para el objetivo HTML5, entonces no podrás usar las funciones de creación de server debido a las restricciones del navegador y tendrás que escribir tu propio server para esta plataforma usando algo como Electron o NodeJS. También tenga en cuenta que si su juego es multiplataforma y prevé conectarse a una instancia HTML5 del juego desde un server que se ejecuta en un dispositivo móvil o de escritorio, entonces su server sólo puede comunicarse con la instancia HTML5 utilizando el protocolo Web Socket (esto se hace utilizando la constante network_socket_ws o network_socket_wss - ver network_create_server() para más información). Esto significa que tu server puede tener que crear dos sockets en diferentes puertos, uno para conectarse a los objetivos no HTML5 (Android, Windows, etc...) y otro para conectarse al objetivo HTML5.

Existen las siguientes funciones para que puedas utilizarlas en la red:

  • network_create_server
  • network_create_server_raw
  • network_create_socket
  • network_create_socket_ext
  • network_connect
  • network_connect_async
  • network_connect_raw
  • network_connect_async_raw
  • network_resolve
  • network_set_config
  • network_set_timeout
  • network_send_broadcast
  • network_send_packet
  • network_send_raw
  • network_send_udp
  • network_send_udp_raw
  • network_destroy
