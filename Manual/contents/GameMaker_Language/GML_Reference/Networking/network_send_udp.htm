network_send_udp

Con esta función puede enviar datos a través de la red usando UDP a un server. La función toma el id del socket para conectarse, la URL para conectarse y el puerto a usar. A continuación, debe proporcionar el identificador del búfer que contiene los datos que se van a enviar (para más información sobre buffers, véase Referencia - Búferes) y, por último, el tamaño (en bytes) de los datos. UDP es "sin conexión" en el sentido de que en realidad no se hace una conexión, sólo se envía un paquete directamente a una IP, y el server recibe los datos entrantes de una dirección IP y tiene que tratarlos "tal cual". La función devolverá el número de bytes de datos enviados, o un número menor que 0 si el envío ha fallado. Cabe destacar que el tamaño final de los datos enviados que devuelve esta función incluirá también la información de la cabecera GameMaker, que son 12 bytes adicionales.

Esta función no funcionará cuando se utilice en un proyecto que se ejecute en el objetivo HTML5, y tampoco los proyectos HTML5 podrán recibir UDP.

Sintaxis:

network_send_udp(socket, url, port, bufferid, size);

Argumento Tipo Descripción
socket ID de la toma de red The id of the socket to use.
url Cadena The url or IP to connect to (a string).
port Real The port to connect to.
bufferid ID del búfer The id of the buffer to get the data from.
size Real The size (in bytes) of the data.

Devuelve:

Real

Ejemplo:

network_send_udp(sock, "www.macsweeneygames.com", 6510, buff, buffer_tell(buff));

El código anterior enviará un paquete UDP al server definido por la URL en el puerto 6510. Los datos se toman del buffer indexado en la variable "buff".
