network_send_raw

Con esta función puedes enviar un paquete de datos "en bruto" a través de la red. La función toma el id de socket para conectarse y luego debe suministrar el id de buffer que contiene los datos a ser enviados (para más información sobre buffers ver Referencia - Buffers) y finalmente el tamaño (en bytes) del paquete de datos. Los datos enviados no son formateados por GameMaker de ninguna manera y los dispositivos receptores obtendrán los datos como un flujo, lo que significa que tendrá que manejarlo usted mismo. La función devolverá el número de bytes de datos enviados, o un número menor que 0 si el envío ha fallado.

Argumento de las opciones

El último argumento es opcional, y sólo se utiliza con WebSockets. Permite elegir entre enviar datos binarios o de texto. Cualquiera de estas constantes puede ser especificada en este argumento:

Red Enviar Tipo Constante
Constante Descripción
network_send_binary Enviar un mensaje binario
network_send_text Enviar un mensaje de texto

Las APIs de algunas plataformas sólo aceptan mensajes de texto cuando se utilizan WebSockets (por ejemplo, Twitch), por lo que la constante network_send_text puede utilizarse en estos casos. Si no se especifica este argumento, se envían datos binarios por defecto.

Sintaxis:

network_send_raw(socket, bufferid, size, [options]);

Argumento Tipo Descripción
socket ID de la toma de red The id of the socket to use.
bufferid ID del búfer The id of the buffer to get the data from.
size Real The size (in bytes) of the data.
options Red Enviar Tipo Constante OPTIONAL Se utiliza para que los WebSockets elijan entre datos de texto o binarios; si no se especifica, se envían los datos binarios.

Devuelve:

Real

Ejemplo:

buff = buffer_load("player_save.dat");
network_send_raw(sock, buff, buffer_get_size(buff));

La información anterior carga un dato de buffer previamente guardado en memoria y devuelve el identificador de buffer que se almacena en la variable "buff". Este buffer completo se envía entonces como un paquete de datos en bruto a través de la red utilizando el socket identificado por la variable "sock".
