gamepad_test_mapping

Esta función se puede utilizar para establecer el mapeo del gamepad en aquellos objetivos que lo permitan. Se suministra el índice de la "ranura" para el gamepad a establecer, y luego el mapa string, que debe haber sido creado utilizando el formato SDL con los siguientes campos:

<guid>,<description>,platform:<platform-name>,<bindings>

Los datos que deben incluirse en cada uno de estos campos son:

  • guid - un string de dígitos y letras que identifican de forma exclusiva un tipo de dispositivo (se puede recuperar con la función gamepad_get_guid() función)
  • description - una descripción Ascii del dispositivo (se puede recuperar con la función gamepad_get_description() función)
  • platform-name - será "platform:android", "platform:windows", "linux" o "mac" (tenga en cuenta que esto se añadirá automáticamente a string cuando llame a esta función, por lo que no es necesario suministrarlo cuando construya su propia asignación strings)
  • bindings - un conjunto de entradas separadas por un ',' para vincular la entrada real a constantes específicas de GML

La lista de entradas separadas por comas sigue un formato específico para cada entrada:

<abstract-gp-name>:<value-binding>

Aquí <value-binding> (es decir, la entrada directa del gamepad) puede ser cualquiera de las siguientes:

  • a<N> - Representa un eje del gamepad, donde el <N> es uno de los ejes de 0 a gamepad_axis_count() -1
  • b<N> - Representa un botón del gamepad, donde el <N> es uno de los botones de 0 a gamepad_button_count() -1
  • h<N>.<mask> - Esto representa un sombrero del gamepad, donde la <N> representa un sombrero de 0 a gamepad_hat_count() -1, y el <mask> será 1, 2, 4 u 8.

Cada valor de entrada debe estar vinculado a un nombre SDL, que se expresa como <>abstract-gp-name>, que a su vez está vinculado a una constante GML. La siguiente tabla muestra la equivalencia entre el nombre SDL y la constante de GameMaker:

Cuando haya construido su mapa string debería tener un aspecto similar al siguiente:

"050000005e040000fd020000ffff3f00,Xbox Wireless Controller,a:b0,b:b1,start:b4,dpdown:h0.4,dpleft:h0.8,dpright:h0.2,dpup:h0.1,guide:b6,leftshoulder:b9,leftstick:b7,lefttrigger:a4,leftx:a0,lefty:a1,rightshoulder:b10,rightstick:b8,righttrigger:a5,rightx:a2,righty:a3,x:b2,y:b3,platform:android"

Sintaxis:

gamepad_test_mapping(index, mapping_string);

Argumento Tipo Descripción
index Real Which gamepad index "slot" to set.
mapping_string Cadena The map string to use (see the description for more information).

Devuelve:

N/A

Ejemplo:

var mapping = gamepad_get_guid(global.padIndex) + "," + gamepad_get_description(global.padIndex);
var len = array_length(global.PadInstances);
for (i = 0; i < len; i += 2)
{
    var left = global.PadInstances[i];
    var right = global.PadInstances[i+1];
    mapping += "," + left.sdlLabel + ":" + right.binding;
}
gamepad_test_mapping(global.padIndex, mapping);

El código anterior loop a través de un número de instancias y utilizar los valores de las diferentes variables que contienen para crear un mapeo string, que luego se establece para su uso en el gamepad en el índice de ranura dada.
