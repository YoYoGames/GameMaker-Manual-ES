Entrada del teclado

Al tratar con el teclado en GameMaker tienes una variedad de funciones que pueden ser usadas para reconocer diferentes estados del teclado como presionado o liberado. También hay algunas que almacenan todas las pulsaciones de teclas como string o que pueden decirte cuál fue la última tecla pulsada, así como otras que te permiten borrar el estado del teclado por completo.

NOTA Estas funciones están diseñadas únicamente para plataformas de escritorio Windows/Mac/Ubuntu. Es posible que algunas de las variables y constantes incorporadas no sean válidas en otras plataformas y muchas de las funciones no funcionarán en los móviles.

Cada carácter de entrada de una tecla (o varias teclas) se define por su código UTF8, que es un valor numérico. Este valor puede ser recuperado para cualquier carácter utilizando la función ord() pero, GameMaker también tiene una serie de constants para las teclas especiales del teclado más utilizadas y una función especial. Típicamente usted usaría una combinación de ord() con las funciones de keyboard_check*(), algo así:

if keyboard_check(ord("A"))
{
    hspeed = -5;
}

Así, lo anterior comprobará la tecla "A" y si está siendo pulsada entonces fijará la velocidad horizontal del object a -5. Tenga en cuenta que el uso de ord() de esta manera sólo funcionará correctamente si la entrada string tiene sólo un carácter de longitud y es un número del 0 al 9 o un carácter romano en mayúsculas de la A a la Z. La función ord() devolverá un valor UTF8 completo, pero las funciones keyboard_check*() sólo detectarán la A - Z y el 0 - 9. ¿Pero qué pasa si quieres utilizar las teclas de dirección? ¿O si quieres modificar una acción usando la tecla "shift"? Bueno, para eso GameMaker tiene una serie de constantes vk_ (vk_ significa tecla virtual) que puedes usar en lugar de ord():

Constante de la llave virtual (vk_*)
Constante Descripción
vk_nokey código de teclas que representa que no se ha pulsado ninguna tecla
vk_anykey código de teclas que representa que se ha pulsado cualquier tecla
vk_left código de la tecla de flecha izquierda
vk_right código de la tecla de flecha derecha
vk_up código de la tecla de flecha hacia arriba
vk_down código de la tecla de flecha hacia abajo
vk_enter tecla de entrada
vk_escape tecla de escape
vk_space clave de espacio
vk_shift cualquiera de las teclas de mayúsculas
vk_control cualquiera de las teclas de control
vk_alt tecla alt
vk_backspace tecla de retroceso
vk_tab tecla de tabulación
vk_home llave de casa
vk_end tecla de finalización
vk_delete tecla de borrar
vk_insert Insertar la llave
vk_pageup tecla pageup
vk_pagedown tecla de paginación
vk_pause tecla de pausa/interrupción
vk_printscreen tecla printscreen/sysrq
vk_f1 ... vk_f12 código de las teclas de función F1 a F12
vk_numpad0 ... vk_numpad9 teclas numéricas del teclado numérico
vk_multiply tecla de multiplicación en el teclado numérico
vk_divide tecla de división en el teclado numérico
vk_add tecla añadir en el teclado numérico
vk_subtract tecla de sustracción del teclado numérico
vk_decimal teclas de puntos decimales en el teclado numérico

Las siguientes constantes sólo pueden utilizarse con keyboard_check_direct():

Constante de la llave virtual (vk_*)
Constante Descripción
vk_lshift tecla de mayúsculas a la izquierda
vk_lcontrol tecla de control izquierda
vk_lalt tecla alt izquierda
vk_rshift tecla de mayúsculas a la derecha
vk_rcontrol tecla de control derecha
vk_ralt tecla alt derecha

El siguiente es un pequeño ejemplo de cómo utilizar las vk_ constantes:

if keyboard_check_pressed(vk_tab)
{
    instance_create_layer(x, y, "Controllers", obj_Menu);
}

El código anterior detectará si la tecla "Tab" está presionada y creará una instancia de object "obj_Menu" si lo está.

Si necesita comprobar un carácter clave que no sea 0 - 9, A - Z o una de las constantes VK, entonces debería comprobar una de las variables keyboard_*, como keyboard_lastchar por ejemplo:

var _key = keyboard_lastchar;
if ord(_key) == ord("ç")
{
    show_debug_message("ç key pressed");
}

Las siguientes funciones están disponibles para su uso cuando se trabaja con un teclado:

NOTA: Estas funciones no funcionarán cuando se utilice un teclado virtual en pantalla.

  • io_clear
  • keyboard_check
  • keyboard_check_pressed
  • keyboard_check_released
  • keyboard_check_direct
  • keyboard_clear
  • keyboard_set_map
  • keyboard_get_map
  • keyboard_unset_map
  • keyboard_set_numlock
  • keyboard_get_numlock

También hay algunas funciones que pueden utilizarse para simular la pulsación de teclas:

  • keyboard_key_press
  • keyboard_key_release

Por último, las siguientes variables están incorporadas a GameMaker y se puede acceder a ellas para obtener información sobre el estado del teclado y la entrada:

NOTA: Cuando se utiliza el teclado virtual, sólo se actualiza la variable keyboard_string se actualizará con la entrada del teclado.

  • keyboard_key
  • keyboard_lastkey
  • keyboard_lastchar
  • keyboard_string
