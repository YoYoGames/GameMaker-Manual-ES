Teclas y teclados virtuales

Cuando se desarrollan juegos para dispositivos de pantalla táctil, a menudo es importante obtener la entrada del teclado del usuario. Cuando se utiliza GameMaker esto se puede hacer de una de dos maneras:

  • Crea tus propias teclas virtuales en pantalla para obtener entradas
  • Abrir el teclado virtual del sistema operativo y obtener información de él

Lo que utilices dependerá mucho del juego y de lo que necesites, pero en general utilizarás las teclas virtuales cuando quieras tener botones en pantalla para el control del jugador o para los menús, mientras que utilizarás el teclado virtual cuando necesites la entrada del jugador, como obtener un nombre o establecer un valor.

Para que tus juegos funcionen correctamente en dispositivos móviles con pantalla táctil, puedes necesitar asignar áreas de la pantalla para que respondan a ciertas cosas cuando son tocadas por el usuario. Esto se hace asignando una "Tecla virtual" a una tecla estándar del teclado mapeada internamente y luego utilizando los eventos de teclado que generan para controlar tu aplicación. Vale la pena señalar que estas "teclas virtuales" están mapeadas a la posición de la pantalla en lugar de la posición de room y por lo tanto los valores x/y son absolutos, basados en el tamaño de la pantalla. Esto significa que no necesitas preocuparte por el uso de las vistas o las coordenadas relativas de room, y puedes simplemente definir las teclas en el evento Create de un object (sólo necesitas definir una tecla virtual una vez por habitación), y luego dibujar tu tecla sprites en el evento Draw GUI, dimensionando la capa GUI para que sea igual a la pantalla.

NOTA Estas funciones no funcionarán en las plataformas Windows, Ubuntu (Linux), ni en MacOS, que utilizan una pantalla táctil, aunque recibirán los clics del ratón como "toques".

Estas funciones son compatibles con los objetivos HTML5 y Opera GX.

Las funciones de las teclas virtuales disponibles son:

  • virtual_key_add
  • virtual_key_show
  • virtual_key_hide
  • virtual_key_delete

Si necesitas introducir texto o números en tu juego en un dispositivo de pantalla táctil, entonces puedes utilizar las funciones del teclado virtual proporcionadas para utilizar el teclado del SO del dispositivo. Estas funciones llamarán u ocultarán el teclado del sistema operativo y aceptarán la entrada de datos de éste, sin embargo, no activarán los eventos regulares del teclado, sino que actualizarán la variable keyboard_string con la entrada del usuario (esto se debe al hecho de que el teclado virtual puede introducir palabras enteras y no sólo caracteres individuales).

Al abrir o cerrar el teclado virtual se generarán Eventos Asíncronos del Sistema, en los que el mapa async_load DS se llenará con los siguientes pares clave/valor:

  • "event_type" - el tipo de evento del sistema recibido, que será "virtual keyboard status" para los teclados virtuales.
  • "screen_height" - la altura del teclado virtual (en píxeles). Será 0 si el teclado es invisible.
  • "keyboard_status" - el estado actual del teclado, devuelto como una de las siguientes cadenas:
    • "oculto"
    • "oculto"
    • "mostrando"
    • "visible"

Las funciones del teclado virtual disponibles son:

NOTA Estas funciones sólo son válidas para las plataformas de destino Android (incluido AndroidTV), iOS (incluido tvOS) y UWP .

  • keyboard_virtual_show
  • keyboard_virtual_hide
  • keyboard_virtual_status
  • keyboard_virtual_height
