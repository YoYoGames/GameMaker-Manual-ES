virtual_key_add

Esta función le permite asignar "toques" de un área de la pantalla a eventos del teclado. Esto significa que una vez que hayas asignado un área a una tecla virtual, todos los toques en esa área activarán el evento de teclado correspondiente a la tecla que hayas asignado al área. También puede asignar cada tecla virtual que defina a una variable, que luego puede utilizarse en las funciones posteriores de las teclas virtuales para mostrarlas, ocultarlas y eliminarlas. Estas teclas se asignan por habitación y serán eliminadas automáticamente por GameMaker al cambiar rooms.

La posición real de la llave virtual se basa en la posición de la pantalla y no en la posición de room, por lo que los valores x/y son absolutos en la pantalla. Esto significa que no necesitas preocuparte por el uso de vistas o las coordenadas relativas de room, y puedes simplemente definir las llaves en el evento Create de un object (sólo necesitas definir una llave virtual una vez por habitación), y luego dibujar tu llave sprites en el evento Draw GUI, dimensionando la capa GUI para que sea la misma que la pantalla.

Sintaxis

virtual_key_add(x, y, w, h, keycode);

Argumento Tipo Descripción
x Real The x coordinate (left side) of the virtual key on the screen
y Real The y coordinate (top side) of the virtual key on the screen
w Real The width of the virtual key
h Real The height of the virtual key
keycode Constante de la llave virtual (vk_*) Which keyboard key event should be triggered by touching this area

Devuelve:

Virtual Key ID

Ejemplo:

global.Left = virtual_key_add(32, 32, 64, 64, vk_left);

El código anterior crea una tecla virtual de 64x64 píxeles cuadrados, posicionada en la pantalla en (32, 32) que activará el evento vk_left cuando se toque y asigna el índice de esta tecla virtual a una variable global.
