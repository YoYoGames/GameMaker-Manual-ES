teclado_virtual_show

Esta función se puede utilizar para mostrar el teclado virtual en el dispositivo que ejecuta el juego. Cuando se llama a esta función es necesario proporcionar una de las siguientes constantes para cada uno de los tres primeros argumentos:

  • Tipo de teclado: Esta constante se utiliza para establecer qué juego de teclas estará disponible en el teclado virtual. Los tipos de teclado disponibles son:
Tipo de teclado virtual constante
Tipo de teclado Descripción
kbv_type_default El tipo de teclado por defecto para el sistema actual.
kbv_type_ascii Un teclado sólo ASCII.
kbv_type_url Un teclado normal optimizado para la introducción de URL. Suele contar con un ".com" u otras teclas de dominio, así como con las teclas "/" y ".".
kbv_type_email Un teclado normal optimizado para la introducción de mensajes de correo electrónico. Suele incluir los caracteres "@" y ".".
kbv_type_numbers Un teclado sólo de números, que suele aparecer como un teclado numérico.
kbv_type_phone Un teclado telefónico. Normalmente, sólo números con las teclas "*" y "#".
kbv_type_phone_name Un teclado optimizado para introducir tanto un número de teléfono como un nombre. Suele ser similar a un teclado ASCII, pero con una selección limitada de caracteres especiales.

  • Tipo de retorno: Esta constante se utiliza para establecer lo que se muestra en la tecla de retorno/acción del teclado virtual. Los tipos de retorno disponibles son:
Constante del tipo de retorno del teclado virtual
Tipo de devolución Descripción
kbv_returnkey_default El título de la tecla de retorno por defecto para el sistema actual.
kbv_returnkey_go Establece el título de la tecla de retorno como "Go".
kbv_returnkey_google Establece el título de la tecla de retorno a "Google", o a un icono de búsqueda genérico en algunos casos.
kbv_returnkey_join Establece el título de la tecla de retorno como "Go".
kbv_returnkey_next Establece el título de la tecla de retorno en "Siguiente".
kbv_returnkey_route Establece el título de la tecla de retorno como "Ruta".
kbv_returnkey_search Establece el título de la tecla de retorno a "Buscar", o a un icono de búsqueda genérico en algunos casos.
kbv_returnkey_send Establece el título de la tecla de retorno en "Enviar".
kbv_returnkey_yahoo Establece el título de la tecla de retorno a "Yahoo", o a un icono de búsqueda genérico en algunos casos.
kbv_returnkey_done Establece el título de la tecla de retorno como "Hecho".
kbv_returnkey_continue Establece el título de la tecla de retorno en "Continuar".
kbv_returnkey_emergency Establece el título de la tecla de retorno como "Llamada de emergencia".

  • Tipo de autocapitalización: Esta constante se utiliza para determinar cómo se deben autocapitalizar las palabras escritas a través del teclado virtual. Los tipos de autocapitalización disponibles son:
Constante del tipo de mayúsculas del teclado virtual
Tipo de autocapitalización Descripción
kbv_autocapitalize_none La autocapitalización está desactivada.
kbv_autocapitalize_words Las palabras se escriben en mayúsculas.
kbv_autocapitalize_sentences Las frases se escriben en mayúsculas.
kbv_autocapitalize_characters Todos los caracteres irán en mayúsculas.

El último argumento es para activar/desactivar el texto predictivo, y esto se establecería en true para permitirlo, y false en caso contrario, pero ten en cuenta que el hecho de que se permita no significa que se vaya a utilizar ya que eso dependerá de las preferencias del usuario en el dispositivo. Cuando esté en modo de texto predictivo, el teclado virtual no generará eventos normales de pulsación de teclas de GameMaker. En su lugar, sólo actualizará el último carácter pulsado y las variables del teclado string. Esto se debe a la incapacidad de detectar si un cambio en el campo de texto interno utilizado para detectar la pulsación de teclas proviene de una tecla real del teclado virtual, o de una sugerencia de texto. En estos casos se querrá leer la entrada de keyboard_string en lugar de leer cualquier tipo de entrada de tecla en bruto.

También es importante tener en cuenta que el usuario obtendrá diferentes teclados con diferentes capacidades dependiendo del SO de la plataforma, con las siguientes advertencias de uso por objetivo:

  • UWP: La implementación de UWP no soporta return_key_type, autocapitalization_type o predictive_text_enabled. Pasar estos parámetros en una función no causará ningún error, simplemente no tendrán ningún efecto en el teclado.

  • Android: En Android, el tipo de teclado kbv_type_phone_name no es compatible y mostrará un teclado ASCII en su lugar, y el tipo de tecla de retorno sólo puede ser kbv_returnkey_go, kbv_returnkey_search, kbv_returnkey_next, kbv_returnkey_send o kbv_returnkey_done. Si se utilizan otros tipos de teclas de retorno en esa plataforma, el sistema utilizará por defecto las teclas de retorno de sustitución que se indican a continuación:
Tecla de retorno no disponible Llave de repuesto
kbv_returnkey_google
kbv_returnkey_yahoo		 kbv_returnkey_search
kbv_returnkey_join
kbv_returnkey_route
kbv_returnkey_emergency		 kbv_returnkey_go
kbv_returnkey_continue kbv_returnkey_next

  • AndroidTV / FireTV: Los tipos de teclas de retorno personalizados no son compatibles con los teclados ASCII - la tecla de retorno por defecto siempre se mostrará.

  • tvOS: El texto predictivo y las mayúsculas automáticas no son compatibles con tvOS, y la altura del teclado devuelta por los eventos del sistema y la función keyboard_virtual_height() se ajustará por defecto a la altura de la pantalla debido a que abarca toda la pantalla y a que las funciones de dimensión rectangular del teclado están desactivadas en tvOS. También hay que tener en cuenta que los eventos del teclado físico (bluetooth) no se emitirán a menos que se haya abierto el teclado virtual.

Al llamar a esta función se generará un Evento Asíncrono del Sistema, en el que el mapa async_load DS se rellenará con los siguientes pares clave/valor:

  • "tipo_de_evento" - será "estado del teclado virtual" cuando sea activado por las funciones del teclado virtual.

  • "screen_height" - la altura del teclado virtual (en píxeles). Será 0 si el teclado es invisible.

  • "keyboard_status" - el estado actual del teclado, devuelto como una de las siguientes cadenas:
    • "oculto"
    • "oculto"
    • "mostrando"
    • "visible"

Sintaxis

keyboard_virtual_show(keyboard_type, return_key_type, autocapitalization_type, predictive_text_enabled);

Argumento Tipo Descripción
keyboard_type Tipo de teclado virtual constante Determines the keyset available on the virtual keyboard (see tables at the top).
return_key_type Constante del tipo de retorno del teclado virtual Determines what is shown on the return/action key of the virtual keyboard (see tables at the top).
autocapitalization_type Constante del tipo de mayúsculas del teclado virtual Determines how/if the words typed via the virtual keyboard will be autocapitalized (see tables at the top).
predictive_text_enabled Booleano Set to true/false para activar/desactivar la introducción de texto predictivo.

Devuelve:

N/A

Ejemplo:

if input == false
{
    input = true;
    keyboard_virtual_show(kbv_type_numbers, kbv_returnkey_default, kbv_autocapitalize_none, false);
}

El código anterior mostrará el teclado virtual del sistema operativo si la variable dada no está establecida en true.
