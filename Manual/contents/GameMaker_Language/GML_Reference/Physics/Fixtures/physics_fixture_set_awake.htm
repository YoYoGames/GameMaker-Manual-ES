physics_fixture_set_awake

Cuando se inicia un room con física, o se vinculan los fixtures a las instancias, los fixtures siempre se consideran "despiertos"... es decir, están procesando eventos e interactuando con las instancias circundantes. Sin embargo, esto a veces puede dar lugar a a problemas, especialmente si tienes varias instancias con fixtures que están al lado cuando se inicia un room (piensa en una torre hecha de varias instancias con fixtures) ya que al estar "despiertas" se moverán y posiblemente cambien de posición debido al inicio repentino de la gravedad y las colisiones que les afectan. En estos casos se puede utilizar esta función para flag el fixture como si estuviera "dormido", en cuyo caso no estará procesando nada cuando se cree por primera vez en el room.

Sintaxis:

physics_fixture_set_awake(fixture, state)

ArgumentoTipo Descripción
fixture the index of the fixture
flag whether a fixture is awake (true) o no (false)

Devuelve:

Ejemplo:

physics_fixture_set_awake(fix_Cloud, true);

El código anterior flag el accesorio está "despierto" cuando está vinculado a una instancia.
