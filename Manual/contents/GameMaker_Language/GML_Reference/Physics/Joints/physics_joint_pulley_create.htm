physics_joint_pulley_create

La articulación de polea se utiliza para conectar dos instancias dentro del mundo de la física de tal manera que el movimiento de una influya directamente en el movimiento de la otra. Estas articulaciones se anclan primero en el espacio del mundo en dos puntos, cada uno conectado a una instancia en su centro de masa. Esto puede cambiarse estableciendo las coordenadas locales de anclaje x/y en relación con el origen de la instancia, lo que significa que la conexión real de la instancia al punto de anclaje del mundo de la física puede desplazarse a otro lugar distinto de su origen. La siguiente imagen ilustra esto:

Si se observa la instancia 2 se puede ver que su punto de anclaje local ha sido desplazado 20 píxeles a la "izquierda" de su origen (¡Recuerde! Los puntos de anclaje mundiales se definen utilizando las coordenadas de room, mientras que los puntos de anclaje locales se definen en relación con el origen de la instancia). También puede especificar una relación para la articulación de la polea, que indica que un lado o el otro se mueven más rápido, con un valor por debajo de 1 que hace que la primera instancia se mueva más rápido (por ejemplo, 0,5 hará que se mueva el doble de rápido) y un valor por encima de 1 que hace que la segunda instancia se mueva más rápido (por ejemplo, 2 hará que se mueva el doble de rápido).

Al igual que con todas las articulaciones, si se establece el valor "col" a true entonces las dos instancias pueden interactuar y colisionar entre sí, pero sólo si tienen eventos de colisión, sin embargo, si se establece a false, no van a colisionar no importa qué.

Sintaxis:

physics_joint_pulley_create(inst1, inst2, w_anchor1_x, w_anchor1_y, w_anchor2_x, w_anchor2_y, l_anchor1_x, l_anchor1_y, l_anchor2_x, l_anchor2_y, ratio, col)

Argumento Tipo Descripción
inst1 ID de la instancia The first instance to connect with the joint
inst2 ID de la instancia The second instance to connect with the joint
w_anchor1_x Real The x coordinate where the first part of joint is anchored, within the game world
w_anchor1_y Real The y coordinate where the first part of the joint is anchored, within the game world
w_anchor2_x Real The x coordinate where the second part of the joint is anchored, within the game world
w_anchor2_y Real The y coordinate where the second part of the joint is anchored, within the game world
l_anchor1_x Real The local x coordinate where joint 1 is anchored to the first instance
l_anchor1_y Real The local y coordinate where joint 1 is anchored to the first instance
l_anchor2_x Real The local x coordinate where joint 2 is anchored to the second instance
l_anchor2_y Real The local y coordinate where joint 2 is anchored to the second instance
ratio Real Set the velocity ratio between two instances
col Booleano Whether the two instances can collide (true) or not (false)

Devuelve:

Physics Joint ID

Ejemplo:

var t_fix, inst1, inst2;
t_fix = physics_fixture_create();
physics_fixture_set_circle_shape(t_fix, 20);
inst1 = instance_create_layer(150, room_height - 90, "Instances", obj_Block);
inst2 = instance_create_layer(300, room_height - 90, "Instances", obj_block);
physics_fixture_bind(t_fix, inst1);
physics_fixture_bind(t_fix, inst2);
physics_joint_pulley_create(inst1, inst2, 150, room_height - 140, 300, room_height - 140, 0, 0, 0, 0, 2, 0);
physics_fixture_delete(t_fix);

El código anterior crea y define un nuevo fixture, crea dos instancias de "obj_Block", y luego vincula este fixture a dos de ellas. A continuación, define una articulación de polea entre estas instancias, sin desplazamiento para las articulaciones, una relación de 2:1 (lo que significa que inst2 se moverá más rápido). No se producen colisiones entre las dos instancias en la polea.
