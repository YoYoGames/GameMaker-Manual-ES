Variables físicas

Al igual que con el sistema "tradicional" de colisiones y movimientos, el sistema de física también tiene una serie de variables de instancia asignadas a cada instancia de un object cuando se crea. Algunas de estas propiedades pueden ser definidas por el usuario en GML, pero muchas son de sólo lectura y otras más sólo son aplicables en el evento de colisión.

Las siguientes variables existen en todos los objects habilitados para la física y pueden ser leídas y escritas (es decir, cambiadas) usando el código de las instancias y las funciones de script:

  • phy_active
  • phy_angular_velocity
  • phy_angular_damping
  • phy_linear_velocity_x
  • phy_linear_velocity_y
  • phy_linear_damping
  • phy_speed_x
  • phy_speed_y
  • phy_position_x
  • phy_position_y
  • phy_position_xprevious
  • phy_position_yprevious
  • phy_rotation
  • phy_fixed_rotation
  • phy_bullet

Ciertas variables son de sólo lectura, lo que significa que la información que contienen no puede ser cambiada a través del código de instancia o de las funciones de script (pero puede ser cambiada internamente por GameMaker, por lo que son variables y no constantes). La siguiente lista muestra las variables de sólo lectura que existen en todos los objetos habilitados para la física:

  • phy_speed
  • phy_com_x
  • phy_com_y
  • phy_dynamic
  • phy_kinematic
  • phy_inertia
  • phy_mass
  • phy_sleeping

Por último, hay una serie de variables temporales que existen en la física habilitada objects y se crean sólo en el evento de colisión. Estas variables son como las enumeradas anteriormente y son de sólo lectura:

  • phy_collision_points
  • phy_collision_x
  • phy_collision_y
  • phy_col_normal_x
  • phy_col_normal_y
