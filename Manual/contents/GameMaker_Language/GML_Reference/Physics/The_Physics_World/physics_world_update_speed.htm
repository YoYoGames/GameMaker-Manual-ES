physics_world_update_speed

GameMaker actualiza las cosas basándose en la velocidad de room, de modo que si la ajustas a, digamos, 30, GameMaker ejecutará 30 pasos en el transcurso de un segundo. Sin embargo, para las funciones de física eso puede no ser suficiente y usted puede querer que las cosas se actualicen a una velocidad ligeramente más rápida para aumentar la estabilidad o la precisión. Para ello utilizamos la función physics_world_update_speed() que establece la velocidad de actualización del sistema de física independientemente de la velocidad de room. Esto significa que puedes tener una velocidad de room de 30, pero establecer la física a 60, duplicando efectivamente la velocidad a la que el sistema de física se actualiza y realiza sus cálculos en comparación con la velocidad a la que se actualiza el paso.

NOTA: actualmente no se puede configurar a una velocidad inferior a room, pero es posible que las futuras actualizaciones lo cambien.

Sintaxis:

physics_world_update_speed(speed)

Argumento Tipo Descripción
speed Real The number of times per second that the physics system updates

Devuelve:

N/A

Ejemplo:

physics_world_update_speed(room_speed * 2);

El código anterior establece que la velocidad de actualización del sistema de física es el doble de la velocidad de room.
