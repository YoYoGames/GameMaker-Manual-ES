physics_particle_create

Con esta función puedes crear una sola partícula en cualquier lugar de tu juego room, estableciendo ciertas flags y propiedades. Las "banderas" son el valor de retorno de una combinación de las siguientes constantes:

Física Partícula Flag Constante
Constante Descripción
phy_particle_flag_water Las propiedades por defecto de una partícula de cuerpo blando.
phy_particle_flag_zombie Una partícula zombi es aquella que se destruirá después de un solo paso con todas las demás marcadas de esta manera.
phy_particle_flag_wall Esto define la partícula como estática, esencialmente creándola como un inamovible object en la simulación física, ya que permanecerán en una posición fija sin importar lo que colisione con ellas. Deberías usar este flag en lugar de establecer la densidad a 0.
phy_particle_flag_spring Las partículas del muelle producen el efecto de estar unidas entre sí, como si se tratara de un muelle. Las partículas creadas con este flag están "conectadas" por parejas, y cada partícula está conectada a la que estaba más cerca de ella en el momento de la creación. Una vez emparejadas, las partículas no cambian de "pareja" , y cuanto más lejos las separe una fuerza externa, mayor será la potencia con la que colisionarán cuando se retire esa fuerza externa. Obsérvese que, por mucho que las partículas emparejadas se alejen unas de otras, la conexión entre ellas no se romperá.
phy_particle_flag_elastic Las partículas elásticas se deforman y también pueden rebotar cuando chocan con otros cuerpos rígidos en la simulación física.
phy_particle_flag_viscous Una partícula viscosa es aquella que se muestra "pegajosa", como el aceite. Las partículas viscosas se agrupan y se pegan más.
phy_particle_flag_powder Las partículas de polvo producen un efecto de dispersión como el que se puede ver con la arena o el polvo.
phy_particle_flag_tensile Las partículas de tracción se utilizan para producir el efecto de la tensión superficial, o la curvatura tensa en la superficie de un cuerpo de líquido. Pueden utilizarse, por ejemplo, para crear la tensión superficial que se observa en una gota de agua. Una vez que se rompe la tensión, las partículas rebotan como si fueran elásticas, pero también siguen atrayéndose entre sí. Como resultado, las partículas tienden a formar grupos al rebotar.
phy_particle_flag_colourmixing Las partículas que mezclan colores adoptan parte del color de otras partículas con las que colisionan. Ten en cuenta que si sólo una de las dos partículas que colisionan es de mezcla de colores, la otra partícula conserva su color anterior a la colisión.

Estos flags utilizan el enmascaramiento de bits para crear un valor de salida final que luego se comprueba para establecer las diferentes propiedades básicas de la partícula (siendo la propiedad base siempre la de phy_particle_flag_water). Por ejemplo, si se quiere simular un líquido viscoso con tensión superficial se utilizaría el bitwise o "|" para flag los bits apropiados como se muestra en el ejemplo siguiente. De esta manera puedes establecer diferentes propiedades (aparte de las propiedades globales) para cada partícula individual creada.

Además de flags, también se puede establecer la posición en la room en la que se creará la partícula, la velocidad horizontal y vertical inicial, el color y el alfa iniciales, así como la categoría de usuario. Los valores de color y alfa sólo se utilizan al dibujar la partícula mediante la función physics_particle_draw()pero el valor de la categoría se utiliza en otras muchas funciones. Este valor es un valor entero arbitrario que le das a cada partícula que quieres que tenga propiedades similares, y se puede utilizar para establecer flags más tarde para esa categoría específica, así como una serie de otras cosas. Por ejemplo, si ha definido una partícula de "agua" y otra de "aceite", le daría a todas las partículas marcadas como "agua" la categoría de usuario 1 y a todas las partículas creadas como "aceite" la categoría de usuario 2, lo que le permitiría dirigirse posteriormente a una sola categoría para cambiar su flags u obtener datos específicos.

NOTA: Puede utilizar cualquier valor entero para la categoría, excepto 0, que está reservado por la simulación y se utiliza para seleccionar todas las categorías en otras funciones (como para el dibujo).

La función devuelve un valor de índice (o ID) único para la partícula que puede ser almacenado en una variable para ser utilizado en otras funciones que hacen referencia a la partícula directamente.

Sintaxis:

physics_particle_create(flags, x, y, xv, yv, col, alpha, category)

Argumento Tipo Descripción
flags Física Partícula Flag Constante(s) The flags to set on the particle.
x Real The x position to create the particle at.
y Real The y position to create the particle at.
xv Real The initial horizontal velocity.
yv Real The initial vertical velocity.
col Color The base colour to use for the particle.
alpha Real The base alpha to use for the particle.
category Real The user defined category to which the particle belongs.

Devuelve:

Physics Particle ID

Ejemplo:

var flags = phy_particle_flag_water | phy_particle_flag_viscous | phy_particle_flag_tensile;
var x_vel = lengthdir_x(5, image_angle);
var y_vel = lengthdir_y(5, image_angle);
var p = physics_particle_create(flags, x, y, x_vel, y_vel, c_white, 1, 1);

El código anterior creará variables para establecer el tipo de partícula y su velocidad inicial, y luego las utilizará para crear una partícula en la posición x/y de las instancias.
