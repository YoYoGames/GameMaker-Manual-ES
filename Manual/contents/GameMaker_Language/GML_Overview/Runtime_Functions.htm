Funciones de tiempo de ejecución

La definición general de una función es algo así:

Una función tiene una entrada y una salida, y la salida está relacionada con la entrada.

En GameMaker esto se traduce en dos cosas diferentes, pero ambas funcionan de la misma manera:

  • Funciones de tiempo de ejecución - Una función de tiempo de ejecución es aquella que se suministra como parte del Lenguaje GameMaker (GML). Éstas están incorporadas al lenguaje y pueden ser usadas para hacer que ocurran cosas en sus juegos y hay un gran número de funciones GML runtime disponibles para usted, todas las cuales están explicadas en la sección de Referencia del Lenguaje del manual.
  • Funciones de script y métodos- Guión funciones y Metos Variables son funciones que has creado tú mismo utilizando los bloques básicos del Lenguaje GameMaker (GML), y éstas pueden incluir funciones de runtime como parte de su código.

En ambos casos, una función tiene la forma de un nombre de función, seguido de los argumentos de entrada entre paréntesis () y separados por comas (si la función no tiene argumentos de entrada, sólo se utilizan paréntesis). Esta página se centra en las funciones de tiempo de ejecución (es decir, las que están incorporadas a GameMaker para formar el GameMaker Language), pero las reglas generales que se muestran a continuación se aplican a todos los tipos de funciones.

A continuación se presenta un esquema de la estructura de una función:

<function>(<arg0>, <arg1> ,... <argN>);

Algunas funciones pueden devolver valores y pueden ser utilizadas enexpresiones, mientras que otras simplemente ejecutan comandos, como se ilustra en los siguientes dos ejemplos de funciones de runtime:

// Destroy the calling instance
// This requires no arguments and returns nothing
instance_destroy();

// Get the distance from the current instance position to the mouse position
// This takes four arguments and returns a real value
dist = point_distance(x, y, mouse_x, mouse_y);

NOTA En cualquier lugar del manual donde aparezca "N/A" como valor de retorno de una función, significa que la función no está destinada a devolver ningún valor como parte de su operación. Si intenta comprobar un valor de retorno de estas funciones, obtendrá el valor undefined.

A veces, el valor de retorno de una función puede ser un valor que desee utilizar en una asignación, pero debe tener en cuenta que es imposible utilizar cualquier función directamente como lado izquierdo de una asignación. Por ejemplo, el siguiente código para establecer la velocidad en una instancia le daría un error:

instance_nearest(x, y, obj).speed = 0;

El valor de retorno de la expresión en ese ejemplo de código es un número entero (el valor de identificación único para la instancia más cercana) y por lo tanto debe ser encerrado entre paréntesis para ser utilizado de esta manera y dirigir correctamente la instancia requerida (ver Direccionamiento de variables en otras instancias para más información). El código anterior se escribiría correctamente como

(instance_nearest(x, y, obj)).speed = 0;

//or more correctly still

var inst = instance_nearest(x, y, obj);
inst.speed = 0;

La sección de Referencia del Lenguaje del manual enumera todas las funciones de runtime disponibles para usted y le da ejemplos de cómo se pueden utilizar, así como otra información importante sobre lo que pueden devolver o cualquier evento que puedan desencadenar, etc... Para obtener más información sobre las funciones y métodos definidos por el usuario en script, consulte la sección sobre funciones de script y la sección sobre variables de métodos.
