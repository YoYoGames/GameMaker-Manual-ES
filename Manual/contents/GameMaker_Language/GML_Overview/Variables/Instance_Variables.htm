Variables de instancia

Una variable de instancia se crea dentro de una instancia de un object y se considera única para esa instancia - es decir, muchas instancias del mismo object pueden tener la misma variable, pero cada variable puede tener un valor diferente ya que son únicas para cada instancia. ¿Pero cómo se crea una variable de instancia? Bueno, se crean nuevas variables simplemente asignándoles un valor como se muestra en este pequeño ejemplo (esto se llama declarar la variable):

potions = 12;
life = 100;
name = "Jock MacSweeney";
strength = 5.5;
armour = -2;

Como puede ver, sólo tiene que dar el nombre y luego un valor (el valor puede ser cualquier tipo de datos) para establecer esa variable y tenerla lista para su uso dentro de una instancia de la object para la que está codificando (tenga en cuenta que el valor puede venir del valor de retorno de una función o del resultado de una operación entre otras variables, etc...). Estas variables pueden ser usadas y modificadas de varias maneras dentro de la instancia, por ejemplo este código podría estar en un evento de colisión y utilizarse para quitar una cantidad de la variable "life":

life -= 5 + armour;

Si "life" está en 100, ahora tendrá un valor de 97 (100 - (5 + -2) = 97). Ahora, ese es un ejemplo simple, y usted podría reemplazar "armour" por el valor real de -2, pero ¿qué sucede si ese valor está codificado en varios lugares y luego decide cambiarlo? Usted tendría que ir a través de TODO su código y cambiar cada -2 a lo que el nuevo valor lo que lleva mucho tiempo y es muy propenso a errores. Pero si utiliza una variable, todo lo que tiene que hacer es reasignarle un nuevo valor y el código utilizará automáticamente ese nuevo valor a partir de entonces, haciendo las cosas mucho más flexibles y mucho más fáciles de arreglar en caso de que haya un problema. También hay que tener en cuenta que, incluso si un valor no va a cambiar, es mucho más fácil recordar lo que significa una variable llamada "life" en lugar de limitarse a mirar un número.

GameMaker tiene una colección de variables de instancia "construidas" también, por lo que debe estar al tanto de ellas ya que puede nombrar una de sus propias variables de instancia igual o desear tener su propia variable global con el mismo nombre y preguntarse por qué por qué se producen errores. Sin embargo, son fáciles de detectar, ya que se muestran en un color diferente en el editor de código y también aparecen en la función de autocompletar y se muestran en una barra en la parte inferior del editor de código.
