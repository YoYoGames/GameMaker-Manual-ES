Código de comentarios

Cuando se trabaja en colaboración con otras personas, cuando se trata de un proyecto grande, o incluso sólo para su propia referencia y debugging, dejar comentarios en su código de eventos y scripts es muy importante. En GameMaker tienes varios mecanismos que te ayudan a dejar notas y comentarios sobre secciones de código e incluso bloques enteros de código que pueden verse en la lista de eventos de object que estás editando en ese momento. De esta manera puedes dejar notas para ti mismo y para tus colegas, o para explicar una pieza de código particularmente complicada, o incluso simplemente dejar un recordatorio sobre lo que hace algo, o lo que quieres que haga algo en el futuro.

Para dejar un simple comentario de una línea, lo primero que puede hacer es utilizar // antes del texto. Por ejemplo:

//initialize variables
sr = 10;
hp = 100;

También puede dejar comentarios de varias líneas, para dar crédito, para omitir una sección completa de código para debugging, o incluso para dejar notas sobre cómo quiere cambiar el código en el futuro. Para ello puede utilizar /* ... */ como este:

/*
Setup GIF Variables to initial values (these can be edited so you don't need to use the setter functions all the time if you are always using the same values)

IMPORTANT! Please see the individual "set" scripts for more extensive information about each of the different variables.
*/

frame_num = 30;
frame_steps = 5;
frame_width = 256;
frame_height = 256;
frame_quality = gifcam_quality_high;

Aparte de los comentarios, también puede utilizar las etiquetas especiales #region para marcar una sección de código para el Plegado de Código, y cada región puede contener un comentario. El funcionamiento es el siguiente: añades una etiqueta #region -junto con cualquier texto de comentario- a la parte superior de una sección de código y luego añades la correspondiente etiqueta #endregion a la parte inferior de la sección y entonces puedes "plegar" el código, es decir, ocultarlo para poder reducir el número de líneas de código visibles en el editor en cualquier momento:

El código real es algo así:

#region This is an example region

width = sprite_get_width(sprite);
height = sprite_get_height(sprite);
xoff = sprite_get_xoffset(sprite);
yoff = sprite_get_yoffset(sprite);

#endregion Further comment here if required

Finalmente, vale la pena mencionar que cuando se trabaja con eventos en el Editor de Objetos, se puede utilizar un comentario especial "description" al comienzo del evento para decir cuál es el contenido general del evento, y esto se mostrará en la ventana de eventos del Editor Object. Esto debe ir en la primera línea del editor de código, y tiene el siguiente formato:

/// @description <This Is An Event Description>

Esto se mostrará ahora en la ventana de eventos, por ejemplo

Tenga en cuenta que puede definir un comentario por defecto que se creará para todos los eventos desde las Preferencias del Editor de Objetos.

Finalmente, si estás escribiendo funciones de script y quieres tener algún control sobre cómo son tratadas por el IDE (por ejemplo: tener autocompletado o mostrar los parámetros de script ) entonces deberías escribir tus comentarios usando la notación JSDoc como se explica aquí.
