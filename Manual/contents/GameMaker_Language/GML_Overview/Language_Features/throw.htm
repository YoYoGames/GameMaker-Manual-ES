La sentencia throw runtime le permite generar sus propias excepciones en tiempo de ejecución, terminando el juego y mostrando un mensaje de error, utilizando la siguiente sintaxis:

throw (<expression>);

La expresión utilizada puede ser un valor o un string o cualquier otro data type, y esto generará un error de excepción que es - por defecto - mostrado en la pantalla y al cerrar el mensaje de error el juego terminará. Por ejemplo llamando a esto:

throw ("Hello World!");

provocará que se muestre el siguiente error de excepción no controlada:

No obstante, puede hacerse cargo de este mensaje de error y utilizar su propio código de gestión llamando a la función la función exception_unhandled_handler(). Esta función de tiempo de ejecución le permite suministrar un código personalizado method que se llamará cada vez que se produzcan excepciones no controladas en el juego.
