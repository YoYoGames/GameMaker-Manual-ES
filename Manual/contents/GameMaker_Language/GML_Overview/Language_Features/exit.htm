salir

La declaración exit tiene la siguiente sintaxis:

exit;

exit simplemente termina la ejecución de la función actual del script, methodo event. Tenga en cuenta que hay una ligera diferencia en el uso aquí dependiendo del ámbito:

  • Si utiliza exit en una función o método de script, simplemente saldrá de la función y volverá al código que llamó a la función.
  • Si utiliza exit en un bloque de código dentro de un evento de un objeto, saldrá de todo el evento aunque haya varios bloques de código separados después de llamar a exit.
  • Si usa exit en un evento padre y llama a ese evento usando event_inherited() en un evento hijo object, sólo el evento padre saldrá y el evento hijo (que llamó a event_inherited()) continuará.

Cuando se utiliza en un evento, exit se usa típicamente para evitar que una instancia ejecute más código cuando una condición específica se ha cumplido (o no). El código que se muestra a continuación es un ejemplo de cómo se podría utilizar, en este caso dentro de un evento de colisión, aunque se puede utilizar en cualquier evento.

if (!visible)
{
    exit;
}

other.hp -= attack;
other.coins -= 4;
coins += 4;

El código anterior comprueba si la instancia actual no es visible, en ese caso sale del bloque de código, en caso contrario sigue adelante y ejecuta el resto del código.

NOTA: No termina la ejecución del juego. Para ello es necesario utilizar la función game_end().
