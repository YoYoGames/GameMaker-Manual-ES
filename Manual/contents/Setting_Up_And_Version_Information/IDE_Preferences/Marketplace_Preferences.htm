Preferencias del mercado

Las Preferencias del Mercado se utilizan para configurar el Certificado del Mercado para cargar o comprar assets. El formulario del certificado consta de las siguientes secciones para cuando ya haya creado un certificado (véase más abajo en esta página para saber cómo crear un certificado):

  • Archivo de certificado: En esta sección se especifica el archivo *.pem que el Marketplace utiliza para la certificación de assets. Si tiene un certificado previamente creado, puede hacer clic en el botón del explorador de archivos y añadirlo al foro, pero si no tiene un certificado y desea crear uno, debe omitir cualquier información aquí y en su lugar hacer clic en el botón Crear nuevo certificado (explicado a continuación).
  • Contraseña del certificado: Aquí debe introducir la contraseña del certificado para el archivo seleccionado anteriormente. La contraseña se crea cuando se crea por primera vez el archivo *.pem (véase Crear un nuevo certificado, más adelante) y se vinculará de forma irreversible al archivo de certificado creado.
  • Comprobar: Una vez que haya rellenado los campos anteriores, debe hacer clic en este botón para asegurarse de que la contraseña y la selección de archivos son correctas y funcionan juntas.

Confirmar diálogos

Aquí puede seleccionar el comportamiento de los diálogos de confirmación cuando se trabaja con un recurso desde el Navegador Asset. Las opciones disponibles son:

  • Respuesta automática a la eliminación de contenidos sin conexión: Cuando borras un marketplace asset de tu biblioteca sin conexión a la web, normalmente recibes una advertencia preguntando si estás seguro de que quieres proceder. Al establecer esta opción en "Aceptar" el mensaje se suprimirá como si hubieras pulsado el botón "Aceptar" para proceder. El valor por defecto es "Mostrar mensaje".

Crear un nuevo certificado

Si no ha creado previamente un archivo de certificado para utilizar la página web Marketplace, deberá hacer clic en el botón denominado Crear nuevo certificado que se encuentra en la parte inferior de este formulario. Al hacerlo, se abrirá una nueva ventana en la que se le pedirá que que introduzca la siguiente información:

  • Nombre del editor: Este es el nombre que se utilizará como nombre del editor para asset. En el panel de control de Marketplace puede configurar diferentes editores si lo desea y cada uno debe tener su propio archivo *.pem correspondiente.
  • Contraseña del certificado: Aquí debe dar una contraseña única para el archivo del certificado que se va a crear. Tenga cuidado con lo que elija ya que no podrá cambiarlo más tarde y si no tiene la contraseña correcta para el archivo del certificado en las secciones anteriores no podrá actualizar ningún Marketplace assets existente (ni cargar nuevos).

Con estos datos, puede hacer clic en el botón Crear certificado para generar el archivo *.pem. Se le pedirá que indique una ubicación de almacenamiento y un nombre para el archivo, y deberá tomar nota de ello para futuras referencias. Una vez hecho esto, el nuevo archivo y la contraseña se añadirán automáticamente en los campos del formulario que aparecen más arriba y podrá hacer clic en el botón Comprobar para comprobar que el certificado se ha generado correctamente.

NOTA: Le recomendamos que haga una captura de pantalla de los ajustes reales, así como una copia del archivo *.pem . Ambos deben ser guardados juntos en una ubicación segura para futuras referencias, o en caso de que necesite volver a instalar GameMaker en algún momento. No puedes actualizar assets que ya ha sido cargado a menos que el paquete actualizado esté firmado usando el mismo certificado con el que se creó el original.
