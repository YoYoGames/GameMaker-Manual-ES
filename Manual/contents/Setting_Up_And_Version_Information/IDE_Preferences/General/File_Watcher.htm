Preferencias de File Watcher

El observador de archivos es el nombre que recibe la parte de IDE que vigila los archivos relacionados con un proyecto para detectar cualquier cambio debido a software externo (por ejemplo, la edición de una imagen en un editor externo) o debido a cambios internos como la edición de un script asset . En general, estos ajustes no deberían necesitar ser editados, pero en algunos sistemas - como los que utilizan un SSD como unidad de almacenamiento para los archivos del proyecto - el vigilante de archivos puede informarle de los cambios cuando hace cosas como simplemente guardar el proyecto o cada vez que edite cualquier proyecto asset, en cuyo caso es posible que tenga que cambiar estos ajustes.

Los ajustes disponibles son:

  • Habilitar Vigilancia de Archivos: Se puede utilizar para activar o desactivar el sistema de vigilancia de archivos. Si se desactiva, IDE dejará de detectar los archivos modificados por fuentes externas, por lo que hay que tener cuidado al desactivar esta función. El vigilante de archivos está activado por defecto.
  • Ajustar la tolerancia del File Watcher en MS después de guardar En el IDE: Esto ajusta el tiempo entre guardar un archivo y que el File Watcher compruebe los cambios. Usted ajustaría este valor si estuviera guardando su proyecto y el File Watcher estuviera aparece erróneamente diciendo que algo ha cambiado, en cuyo caso aumentaría el retraso hasta que el mensaje no aparezca más. El valor por defecto es de 2000ms, pero puede ir desde 1ms hasta 10.000ms.
