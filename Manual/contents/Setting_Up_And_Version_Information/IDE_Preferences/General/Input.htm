Preferencias de entrada

Las preferencias deentrada controlan ciertos aspectos de la entrada del ratón y del teclado dentro de IDE. Las opciones disponibles son:

  • Tiempo de doble clic lento (ms): Algunas áreas de IDE requieren un doble clic más lento para funcionar correctamente, por ejemplo al renombrar un recurso (un doble clic rápido abrirá el editor de recursos, mientras que un doble clic lento le permitirá renombrarlo). Esta opción permite establecer la velocidad de detección, con un valor por defecto de 500ms.
  • Tiempo de bloqueo de desplazamiento (ms): Esta opción se ocupa del tiempo que GameMaker debe esperar entre las ventanas de desplazamiento que se superponen. Esencialmente, cuando usted está desplazando en la ventana "A", el cursor podría pasar por encima de otra área desplazable en la ventana "B", por lo que esta preferencia define el tiempo que tiene que esperar después de dejar de desplazar "A" antes de que pueda desplazar "B", para que no comience accidentalmente a desplazar diferentes paneles. El valor por defecto es de 500ms.
  • Envolver el cursor alrededor de la pantalla cuando se hace un paneo: Cuando se habilita, el cursor se envuelve alrededor de los límites de la pantalla cuando se hace un paneo workspaces. Por defecto está activado.
  • Velocidad de desplazamiento: Controla la velocidad de desplazamiento dentro de los diferentes workspaces. El valor por defecto es 1.
  • Distancia de arrastre de la pestaña para desmaximizar el escritorio (px): Cuando arrastra una pestaña de workspace fuera de su posición por defecto dentro de IDE, si sólo hay un workspace activo en ese momento, la ventana de GameMaker se des-maximizará y se moverá con el ratón (si tiene más de un workspace activo se creará una nueva instancia de IDE para mantener el workspace). Este ajuste controla la distancia que tiene que arrastrar la pestaña antes de que el workspace la reconozca, y el valor por defecto es de 50px.
  • Nivel mínimo de zoom para interactuar con el contenido de las ventanas (%): Esta opción controla el punto de corte del zoom a partir del cual el ratón ya no podrá interactuar con las ventanas de workspace (es decir, añadir nodos, arrastrar ventanas, etc...). El valor por defecto es 40.
  • El zoom con la rueda del ratónrequiere la tecla Ctrl: Cuando esta opción está marcada, todos los editores (y el principal workspace) tendrán el zoom habilitado con / + la rueda del ratón , mientras que al desmarcarla se habilitará el zoom sólo con la rueda del ratón Icono MMB.
  • Ctrl + MMB para abrir el evento heredado en lugar del manual: Cuando esta opción está marcada, manteniendo / + el botón central del ratón en un object se abrirá el evento heredado objects (si tiene uno) en lugar del manual.
  • Añade un sesgo a la selección de texto para evitar saltos de línea: Cuando esta opción está marcada, la altura de la línea concebida mientras se selecciona una sola línea en cualquier ventana de texto será rellenada, haciéndola menos propensa a la imprecisión vertical. Esta opción está activada por defecto.
  • Habilitar el modo portátil en todos los dispositivos: Al marcar esta opción se mostrará el botón de modo portátil en la página IDE, lo que le permitirá a switch activarlo o desactivarlo según sea necesario, independientemente de que el dispositivo que ejecute GameMaker sea un portátil o no. Esta opción está desmarcada por defecto.
  • Forzar el modo portátil: Este ajuste cambia si el Modo Portátil es utilizable o no, y está activado por defecto. Puede desactivar esta opción para asegurarse de que el modo portátil no se active automáticamente cuando inicie la página IDE en un ordenador portátil.
