Preferencias de alimentación en tiempo real

Las preferencias del Runtime Feeds se utilizan para mostrar y cambiar el Runtime actualmente instalado, así como el runtime en uso. GameMaker puede tener varias herramientas diferentes de runtime instaladas en cualquier momento y usted puede establecer cuál usar desde esta preferencia. Lo ideal es utilizar el runtime más actualizado, pero a veces puede tener un problema, o puede ser necesario utilizar uno más antiguo para la comparabilidad, etc ... y por lo que puede cambiar las herramientas runtime que GameMaker utiliza para construir su juego aquí.

La sección inicial de estas preferencias está relacionada con la procedencia de las herramientas de runtime y requiere que se introduzca la URL de un canal RSS. Esta fuente se escaneará en busca de actualizaciones de las herramientas de runtime, que podrán descargarse e instalarse. Esta sección mostrará la URL de la fuente de runtime por defecto, pero puede hacer clic en el botón Añadir para crear una nueva fuente de Runtime en la que debe indicar el nombre y la URL de la fuente RSS y, a continuación, hacer clic en Aplicar para indicar a GameMaker que acceda a ella. Tendrá que cerrar y volver a abrir la ventana de Preferencias para que se muestre el nuevo canal.

Una vez que se ha creado un feed de Runtime, aparecerá en esta sección como una subcategoría, y siempre debería tener al menos un feed en la lista (por defecto sería el feed principal ). Las preferencias de Runtime Feed se utilizan para seleccionar, instalar y desinstalar las diferentes herramientas de runtime. En la parte superior tienes las herramientas de runtime que se están utilizando, la fecha en la que se crearon y la ubicación en la que están almacenadas en el disco duro. Debajo de estos detalles puedes encontrar una lista de todos los runtimes disponibles, y puedes elegir cualquiera de esta lista para cambiar a ser el runtime "en ejecución" utilizado para compilar tus proyectos.

Para seleccionar un runtime diferente para usar, simplemente búsquelo en la lista y luego haga doble clic en . Si el runtime no ha sido descargado e instalado, se le pedirá que lo haga, y una vez instalado se le pedirá que cierre y reinicie el IDE para que el nuevo runtime pueda ser inicializado.

Esta lista detalla todos los runtimes disponibles, su número de versión, la fecha en que se publicaron y, si están instalados, el path en el que se han instalado. La parte derecha también muestra los diferentes iconos de estado, que son los siguientes:

Esto marca el runtime actual que ha sido seleccionado para su uso al construir sus proyectos.
Al hacer clic, se iniciará la descarga e instalación del runtime seleccionado (se mostrará una barra de estado en la parte superior derecha del IDE).
Al hacer clic en esta opción, se eliminará la dirección runtime seleccionada del disco duro.

Puede actualizar la lista de runtime en cualquier momento desde la fuente RSS dada haciendo clic en el botón de actualización , y puede limpiar su disco duro de viejos runtimes haciendo clic en el botón "Limpiar Runtimes ahora".
