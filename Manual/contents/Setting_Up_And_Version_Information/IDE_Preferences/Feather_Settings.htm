Opciones de base

La configuración básica de la pluma contiene las siguientes opciones:

  • Activar Feather: Activa o desactiva Feather aquí. Si se desactiva, las funciones de Feather no estarán disponibles en IDE.
  • Retraso en el análisis del documento: La cantidad de tiempo que Feather espera para comprobar su script después de hacer un cambio en él.
  • Compactar información sobre herramientas: Al activar esta opción, se eliminará cierta información sobre las herramientas que aparece al pasar el ratón por encima de las palabras clave, como las descripciones de los parámetros de las funciones, lo que hará que la información sobre las herramientas sea más pequeña.
  • El número máximo de campos de struct que se muestran mediante tooltips: Al pasar el ratón por encima de una referencia struct, una información sobre herramientas muestra las variables dentro de ese struct, hasta el número introducido aquí.
  • Refactorizar las referencias al nombre asset cuando se renombra asset en el navegador Asset: Activando esta opción te aseguras de que cuando cambias el nombre de un asset en tu navegador Asset, cualquier mención a él en tu código también se edita para que esas referencias no se rompan.
  • Preferir el estilo de comentario JSDoc multilínea al crear desde Quick Fix: Cuando se generan comentarios JSDoc para una función a través del menú Quick Fixes, el JSDoc generado utilizará comentarios de una o varias líneas, dependiendo de esta opción.
  • Activar el modo de tipo estricto: Cuando se activa esta opción, Feather es más estricto a la hora de asegurarse de que se utiliza la dirección data types correcta en el código.

Destacando

Esta sección contiene ajustes para cambiar los colores utilizados para subrayar parte de su código para marcar un error, advertencia o sugerencia:

  • Color del error: El color utilizado para los errores.
  • Color de la advertencia: El color utilizado para las advertencias.
  • Color de la sugerencia: El color utilizado para las sugerencias.

Gravedad del mensaje

Esta sección contiene todas las reglas que Feather utiliza para comprobar su código, y le permite ajustar la gravedad de cada regla.

Perfil

En este cuadro desplegable puede elegir entre un par de preajustes para las Reglas GM*. Están disponibles los siguientes perfiles:

  • Ninguno: Este perfil no mostrará ningún mensaje de error.
  • ErroresSintácticos: Si utiliza este perfil, se mostrarán los errores de sintaxis, junto con lo que se muestra en el perfil Ninguno.
  • TypeErrors: Este perfil también muestra errores de tipo además del perfil SyntaxErrors .
  • Todo: Este perfil muestra todo lo que Feather puede analizar.

Reglas GM*

Puede establecer una regla como "Error", "Advertencia" o "Sugerencia", lo que afecta a la forma en que el código que rompe la regla se le informa en el Editor de Código y en la ventana de Mensajes de Pluma.

Puede establecer una regla como "Ignorar", lo que hará que se ignoren todos los casos en los que se infrinja esa regla.

Todas las reglas que empiezan por "GM1000" comprueban su sintaxis en busca de posibles errores fatales, y las que empiezan por "GM2000" son reglas de buenas prácticas que le ayudan a prevenir errores en su juego.

Normas de denominación

Esta sección le permite establecer reglas de nomenclatura para assets, variables y otras partes de GML Code.

La regla GM2017 en "Gravedad del mensaje" debe estar activada para que las reglas de nomenclatura tengan efecto. Por defecto, está desactivada.

Los ajustes de esta sección son los siguientes:

  • Lista de bloqueo de identificadores: Se trata de una lista separada por espacios de identificadores que serán ignorados para todas las reglas de nomenclatura.

    Por ejemplo, digamos que tienes una regla para usar obj_ como prefijo para objects, pero quieres que tu gestor y red objects permanezcan como están y no usen el prefijo obj_.

    En ese caso, escribirías manager network en el campo Indentifier Blocklist. Feather ignorará esos objects y cualquier otro identificador (variables, enums, parámetros, etc.) con los mismos nombres.
  • <identificador> Regla de nomenclatura: Puede establecer reglas de nomenclatura para cada tipo de identificador de la lista, como assets, macros, enums, parámetros de funciones, variables, etc.

    Cada desplegable de reglas de nomenclatura tiene las siguientes opciones:
    • Prefijo: Texto que debe aparecer antes del nombre del identificador, que puede ser obj_ para objects, _ para variables locales, etc. dependiendo de sus preferencias.
    • Sufijo: Texto que debe aparecer después del nombre del identificador.
    • Estilo de nomenclatura: Elige el estilo de nomenclatura para el identificador. Las opciones son:
      • Sin restricciones: No se fuerza ningún formato en este tipo de identificador.
      • UpperCamelCase: MiSalud, ObjGrappleHook, MaxInventoryItemsAllowed, etc.
      • lowerCamelCase: myHealth, objGrappleHook , maxInventoryItemsAllowed, etc.
      • ALL_UPPER: MY_HEALTH , OBJ_GRAPPLE_HOOK, MAX_INVENTORY_ITEMS_ALLOWED, etc.
      • all_lower: my_health, obj_grapple_hook, max_inventory_items_allowed, etc.
