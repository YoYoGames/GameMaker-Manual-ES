Preferenciasvisuales de GML

La página web GML Visual Preferencias se utilizan para definir el aspecto y la sensación del editor Visual. Las diferentes opciones disponibles se enumeran a continuación junto con una descripción de lo que hace cada una.

  • Ubicación por defecto de la caja de herramientasGML Visual: Por defecto, la caja de herramientas de GML Visual (donde se encuentran las diferentes librerías de acciones) se sitúa a la derecha del Editor, pero puede utilizar esta opción para que se sitúe por defecto a la izquierda.
  • Modo de visibilidad del nodo GML Visual: Un "resumen" de nodo es el breve texto descriptivo que aparece junto a ciertas acciones en la Acción Workspace, por ejemplo esta acción de Destruir Instancia:

    Puede elegir entre activarlas siempre, activarlas sólo si la acción está colapsada (la configuración por defecto) o desactivarlas por completo.
  • Respuesta automática a la conversión de GML Visual a GameMaker Language: Cuando conviertes unGML Visual evento o script en código GML, por defecto aparecerá un mensaje advirtiéndole de lo que va a hacer y preguntándole si quiere continuar. Si se ajusta esta opción a "OK", simplemente se aprobará el cambio sin mostrar el mensaje.
  • Respuesta automática a la eliminación de nodos de GML Visual: Cuando elimine un nodo de GML Visual nodo de evento, por defecto aparecerá un mensaje advirtiéndole de lo que va a hacer y preguntándole si quiere continuar. Si se establece esta opción en "Aceptar", simplemente se aprobará el borrado sin mostrar el mensaje.
  • Tamaño máximo de las miniaturas de las acciones de GML Visual: En la parte inferior de la caja de herramientas de acción tiene un control deslizante para cambiar el tamaño de los GML Visual iconos. Este ajuste se utilizará para definir el tamaño máximo permitido (por defecto 64px).
  • Tamaño mínimo de las miniaturas de las acciones de GML Visual: En la parte inferior de la caja de herramientas de acción tiene un control deslizante para cambiar el tamaño de los GML Visual iconos. Este ajuste se utilizará para definir el tamaño mínimo permitido (por defecto 16px).
  • Tamaño por defecto de las miniaturas de las acciones de la caja de herramientas de GML Visual: Este ajuste se utilizará para definir el tamaño por defecto de todos los iconos de acción de la caja de herramientas (por defecto 32px).
  • Número máximo de acciones utilizadas recientemente para mostrar: En la caja de herramientas de acciones hay una biblioteca en la parte superior que contiene las acciones que has utilizado más recientemente. Puedes utilizar esta opción para cambiar el número máximo de acciones recientes que se mostrarán (por defecto 5).
  • Grosor de la cadena que conecta las acciones de GML Visual: Define el grosor del gráfico de la cadena que une las diferentes acciones en un GML Visual bloque de código (por defecto 3px).
  • Tiempo de bajada del ratón para empezar a arrastrar nodos (ms): Cuando haces clic en un nodo y lo arrastras, estarás reposicionando el nodo dentro del código de la acción. Sin embargo, si mantienes el ratón pulsado durante un momento y luego te mueves, estarás moviendo la posición del nodo dentro de la workspace pero no dentro de la orden del código. Este ajuste establece el número de milisegundos que hay que esperar con el botón del ratón pulsado antes de que el nodo pueda moverse (no reposicionarse). El valor por defecto es 350.
  • Ocultar el panel de visión general del nodo GML Visual: Aquí puede elegir ocultar el panel de visión general del Editor de Código de Acción. Por defecto está desactivado y al activarlo se eliminará la lista de texto plano de las cadenas de acción del editor.
