Preferencias delcódigo GML

Las Preferencias de Código GML se utilizan para definir cómo se mostrarán algunas de las características del lenguaje para la codificación en el IDE de GameMaker. Las opciones para el Código GML son las siguientes:

  • Mostrar los tipos de argumentos en la barra de estado: Esta opción se refiere a cuando está utilizando su propio scripts personalizado y ha utilizado la notación JSDoc para identificar los diferentes argumentos, etc... (ver aquí para más información). Cuando esta opción está marcada, y ha especificado los tipos de argumentos en script, entonces se mostrarán en la barra de estado en la parte inferior del editor, encerrados en {}. Esta opción está desactivada por defecto.
  • Mostrar las descripciones de los argumentos en la barra de estado: Si esta opción está marcada, las descripciones que haya dado a su scripts personalizado utilizando la notación JSDoc se mostrarán en la barra de estado en la parte inferior de la ventana del editor. Esta opción está desactivada por defecto.
  • Mostrar la ortografía estadounidense para las funciones del sistema: Por defecto GameMaker utiliza la ortografía inglesa para las funciones, por ejemplo draw_text_colour(), sin embargo esto puede no ser exactamente lo que usted está acostumbrado y por lo que puede switch a la ortografía de EE.UU. utilizando esta opción, por lo que las funciones se convierten en draw_text_color(), por ejemplo. Esta opción está desactivada por defecto.
  • Número máximo de errores de sintaxis a mostrar: Este valor establece el número máximo de errores de sintaxis que se mostrarán en un momento dado. Si encuentra que IDE se retrasa, puede intentar reducir este valor. Por defecto, el valor es 50.
  • Habilitar las advertencias de referencias a variables: Si se activa, se mostrará en flag cualquier problema potencial con las variables, como cuando una se ha definido en algún lugar pero no se utiliza realmente en el código. Si se desactiva, estas advertencias no se mostrarán. Por defecto, esta opción está activada.
  • Habilitar el plegado de sentencias como regiones: Cuando se activa, los paréntesis en su código son capaces de ser doblados como regiones. Esta opción está activada por defecto.
