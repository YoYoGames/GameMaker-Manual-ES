Objetos e instancias

Con el assets que has visto hasta ahora puedes añadir algunas imágenes y sonidos bonitos al navegador del juego asset, ¡pero todavía no hacen nada en el juego en sí! Para eso necesitamos el assets más importante de todos en GameMaker, el Object asset. Objects son un asset especial que usamos para controlar aspectos de un juego y para hacer cosas específicas. La mayoría de las veces tienen un sprite asociado con ellos para que puedas verlos en el juego rooms, pero a veces se utilizan como objetos controladores "detrás de las escenas" para hacer cosas como el seguimiento de las estadísticas, o establecer los tiempos para las escenas de corte, etc... Objects se les puede dar comportamientos y pueden reaccionar a ciertos eventos, así como entre sí, y la mayoría de las cosas que se ven en un juego se basan en objects y sus interacciones con el usuario y entre sí.

Ahora, no podemos hablar de Objects sin hablar de las Instancias. En un juego hecho con GameMaker, los personajes, monstruos, bolas, paredes, etc... que se ven en el juego son en realidad todos considerados como instancias. Una instancia es esencialmente una "copia" de un object que se encuentra en el Navegador Asset y luego se coloca en un room (piense en objects como plantillas para instancias). Así, cuando hablamos de que algo afecta o cambia una instancia, nos referimos a que una copia concreta de un object en un room se ve afectada mientras que el resto no, pero cuando hablamos de afectar o cambiar un object nos referimos a que cualquier cosa que hagamos en él se reflejará también en todas las instancias creadas a partir de ese momento. Así, el object es la plantilla de la instancia, y la instancia es lo que colocamos en un room para hacer nuestro juego.

Para crear un nuevo object asset, es lo mismo que para cualquier otro asset y simplemente se hace clic con el botón derecho del ratón en en la carpeta Objects asset y se selecciona Crear (o se hace clic con el botón derecho del ratón en cualquier lugar del Navegador de Activos y se selecciona Crear Objeto).

Esta es la ventana del Editor de Objetos y muestra las propiedades actuales de object, y lo primero que debe hacer es nombrar el object, de nuevo usando un prefijo como " obj_" o algún otro identificador. En general, para que un object dibuje algo y registre colisiones con instancias de otros objects necesita tener un sprite asignado, que se añade usando el botón . Esto abrirá el Explorador de Activos donde puedes seleccionar el sprite a utilizar (también puedes arrastrar sprites desde el Navegador Asset y soltarlos en el Editor Object para asignarlos):

Si todavía no tiene un sprite apropiado, puede hacer clic en en el botón Nuevo Sprite para crear un nuevo sprite asset y editarlo como desee, igual que si hubiera creado el asset a través de las opciones estándar de creación de sprite, sólo que ahora se asignará automáticamente al object que tenía abierto cuando empezó a crearlo. Una vez que seleccione un sprite también puede hacer clic en los botones Editar Sprite o Editar Imagen para editar las imágenes del propio sprite o del sprite asset. Esto puede ser más rápido que encontrar primero el sprite en el Navegador Asset y luego indicar que quiere editarlo.

Una vez que hayas añadido o creado el sprite, puedes cambiar algunas otras propiedades de object como si debe ser visible o no - por ejemplo - o qué tipo de máscara de colisión debe tener. Para juegos sencillos probablemente no necesites cambiar nada aquí y ahora puedes pasar a añadir Eventos al object.

Los eventos son momentos discretos en el juego loop donde se hace que las cosas sucedan en base a lo que has programado para ellos. GameMaker trabaja con ciclos de estos eventos - desde el momento en que se inicia un room hasta el momento en que se termina hay un juego loop en marcha donde cada paso se ejecuta una serie de eventos, y usted puede elegir colocar código GML o acciones de GML Visual en sus instancias que responden a estos eventos (un paso es un momento en el tiempo del juego, gobernado por la configuración de la velocidad del juego - también llamado un "marco").

Para añadir un evento, haga clic en en el botón Añadir evento:

Arriba puede ver todas las diferentes categorías de eventos, algunas de las cuales contienen otras subcategorías. No vamos a repasar aquí todos los eventos, pero sí cubriremos brevemente los cinco más importantes:

Crear evento Este evento ocurre cuando se crea por primera vez una instancia de object, y es lo primero que ocurre dentro de una instancia que ha sido colocada en room a través del editor room cuando se introduce un room. Esto significa que este evento es el lugar ideal para inicializar variables, iniciar Timelines, establecer paths etc... y hacer cualquier otra cosa que generalmente sólo necesita hacerse una vez o sólo cuando una instancia aparece por primera vez en el room.
Evento del paso GameMaker divide el tiempo del juego en pasos con la velocidad del juego definiendo cuántos de estos pasos se supone que hay por segundo. Un paso, es básicamente el loop que se ejecuta constantemente con todos los eventos que se comprueban y desencadenan como sea necesario mientras el juego se ejecuta, así que como se puede imaginar, el Evento de Paso es un evento que se comprueba cada paso del juego mientras la instancia existe. Ten en cuenta que la categoría de Evento de Paso tiene tres subcategorías para "afinar" el tiempo dentro de la instancia, pero en general usarás el Evento de Paso principal más.
Evento de colisión Obviamente, al hacer un juego, es muy importante que sepas cuando dos (o más) instancias de un object han colisionado, y para ello tenemos el evento de colisión. Este es un evento que colocas en un object y luego especificas contra qué otro object debes comprobar si hay colisiones. Una vez que hayas añadido el evento y elegido el object para comprobarlo puedes añadir código GML o GML Visual para resolver la colisión.
Evento de sorteo Hay varios eventos de desenfundado dentro de la categoría de eventos de desenfundado, pero el que más se utiliza es el que está en la parte superior llamado simplemente "Desenfundado". Este evento general de sorteo será llamado, como el evento de paso, en cada cuadro de juego, pero funciona de dos maneras diferentes. La primera es cuando no añades este evento al object... Cuando no hay un evento de dibujo especificado , entonces GameMaker dibujará por defecto el sprite asignado al object, usando cualquier transformación que haya sido aplicada en otra parte del object (como para cambiar su tamaño o mezcla alfa), así que incluso cuando no hay un evento de dibujo en la lista de eventos, todavía se está ejecutando y todavía dibujará en la pantalla cuando una instancia es colocada en un room. La segunda forma en que funciona es cuando usted agrega el evento mismo e incluye cualquier GML o GML Visual en él. Agregar algo al evento le dice a GameMaker "Quiero controlar lo que se dibuja para las instancias de este objeto" y anula completamente el dibujo por defecto. Esto significa que usted puede tener un object con un sprite asignado, luego establecer el evento draw para dibujar texto y el sprite no será dibujado ya que usted no le ha dicho a GameMaker que lo dibuje, aunque el texto sí.
Destrucción del evento Este evento es el último que se ejecuta cuando se destruye una instancia. A menudo se pasa por alto cuando se añaden comportamientos a objects, pero puede ser muy útil - por ejemplo, para crear efectos de explosión o de partículas cuando se mata a un enemigo, o para reaparecer una nueva instancia de object en otra parte de room, o incluso para añadir puntos a una puntuación.

Para más información sobre los diferentes eventos, consulte aquí.

A continuación, puede hacer clic en un evento de la lista para añadirlo a la página object.

Al añadir un evento se abrirá el editor GML Code o GML Visual, dependiendo del tipo de evento que estés creando. En esta ventana puedes añadir el código o las acciones que deseas que se realicen cada vez que ese evento se active dentro del juego loop.
