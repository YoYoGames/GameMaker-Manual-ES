Lenguaje GameMaker

Puedes usar GameMaker Language para programar tus juegos de dos maneras diferentes: colocando bloques para "escribir" código visualmente, o escribiendo código manualmente.

La primera opción, GML Visual, es la más adecuada para los principiantes y los aficionados que quieren empezar a crear sus propios juegos rápidamente.

La segunda opción, Código GML, le permite aprovechar toda la potencia de GameMaker Language escribiendo su código manualmente.

Mientras que GML Visual es más fácil de empezar, GML Code sigue siendo un lenguaje de programación fácil de usar con potentes características(Funciones, Métodos, Estructuras y Constructores, etc.).

Creación de scripts visuales/de código

Se le pide que elija entre GML Code y GML Visual al crear un nuevo evento Object, un nuevo Script asset, o cualquier otra cosa que resulte en la creación de un script:

Además de poder elegir entre GML Code y GML Visual, esta ventana contiene dos casillas de verificación:

  • No preguntar de nuevo paraeste proyecto: Cuando esta opción está activada, establece la Opción de Juego "Lenguaje de scripting por defecto" a su nueva selección, por lo que no se le pide que elija de nuevo para el proyecto actual.
  • Nopreguntar nunca más: Cuando esta opción está activada, establece la Preferencia "Lenguaje de scripting por defecto" con su nueva selección, de modo que nunca se le pedirá que elija de nuevo, para ningún proyecto.
