Conflictos

En esta sección vamos a ver cómo resolver conflictos entre un repositorio local clonado y un repositorio maestro. Asumiendo que ahora está usando un repositorio local, necesitaremos editar un archivo y luego causar algún tipo de conflicto, así que abra la prueba script que hemos estado usando y cambie el mensaje "Hola Mundo" a algo como esto:

show_debug_message("Hello World, How Are You?");

Ahora deberías guardar el proyecto y confirmarlo en el repositorio local. Ahora, como estamos enlazados a un repositorio maestro remoto (aunque sea uno que hayamos clonado en el mismo disco) necesitamos hacer un Push para que los cambios lleguen al maestro. Para ello, vamos a Source Control > Push Changes, que nos dará una actualización del estado en la pestaña Source Control Output:

Ahora necesitamos abrir nuestro proyecto original (no el clonado), así que busque donde lo guardó en el disco - o simplemente compruebe los Proyectos Recientes en el menú Archivo, ya que debería estar directamente debajo del actual - y cárguelo en GameMaker. Una manera fácil de comprobar que tienes el proyecto correcto es simplemente comprobar el script que tienes y si el mensaje es "Hola Mundo" entonces tienes el correcto, o puedes abrir la ventana de Ver Historia y comprobarlo de esa manera.

Ahora tienes que volver a editar el script, para que quede así:

show_debug_message("This will cause a conflict.");

Esto puede ser guardado en el disco, y los cambios confirmados, pero si usted trata de empujar estos cambios obtendrá el siguiente mensaje:

¡Alguien ya ha empujado los cambios que usted no tiene todavía! Esto significa que necesita hacer un pull en el repositorio y ver cuál es exactamente el problema, así que vaya a Control de Fuentes > Pull Changes. Esto mostrará dos ventanas, la primera es una advertencia sobre los ficheros cambiados (como con las acciones anteriores, querrá elegir Recargar aquí y no Guardar), y la otra es la ventana de Conflictos :

Esta ventana mostrará los archivos en conflicto en una lista a la izquierda, y puede seleccionar los archivos de esa lista utilizando el botón izquierdo del ratón (o seleccionar varios archivos utilizando / o junto con ). Tiene cuatro opciones para tratar los archivos en conflicto, disponibles desde los botones de la derecha:

  • Usar el suyo - Esto sobrescribirá los cambios que haya hecho con el del repositorio remoto.
  • Usar el mío - Esto anulará los cambios del repositorio remoto con los que acaba de hacer.
  • Combinar - Esto intentará abrir una Herramienta de Combinación Externa para manejar el conflicto.
  • Elija la herramienta de fusión - Esto le permitirá configurar la herramienta de fusión (véase la sección sobre herramientas externas de fusión/difusión para más detalles).

Ahora, puede utilizar los botones para resolver el conflicto, pero antes de hacerlo, vamos a abrir el archivo en conflicto para ver exactamente cuál es el problema. Si abres el script que está en conflicto, se verá así:

Esto muestra que "This Will Cause A Conflict" está en el HEAD (esto es lo que hemos empujado al repositorio local) y que "Hello World, How Are You?" está siendo extraído de master, el repositorio remoto. Ahora puede volver a la ventana de conflictos y pulsar en Usar el suyo para extraer el fichero cambiado y sobrescribir el fichero HEAD actual. El script se marcará ahora como cambiado y deberá hacer un nuevo commit y push antes de continuar.

Tenga en cuenta que en realidad podríamos arreglar esto en el GameMaker IDE. Si eliminamos las líneas 1, 2, 3 y 5 para que nos quedemos sólo con show_debug_message("Hello World, How Are You?") y lo guardamos, se eliminará el conflicto, ya que Git asumirá que sabe lo que está haciendo con su propio archivo. Sin embargo, esto es propenso a errores, por lo que se recomienda instalar una herramienta de fusión, luego configurarla y usarla en lugar de intentar fusionar manualmente.
