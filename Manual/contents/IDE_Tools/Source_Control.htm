Control de la fuente

Lagestión del control de fuentes (SCM) es el nombre que recibe el método para trabajar con subversiones o copias de seguridad de tus proyectos a través de un repositorio y una fuente local. Básicamente, una solución SCM es un paquete de software independiente que controla todos los aspectos del mantenimiento, el cambio y la comparación de versiones de tu proyecto a medida que trabajas en él. Esto es especialmente útil para aquellos que trabajan en equipo y necesitan poder controlar quién hace qué y no preocuparse por la pérdida de datos o la realización de cambios que deban deshacerse más adelante, pero los individuos también pueden beneficiarse de este sistema potente y flexible.

GameMaker utiliza Git para su SCM, que debe instalarse y configurarse por separado antes de poder utilizar el SCM en GameMaker. Consulte la documentación de Git para obtener más información específica sobre Git.

A continuación puedes encontrar enlaces a un pequeño tutorial sobre cómo configurar este plugin y utilizar las herramientas SCM con un proyecto.

GameMaker mostrará un menú desplegable en la barra de menús para Source Control, que se llenará con las siguientes opciones una vez que Source Control haya sido activado para su proyecto (vea la sección Configuración del plugin Git más arriba para más detalles):

A continuación, resumimos cada una de las opciones disponibles (la mayoría de ellas se explican con mayor profundidad en las secciones del tutorial anteriores):

  • Crear Repositorio del Proyecto: Esta opción permite crear un repositorio local en el mismo directorio que los archivos del proyecto.
  • Clonar Repositorio: Esta opción le permitirá clonar un repositorio desde un origen a un nuevo destino.
  • Confirmar cambios: Con esta opción puede escenificar los archivos modificados en su proyecto y luego confirmarlos en el repositorio.
  • Empujar cambios: Después de hacer un commit o un merge esta opción se utiliza para empujar los cambios al repositorio maestro.
  • Tirar delos cambios: Con esta opción puede actualizar el repositorio local tirando de los archivos modificados del repositorio maestro.
  • Ver el historial: Esto abrirá la ventana del historial donde puede ver todo el historial de versiones del proyecto y elegir retroceder archivos específicos paths o revisiones completas.
  • Mostrar Conflictos: Esto abrirá la ventana de Conflictos y listará cualquier archivo conflictivo que pueda existir en el proyecto, permitiéndole tratar con ellos a través de GameMaker IDE o usando una Herramienta de Fusión específica.
