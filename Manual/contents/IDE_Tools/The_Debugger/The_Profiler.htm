El perfilador

La página Debugger te permite recorrer tu código y encontrar cualquier problema con la base de código, pero a veces necesitas comprobar detalles más sutiles para optimizar adecuadamente tus juegos. La opción Perfil te ayuda a hacer eso, ofreciéndote una visión general del rendimiento, mostrando el tiempo que se tarda en llamar a las funciones, scripts, e incluso el tiempo que tarda un evento específico en completarse.

Aparte de la ventana principal que muestra todos los diferentes eventos, scripts y las funciones que se llaman, también tiene las siguientes opciones para utilizar el Profiler:

Activar / Desactivar

Este es el botón para activar o desactivar el perfilador. Cuando esté habilitado, el perfilador se iniciará y empezará a ver los eventos, scripts y las funciones que se enumeran en la ventana principal junto con los diferentes valores de datos para cada uno de ellos. La forma en que se muestran estos datos dependerá de la visualización del tiempo y del modo de visualización (ambos se explican con mayor profundidad más adelante).


Pantalla de la hora

Esta casilla le permite switch entre el tiempo medio y el tiempo absoluto para el perfilador. Cuando no está marcada, los valores mostrados en la ventana principal serán el número total de llamadas realizadas a la función, script o evento y el tiempo total empleado (los tiempos están en microsegundos) a lo largo del perfil. Sin embargo, si lo activas, estas columnas muestran el tiempo medio empleado por paso para cada función script o evento, y el número medio de veces que fue llamado.


Ver Modo

Este menú le permite elegir el modo de vista, que puede ser de arriba a abajo o de abajo a arriba.

Seleccionando el modo de vista Top Down se mostrará el perfil de arriba hacia abajo (más o menos lo mismo que la jerarquía de la pila de llamadas) por lo que se mostrará el evento, luego el script y luego cualquier función utilizada. Tenga en cuenta que puede hacer doble clic en en cualquiera de las entradas para que se abra en la ventana de depuración de la fuente si usted tiene una. Si ves un icono de junto a un nombre, entonces hay más script o llamadas a funciones dentro de esa sección, y al hacer clic en se expandirá el árbol para mostrarlas.

Seleccionando el modo de vista de abajo hacia arriba se mostrarán las cosas de abajo hacia arriba para que pueda ver todas las funciones y las llamadas a script individualmente. Cuando se utiliza este modo de vista, al hacer clic en el se expandirá la función o el script para mostrar lo que realmente lo llamó.


Ver objetivo

El objetivo de la vista le permite elegir entre ver el proyecto GML (datos de eventos y código), los procesos del motor de fondo o ambos. Si elige "GML", obtendrá los eventos, funciones y scripts que se llaman en cada paso, mientras que la vista "Motor" sólo mostrará las llamadas al motor requeridas por el proyecto, permitiéndole ver cualquier cuello de botella en la forma en que su proyecto está manejando las cosas. Si selecciona "Ambos", entonces se mostrarán ambos conjuntos de datos pero tenga en cuenta que cuando se combina con la vista "Bottom Up", las diferentes llamadas a script y a las funciones estarán anidadas dentro de las llamadas al motor y tendrá que hacer clic en para expandirlas y verlas.



Los datos del perfil se muestran en cuatro columnas, y al hacer clic en cualquiera de ellas se ordenan los datos según el criterio de la columna.

  • Nombre: Este es el nombre del evento, script o función que se está llamando, y puede tener un al lado para denotar más llamadas a él o desde él (dependiendo del Modo de Vista que hayas seleccionado). Puedes hacer doble clic en sobre cualquiera de los nombres para que se abra el bloque de código o script dentro de la ventana Fuente. Si haces clic en la parte superior de esta columna, los nombres se ordenarán por orden alfabético.
  • Tiempo: mostrará el tiempo total que ha tardado una función (o script o evento) durante la elaboración del perfil, o el tiempo medio que ha tardado cada paso en ser llamado (dependiendo de la opción de visualización de tiempo seleccionada) y se muestra en microsegundos.
  • Llamadas: Aquí puede ver el número total de llamadas al evento, función o script dados, o el número medio de llamadas en un solo paso (de nuevo, dependiendo de la opción de visualización de tiempo seleccionada).
  • Paso%: El valor que se muestra aquí es el porcentaje global de un solo paso que ha dado la llamada.
